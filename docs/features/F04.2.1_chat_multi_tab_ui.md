> **ステータス**: 🟢 設計完了

> **最終更新**: 2026-04-25
> **関連ドキュメント**: F04.2 チャット、F11.3 UI i18n、F11.4 背景色テーマ、F11.1 PWA / オフライン

## 1. 概要

複数のチャットルームを **同一ページ内タブ** で並行参照できる UI を提供する。VSCode のエディタタブと同様、同じルームを複数タブで開くことも許可（重複タブ可）し、過去ログを見ながら別箇所で返信する等のパワーユーザー操作を支援する。あわせて、現状 PC 用2カラムレイアウトをそのままスマホで表示してしまっている問題を解消し、スマホでも同じタブ機構を **横スクロールタブバー** として提供する。

### 解決する課題

1. **スマホで2カラムが破綻** — チャンネル一覧と本文の両方が極狭になり、操作性が著しく低い
2. **ルーム横断の参照が不便** — 別ルームの内容を確認すると現在のスクロール位置や下書きが失われる
3. **複数視点での同一ルーム閲覧ができない** — 過去ログ参照と返信の同時進行ができない

### 提供価値

- PC・スマホで一貫した操作体験
- 同じルームを複数タブ＝マルチパネル参照（VSCode 互換の発想）
- 「ブラウザの新しいウィンドウで開く」相当の体験を右クリックメニューで実現

## 3. 機能要件

### 3.1 タブの基本仕様

| 項目 | 仕様 |
|---|---|
| 最大タブ数 | **10個**（11個目を開こうとすると Toast でエラー通知） |
| 重複タブ | **許可**（同じチャンネルを複数タブで開ける。VSCode 方式） |
| タブ表示順 | 開いた順に右へ追加（並べ替えは将来対応） |
| 各タブの状態 | スクロール位置・下書き・スレッド展開状態を **タブごとに独立保持** |
| タブ切替 | クリックで即座に切替（API 再フェッチなし、メモリ上に保持） |
| タブクローズ | タブの `×` ボタン、または右クリックメニュー「タブを閉じる」 |

### 3.2 `+` ボタン（タブ追加フロー）

タブバー右端の `+` ボタンを押すと、**チャンネル選択ドロップダウン** を表示する。

```
[全体連絡 ×] [役員会議 ×] [+]
                          ↓ クリック
                    ┌──────────────────────┐
                    │ [タブ] [連絡先] [申請]│  ← 既存の3タブ
                    │ ─────────────────── │
                    │ # 全体連絡           │
                    │ # 役員会議           │
                    │ # 雑談              │
                    │ 👤 田中太郎(DM)      │
                    │ 👤 鈴木花子(DM)      │
                    └──────────────────────┘
```

- 既存の `ChatChannelList` / `ContactList` / `ContactRequestList` を **そのまま流用**（ドロップダウン内に埋め込む）
- 選択するとタブが追加され、選択中タブが新タブにフォーカス移動
- 既に開いているチャンネルでも、もう1タブ追加される（重複タブ許可）
- ドロップダウン外クリックで閉じる
- **位置と表示**:
  - `+` ボタンはタブバー右端に **固定表示**（`position: sticky; right: 0` でタブの横スクロール対象外）
  - 背景はタブバーと同色 + 左側にグラデーション（タブが下に流れているのが見えるよう薄い影）
- **デフォルトサブタブ**: 開いた直後は `[タブ]`（チャンネル一覧）。最後に選択したサブタブを `sessionStorage` に保存し、次回オープン時に復元
- **タブ最大数到達時の挙動**: `+` ボタンは `disabled` 状態（薄表示・カーソル `not-allowed`）にし、ホバー時のツールチップで「タブは最大10個までです」を表示。クリックは Toast でも同メッセージ

### 3.3 右クリックコンテキストメニュー

タブを右クリックすると以下のメニューを表示。

| メニュー項目 | 動作 |
|---|---|
| 別ウィンドウで開く | `window.open('/chat?channel={id}')` で新規ブラウザウィンドウを開き、自タブからは閉じる |
| 他のタブを閉じる | 当該タブ以外を全て閉じる |
| 右側のタブを全て閉じる | 当該タブの右側のタブを全て閉じる |
| タブを閉じる | 当該タブのみ閉じる（`×` ボタンと同等） |

スマホ（タッチ環境）では右クリック相当を **長押し（500ms）** で発動する。

### 3.4 スマホレイアウト

| 画面幅 | レイアウト |
|---|---|
| `< 768px`（スマホ） | フルスクリーン1カラム。タブバー上部固定。`+` でドロップダウンを **モーダル**（フルスクリーン）として表示 |
| `≥ 768px`（タブレット・PC） | 既存の2カラム（左サイドバー + 右本文）に加えて、本文上部にタブバー。`+` は左サイドバーをそのまま使う動線 + 右上の専用ボタン両方からアクセス可能 |

**スマホでの `+` ドロップダウン**: 画面が狭いため Bottom Sheet ライクなモーダル表示。タブ・連絡先・申請の3タブを内部に持ち、選択でモーダルを閉じてタブが追加される。

### 3.5 タブの未読バッジ

- 各タブに `chat_channel_members.unread_count` を表示
- 同一チャンネルの重複タブが複数あっても、未読数は **チャンネル単位で同じ値** を全タブに表示（チャンネル状態に紐づくため）
- アクティブタブを開いている間は未読が増えない（既読扱い）→ 既存仕様準拠
- バッジは `9+` で打ち切り（多数の数字でタブ幅が膨張するのを防ぐ）
- **下書きインジケータ**: タブのチャンネルで未送信の下書き（メッセージ入力欄に文字あり）がある場合、タブのラベルの右端に **小さなドット (`•`)** を未読バッジとは別色（黄色）で表示。下書きが空になれば消える
- **非アクティブタブが新メッセージを受信した場合**: 未読バッジの数値が更新され、自動スクロールは行わない。タブ切替で当該タブをアクティブにした時点で既読扱い + 自動的に最下部へスクロール（既存 ChatMessagePanel の挙動準拠）

### 3.6 タブの保持と復元

| タイミング | 挙動 |
|---|---|
| ページリロード | `localStorage` から最後の状態（開いているタブ ID + 順序 + アクティブタブ）を復元 |
| ログアウト | `localStorage` の該当キーをクリア |
| 別ユーザーログイン | `localStorage` キーをユーザー ID で名前空間化（`chatTabs:userId={id}`）。前ユーザーのタブが見えない |
| タブ復元時にチャンネルがアクセス不可 | そのタブのみスキップして他のタブを開く（403/404 を Silent ハンドリング） |

### 3.7 WebSocket 購読の最適化

- 同じチャンネルを N 個のタブで開いても、**WebSocket SUBSCRIBE は1チャンネル1回**（重複購読を避ける）
- メッセージ受信時はチャンネルを監視している全タブに Pinia 経由で配信
- タブを閉じて当該チャンネルが残タブで参照されなくなった場合 UNSUBSCRIBE
- メッセージ受信用の `/topic/channels/{id}` に加えて、**メタイベント受信用の `/topic/channels/{id}/events` も追加購読する**（kick・削除・アーカイブ等のチャンネル状態変化を受信。詳細は §3.10）

### 3.8 タブの自動オープン契機

| 契機 | 挙動 |
|---|---|
| 連絡先から DM 開始 | 新規 DM チャンネルを自動でタブ追加・フォーカス |
| プッシュ通知タップ | 当該チャンネルを **既存タブで参照中ならフォーカス**、なければ新規タブで開く |
| 予約成立・マッチング成立（自動作成） | 通知バナーから「開く」をタップで新規タブ |
| 別ウィンドウから戻った場合 | 新規タブとしては自動追加しない（明示操作が必要） |
| URL クエリ `?channel={id}` での遷移 | 既存タブで開いていればフォーカス、なければ新規タブを追加（クエリパラメータ消費後は履歴に残さず `replaceState` で URL を `/chat` に戻す） |

**URL クエリでの自動オープン仕様**:
- 既存の `pages/chat/index.vue` も `route.query.dm` 等で同様の連動を実装済み
- 本機能では `?channel={id}` を統一窓口とし、各機能（タイムライン引用、メンション通知、予約カード、マッチング成立画面など）からの遷移はこのクエリパラメータで指定する
- 不正な ID（数値以外、存在しない、アクセス不可）は Silent スキップ + Toast「指定されたチャンネルにアクセスできません」

### 3.9 タブを開いた直後のフォーカス挙動

| 状況 | フォーカス先 |
|---|---|
| `+` ドロップダウンから選択して新規タブ追加 | メッセージ入力欄に自動フォーカス（タッチデバイスではキーボードを開かないよう `inputmode` のみ整え、自動 focus は呼ばない） |
| 既存タブをクリックで切替 | 入力欄のフォーカス状態を維持（前回入力中なら継続） |
| URL クエリで自動オープン | フォーカスは入力欄に当てない（受信側でユーザーがアクションを選べる状態） |
| キーボードショートカットで切替 | フォーカスは入力欄に当てない（連続切替操作の妨げを防ぐ） |

### 3.10 タブの自動クローズ・無効化

| 契機 | 挙動 |
|---|---|
| 自分が当該チャンネルから kick された | 当該タブを自動で閉じ、Toast で通知 |
| チャンネルが削除された | 同上 |
| チャンネルがアーカイブされた | タブ自体は閉じず、入力欄を無効化（既存 F04.2 の閲覧専用挙動） |
| ネットワーク切断 | タブ状態は維持、再接続後に最新メッセージを取得 |

#### 3.10.1 サーバーからのイベント送信仕様（Phase 10）

サーバーは以下のタイミングで `/topic/channels/{id}/events` トピックへ STOMP メッセージを送信する。

| 契機 | ペイロード | 送信タイミング |
|---|---|---|
| メンバーが kick された | `{ "type": "MEMBER_KICKED", "userId": <kickedUserId> }` | `ChatMemberService` の kick 処理コミット直後 |
| チャンネルが削除された | `{ "type": "CHANNEL_DELETED" }` | `ChatChannelService.deleteChannel` のコミット直後 |
| チャンネルがアーカイブされた | `{ "type": "CHANNEL_ARCHIVED" }` | `ChatChannelService.archiveChannel` のコミット直後 |
| アーカイブ解除 | `{ "type": "CHANNEL_UNARCHIVED" }` | `ChatChannelService.unarchiveChannel` のコミット直後 |

ペイロードの TypeScript 型（フロント側）:

```typescript
export type ChatChannelEvent =
  | { type: 'MEMBER_KICKED'; userId: number }
  | { type: 'CHANNEL_DELETED' }
  | { type: 'CHANNEL_ARCHIVED' }
  | { type: 'CHANNEL_UNARCHIVED' }
```

#### 3.10.2 クライアントの受信ハンドラ

`useChatApi` の `subscribeChannel(id)` は `/topic/channels/{id}` と `/topic/channels/{id}/events` の **2系統** を購読する。
受信した `ChatChannelEvent` は `useEventBus<ChatChannelEvent>('chat:channel-event')` で emit する（既存の `chat:message` バスとは別）。

`pages/chat/index.vue` で当該イベントバスを listen し、以下の処理を行う:

| イベント type | フロント挙動 |
|---|---|
| `MEMBER_KICKED` （userId が自分の場合のみ） | 該当 channelId のタブを全て自動クローズ（重複タブも全閉じ）・Toast `chat.tab.kickedNotice` |
| `CHANNEL_DELETED` | 該当 channelId のタブを全て自動クローズ・Toast `chat.tab.channelDeleted` |
| `CHANNEL_ARCHIVED` | タブは閉じず、`tab.channel.isArchived = true` を反映（入力欄は ChatMessagePanel が判定して disabled） |
| `CHANNEL_UNARCHIVED` | `tab.channel.isArchived = false` を反映 |

## 4. UI設計

### 4.1 タブバーコンポーネント

新規コンポーネント `ChatTabBar.vue` を作成。

```
┌─────────────────────────────────────────────────────────────┐
│ [全体連絡 (3) ×] [役員会議 ×] [雑談 ×] [+]              ⋮  │
└─────────────────────────────────────────────────────────────┘
                                                          ↑
                                                    タブメニュー
                                                    （任意）
```

- タブ要素1個あたり最小幅 120px、最大幅 200px、ellipsis で省略
- 横スクロール: タブが画面幅を超えたら横スクロール可（`overflow-x: auto`、Tailwind `snap-x`）
- スマホでもスクロールバーは細く、タッチでスワイプ可能
- アクティブタブは下線 + 背景色強調、ホバー時は背景色淡色化
- 未読バッジは数字（`9+` 打ち切り）。`@mention` 受信中は赤背景、それ以外は青背景

### 4.2 メイン構造（`pages/chat/index.vue` リファクタ）

```vue
<template>
  <div class="chat-page">
    <!-- ① タブバー（PC・スマホ共通） -->
    <ChatTabBar
      :tabs="tabs"
      :active-tab-id="activeTabId"
      @switch="switchTab"
      @close="closeTab"
      @open-add="openAddDropdown"
      @context-menu="onContextMenu"
    />

    <!-- ② 本体（タブごとの ChatMessagePanel） -->
    <div class="chat-body">
      <!-- PC のみ表示するサイドバー -->
      <aside v-if="isDesktop" class="chat-sidebar">
        <ChatChannelList ... />  <!-- 既存流用 -->
      </aside>

      <!-- タブごとに ChatMessagePanel を保持（v-show で切替して状態保持） -->
      <div class="chat-main">
        <ChatMessagePanel
          v-for="tab in tabs"
          v-show="tab.id === activeTabId"
          :key="tab.id"
          :channel="tab.channel"
        />
        <ChatEmptyState v-if="tabs.length === 0" />
      </div>
    </div>

    <!-- ③ + ボタンドロップダウン / モーダル -->
    <ChatAddTabDropdown
      v-if="showAddDropdown"
      :is-mobile="!isDesktop"
      @select="addTab"
      @close="showAddDropdown = false"
    />

    <!-- ④ 右クリックメニュー -->
    <ChatTabContextMenu
      v-if="contextMenu.show"
      :tab-id="contextMenu.tabId"
      :position="contextMenu.position"
      @action="onContextMenuAction"
      @close="contextMenu.show = false"
    />
  </div>
</template>
```

`<script setup>` 抜粋:
```typescript
const { tabs, activeTabId, addTab, closeTab, switchTab } = storeToRefs(useChatTabsStore())
const { width } = useWindowSize()  // VueUse
const isDesktop = computed(() => width.value >= 768)
const showAddDropdown = ref(false)
const contextMenu = reactive({ show: false, tabId: null as string | null, position: { x: 0, y: 0 } })

// タブが0個の状態でページに来た場合のヒント表示は ChatEmptyState 側で完結
// URL クエリ ?channel={id} を消費して自動オープン
const route = useRoute()
const router = useRouter()
onMounted(async () => {
  await useChatTabsStore().restore()
  const cid = Number(route.query.channel)
  if (Number.isInteger(cid) && cid > 0) {
    await openByChannelId(cid)  // 既存タブにフォーカス or 新規追加
    router.replace({ query: {} })  // クエリ消費
  }
})
```

**重要設計**: タブごとの `ChatMessagePanel` を `v-if` ではなく **`v-show`** で切り替える。これにより：
- スクロール位置・入力中下書き・スレッド展開状態が保持される
- 切替時の API 再フェッチが不要

ただしタブ数 × メッセージ数のメモリ消費が増えるため、タブ上限 10 + 各 ChatMessagePanel 内のメッセージ仮想スクロール（既存 or 追加実装）で歯止め。

### 4.3 レスポンシブブレークポイント

| 幅 | レイアウト |
|---|---|
| `< 768px` | タブバー上部固定 + 本文フルスクリーン。サイドバーは `+` ドロップダウン（モーダル）でのみアクセス |
| `768px - 1023px`（タブレット） | タブバー + サイドバー幅 220px + 本文 |
| `≥ 1024px`（PC） | タブバー + サイドバー幅 256px + 本文 |

Tailwind の `md:` `lg:` プレフィックスで実装。

### 4.4 既存 ChatMessagePanel の再利用前提

タブの本体表示は既存 `frontend/app/components/chat/ChatMessagePanel.vue` をそのまま使用する。タブ化に伴う改修は **入れない**:
- メッセージ表示・送信・編集・削除・リアクション・スレッド返信・ピン留め・添付・予約送信・転送・全文検索・WebSocket リアルタイム更新・ファイルアップロード等の F04.2 既存機能は **すべてタブ化後も維持**
- F03.4 予約チャット連携の `source_data` カード上部表示も維持（タブ切替で別の予約チャットを開けば、対応する予約カードが上部に表示される）
- F04.9 確認通知の連動も維持

タブ機構は ChatMessagePanel を `v-show` で並列保持する **外側のシェル** として実装し、Panel 内部のロジックには手を入れない。

### 4.5 テーマ連動（F11.4 背景色テーマ）

タブバーの色はテーマトークンに連動:
- 非アクティブタブ背景: `bg-surface-50 dark:bg-surface-800`
- アクティブタブ背景: `bg-surface-0 dark:bg-surface-900`（本文と同色）
- 下線（アクティブインジケータ）: `border-primary`
- 未読バッジ背景: `bg-primary` または `bg-red-500`（@mention 受信中）
- ドラフトインジケータ: `bg-yellow-400`

ユーザーが背景色を変更しても自動追従。HEX 直書きは禁止。

### 4.6 アイコン・ラベル

| 要素 | 表示 |
|---|---|
| TEAM チャンネル | `#` + チャンネル名 |
| ORGANIZATION チャンネル | `#` + チャンネル名 |
| CROSS_TEAM チャンネル | `🔗` + チャンネル名 |
| DIRECT (DM/Kabine) | `👤` + 相手ユーザー名 |
| GROUP_DM (Zimmer) | `👥` + チャンネル名 or 参加者名連結 |
| アーカイブ済み | `🗄️` プレフィックス |

## 5. 状態管理（Pinia ストア）

### 5.1 新規ストア: `useChatTabsStore`

```typescript
// frontend/app/stores/useChatTabsStore.ts
export const useChatTabsStore = defineStore('chatTabs', () => {
  const MAX_TABS = 10
  const tabs = ref<ChatTab[]>([])
  const activeTabId = ref<string | null>(null)

  // タブの追加（重複許可）
  function addTab(channel: ChatChannelResponse): { ok: boolean; reason?: string } {
    if (tabs.value.length >= MAX_TABS) {
      return { ok: false, reason: 'MAX_TABS_REACHED' }
    }
    // crypto.randomUUID() は最新ブラウザのみ。PWA は最新ブラウザを前提とするが
    // フォールバックとして簡易乱数生成を用意（v4 形式に整形）
    const uuid = (typeof crypto !== 'undefined' && 'randomUUID' in crypto)
      ? crypto.randomUUID()
      : generateUuidV4Fallback()
    const tab: ChatTab = {
      id: uuid,
      channelId: channel.id,
      channel,                         // ChatMessagePanel 用にスナップショット保持
      createdAt: Date.now(),
    }
    tabs.value.push(tab)
    activeTabId.value = tab.id
    persist()
    return { ok: true }
  }

  function closeTab(tabId: string) {
    const idx = tabs.value.findIndex(t => t.id === tabId)
    if (idx < 0) return
    tabs.value.splice(idx, 1)
    if (activeTabId.value === tabId) {
      // 直前のタブにフォーカス（なければ次、なければ null）
      activeTabId.value = tabs.value[idx - 1]?.id ?? tabs.value[idx]?.id ?? null
    }
    persist()
  }

  function switchTab(tabId: string) {
    if (tabs.value.some(t => t.id === tabId)) {
      activeTabId.value = tabId
      persist()
    }
  }

  // 現在開いているタブで使われているチャンネルID（WebSocket SUBSCRIBE 判定用）
  const openChannelIds = computed(() => {
    return new Set(tabs.value.map(t => t.channelId))
  })

  // localStorage 永続化（ユーザーIDで名前空間化）
  function persist() {
    const auth = useAuthStore()
    if (!auth.userId) return
    localStorage.setItem(
      `chatTabs:userId=${auth.userId}`,
      JSON.stringify({
        tabs: tabs.value.map(t => ({ id: t.id, channelId: t.channelId })),
        activeTabId: activeTabId.value,
      })
    )
  }

  const UUID_V4_RE = /^[0-9a-f]{8}-[0-9a-f]{4}-4[0-9a-f]{3}-[89ab][0-9a-f]{3}-[0-9a-f]{12}$/i

  async function restore() {
    const auth = useAuthStore()
    if (!auth.userId) return
    const key = `chatTabs:userId=${auth.userId}`
    const raw = localStorage.getItem(key)
    if (!raw) return
    let saved: { tabs: { id: string; channelId: number }[]; activeTabId: string | null }
    try {
      saved = JSON.parse(raw)
    } catch {
      localStorage.removeItem(key)
      return
    }
    // 入力バリデーション（攻撃耐性）
    const validTabs = (saved.tabs ?? []).filter(t =>
      typeof t?.id === 'string' && UUID_V4_RE.test(t.id) &&
      Number.isInteger(t?.channelId) && t.channelId > 0
    )
    const api = useChatApi()
    // 並列フェッチでパフォーマンス担保
    const results = await Promise.allSettled(
      validTabs.map(t => api.getChannel(t.channelId))
    )
    const restored: ChatTab[] = []
    results.forEach((r, i) => {
      if (r.status === 'fulfilled') {
        const t = validTabs[i]
        restored.push({ id: t.id, channelId: r.value.id, channel: r.value, createdAt: Date.now() })
      }
      // rejected (403/404 等) はスキップ
    })
    tabs.value = restored
    activeTabId.value = saved.activeTabId && restored.some(t => t.id === saved.activeTabId)
      ? saved.activeTabId
      : restored[0]?.id ?? null
    // スキップされたタブがあればユーザーに通知
    const skipped = validTabs.length - restored.length
    if (skipped > 0) {
      useToast().info(`${skipped}件のタブが利用できなくなったため閉じました`)
    }
  }

  function clearAll() {
    tabs.value = []
    activeTabId.value = null
    const auth = useAuthStore()
    if (auth.userId) {
      localStorage.removeItem(`chatTabs:userId=${auth.userId}`)
    }
  }

  return { tabs, activeTabId, addTab, closeTab, switchTab, openChannelIds, restore, clearAll }
})
```

  channelId: number                 // チャンネルID

## 7. セキュリティ観点

### 7.1 XSS 対策

- タブラベル（チャンネル名）はサーバーから取得する `channel.name` をそのまま `{{ }}` で表示（Vue の自動エスケープを利用、`v-html` 一切不使用）
- DM 相手の表示名も同様
- 右クリックメニューのテキストは i18n キーから生成、ユーザー入力を含まない

### 7.2 localStorage 経由の情報漏洩

- 永続化するデータは **タブ固有 UUID + チャンネル ID（数値）+ アクティブタブ ID のみ**。チャンネル本文・添付・パスワード等の機密情報は一切含めない
- 永続化対象の網羅:
  - ✅ 含める: `tab.id`（UUID）、`tab.channelId`（数値）、`activeTabId`（UUID）
  - ❌ 含めない: `channel` オブジェクト全体（メンバー一覧・最新メッセージプレビュー等を含むため）、ドラフト本文、添付候補、リアクションキャッシュ
- ユーザー ID で名前空間化（`chatTabs:userId={id}`）し、別ユーザーのタブが他ユーザーセッションで復元されるのを防止
- ログアウト時に該当キーを `clearAll()` で削除（`useAuthStore.logout` のフックで呼び出す）
- 復元時に各チャンネルの認可は **API 呼び出し側で再検証**（`GET /chat/channels/{id}` が 403/404 を返したらタブ追加せず Silent スキップ）
- **入力バリデーション**: localStorage から復元する際、以下の検証を行う:
  - JSON.parse 失敗時は黙ってクリア
  - `channelId` が `Number.isInteger() && > 0` でない場合は当該タブをスキップ
  - `tab.id` が UUID v4 形式（正規表現 `/^[0-9a-f]{8}-[0-9a-f]{4}-4[0-9a-f]{3}-[89ab][0-9a-f]{3}-[0-9a-f]{12}$/i`）でない場合は当該タブをスキップ
  - `activeTabId` が tabs に存在しない場合は `tabs[0]?.id` にフォールバック
  - 異常データはコンソールに warn して全体クリア（攻撃者の細工に耐性を持たせる）

### 7.3 別ウィンドウで開く（`window.open`）

- `window.open(url, '_blank', 'noopener,noreferrer')` で新規ウィンドウを開く（`noopener` で `window.opener` 経由の親ウィンドウ操作を防止）
- URL は同一オリジン固定（`/chat?channel={id}` のフォーマットで生成。任意 URL を開かない）
- ポップアップブロックされた場合（戻り値 `null`）は Toast「ポップアップを許可してください」と案内
- CSP（Content Security Policy）対応: 本機能はインラインスクリプトを使用しない（Pinia ストア経由・コンポーネント定義済みのイベントハンドラのみ）。既存プロジェクトの CSP 設定の影響を受けない

### 7.4 タブの権限境界

- タブを開いている最中にロール変更（昇格・降格）が発生した場合、本文の操作可否は **API 認可で再検証**（既存 F04.2 の挙動）
- タブの存在自体は権限変更で消えない（メンバーシップ喪失時のみ自動クローズ）

### 7.5 CSRF / 認可

- 本機能はフロントエンドの状態管理のみで、新規 API は追加しない
- 既存 F04.2 の認可ルールに完全準拠

## 8. ユーザビリティ観点

### 8.1 視覚的フィードバック

- タブにマウスホバー時、`×` ボタンを表示（非ホバー時は薄く）
- アクティブタブは背景色強調 + 下線。アクティブ以外との視覚差を明確に
- タブを開く / 閉じる時にタブバーが横に詰まる挙動はトランジション 150ms で滑らかに
- 重複タブを開いた時、新規タブにフォーカスが切り替わる（混乱しない）

### 8.2 エラー回復

| エラー | UX |
|---|---|
| 11個目のタブを開こうとした | Toast「タブは最大10個までです」+ 操作キャンセル |
| ネットワーク切断時にタブ追加 | Toast「ネットワークに接続できません」+ 操作キャンセル |
| `+` ドロップダウンで権限のないチャンネルを選択 | 表示しない（ChatChannelList が既に絞り込み済み） |
| タブ復元時にチャンネルアクセス不可 | Silent スキップ + Toast「{N}件のタブが利用できなくなったため閉じました」 |

### 8.3 タッチデバイスでの右クリック相当

- タブを **長押し（500ms）** でコンテキストメニューを表示
- 長押し中は触覚フィードバック（`navigator.vibrate(20)` 対応端末のみ）
- 長押し中は対象タブをハイライト表示
- **iOS Safari の長押し対策**: タブ要素に `-webkit-touch-callout: none` と `user-select: none` を CSS で指定。これがないと iOS Safari で長押し時にブラウザネイティブのコピー / 共有メニューが先に発火してしまう
- **Android Chrome の長押し対策**: `touch-action: manipulation` を指定し、ダブルタップズーム等の干渉を抑制
- カスタム長押し検出は `touchstart` で `setTimeout`、`touchmove` / `touchend` でキャンセル。スクロール意図と区別するため、`touchmove` の移動量が10px超なら長押し扱いしない

### 8.4 スマホでの「+」ドロップダウン

- 画面下端から立ち上がる Bottom Sheet 風モーダル
- ヘッダーに「閉じる」ボタン + ドラッグハンドル
- 既存の `ChatChannelList` / `ContactList` / `ContactRequestList` を切り替えるサブタブ付き

### 8.5 タブクローズ時の保護

- 入力欄に **下書きが残っている** タブを `×` で閉じようとした場合、確認ダイアログ「入力中のメッセージがあります。本当に閉じますか?」を表示し、「破棄して閉じる」「キャンセル」の2択
- ただし以下では確認なしで即クローズ（過剰な摩擦を避ける）:
  - kick / チャンネル削除 / アーカイブによる **自動クローズ**
  - 右クリック「他のタブを閉じる」「右側のタブを全て閉じる」（多数閉じる際に毎回確認すると煩雑）
- **空タブ（下書きなし）の閉じる操作には確認なし**

### 8.6 ナビゲーションとの整合性

- 右上ハンバーガーメニュー（既存）からチャットページに来た場合、タブ状態は保持される
- 別ページに遷移して戻ってきた場合、Pinia ストアでタブ状態を保持（ストアは `localStorage` 経由でも維持される）
- ブラウザの「戻る」ボタン: チャットページ内のタブ切替は履歴に積まない（URL は `/chat` 固定。ハッシュも変更しない）。これにより「戻る」を押すと前ページへ戻る挙動が直感的

### 8.7 タブの過密対策

- タブが10個近くなると視認性が低下するため、各タブのラベルは ellipsis で省略
- タブを多く開いて混乱した時のために「全て閉じる」操作を用意（タブバー右端の ⋮ メニュー内）
- チャンネル名が極端に長い（最大100文字）場合、タブラベルは最大幅 200px で `text-overflow: ellipsis`。ホバー時に完全名を `title` 属性で表示
- アクティブタブはやや幅広（最大 240px）で表示し、識別しやすくする

- ユーザーは `+` ボタンまたは（PC では）サイドバーから普通にチャンネルを選択 → 自動でタブ追加
- 移行期間用のオンボーディング表示は不要（既存ユーザーが従来通りクリックすれば自然にタブ機構を体験できる）
