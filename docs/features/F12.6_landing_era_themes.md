# F12.6: ランディングページ年代別テーマ（イースターエッグ）

> **ステータス**: 🟢 設計確定
> **実装フェーズ**: Phase 12+
> **最終更新**: 2026-04-07
> **関連ドキュメント**: F11.3 UI i18n, F12.5 フロントエンドエラートラッキング

---

## 1. 概要

未ログインユーザー向けのランディングページ（`/`）に **8種類の年代別レトロテーマ** をイースターエッグとして実装する。通常はモダンデザインを表示するが、低確率（1/100）でランダムに過去のWebデザイン年代風／レトロゲーム機風のテーマが発動する遊び心のある機能。

**狙い**:
- 初訪問ユーザーへの**サプライズ体験**による記憶定着・SNS拡散効果
- ブランドの**遊び心**を表現し、堅苦しさを和らげる
- 開発者・デザインオタク向けの**カルチャー的訴求**
- 「Mannschaft の隠しテーマ見たことある？」がコミュニティで話題になる**バイラル効果**

**設計思想**:
- 通常体験（モダン版・SEO 版）を**絶対に損なわない**
- レトロテーマは**いつ会えるかわからない伝説**として希少性を保つ
- ユーザーが望めば**即座に脱出**できる安全策を備える

## 3. テーマカタログ

通常表示の Modern を含めて全 **9 テーマ**。レトロ 8 種は同じ確率で抽選される。

| ID | 名称 | 年代 | キーワード | 難易度 |
|----|------|------|-----------|--------|
| `modern` | モダン | 2026 (現代) | 既存 LandingHero/Stats/... 7 セクション | 既存 |
| `y1998` | Geocities Era | 1998 | テーブルレイアウト・虹色グラデ・Comic Sans・marquee 風スクロール | ★★★ |
| `y2000` | Glass Era | 2000 | 青銀ガラス・Flash 風・左サイドナビ | ★★ |
| `y2005` | Web 2.0 Era | 2005 | 丸角・反射・ベビーブルー・「Beta」バッジ・リボン | ★★ |
| `y2010` | Skeuomorphism Era | 2010 | 皮テクスチャ・ぷっくりボタン・iOS 6 風シャドウ | ★★★ |
| `y2015` | Material Era | 2015 | フラット・elevation・Roboto・Material Design | ★ |
| `y2020` | Neumorphism Era | 2020 | ニューモーフィズム・ダーク・Inter・淡いグラデ | ★ |
| `fc` | ファミコン | 1985 | 8bit ピクセルアート・4色制限・Press Start 2P・PUSH START 点滅 | ★★★★ |
| `sfc` | スーパーファミコン | 1990 | 16bit パララックス・ドラクエ風青枠ウィンドウ・▼コマンド選択 | ★★★★ |

### 各テーマの再現要素詳細

#### `y1998` (Geocities 全盛期)
- **背景**: 黒地 + 星空 GIF風（CSS animation で点滅）/ 虹色グラデーション帯
- **フォント**: Comic Sans MS / Times New Roman
- **テーブルレイアウト**: `<div>` を `display: table` で組み、`border="3"` 風の凹凸 3D 枠
- **訪問者カウンター**: 「You are visitor #001337」風の 7セグ風 GIF 風数字（CSS）
- **マーキー**: HTML5 非推奨タグは使わず CSS `@keyframes` で再現
- **リンク色**: `#0000EE` / 訪問済 `#551A8B` / 強制下線
- **必須記載**: 「Best viewed in Netscape Navigator 4.0」「Made with Notepad」「Under Construction 🚧」
- **謎要素**: 工事中の GIF 風アニメ（CSS）、ダンス踊るベビー（自前 ASCII or SVG）

#### `y2000` (Web 1.0 後期 / Flash 全盛期)
- **配色**: 青銀グラデ（`#003366` → `#0099FF` → `#FFFFFF`）
- **フォント**: Verdana / Tahoma / Trebuchet MS
- **レイアウト**: 左サイドバーナビ（200px 固定）+ 右コンテンツエリア
- **ボタン**: ガラス調（白い反射ハイライト + ベベル）
- **アイコン**: 16×16 の擬似 favicon 風 SVG
- **コーナー**: 「Get Macromedia Flash Player」風バナー風プレースホルダ

#### `y2005` (Web 2.0 革命期)
- **配色**: ベビーブルー（`#A8D8F0`）+ パステルグリーン + 鮮やかオレンジ
- **フォント**: Helvetica Neue / Lucida Grande
- **特徴**: グラデーション + 丸角 + リフレクション
- **ボタン**: 大きな緑「サインアップ」ボタン（リフレクション付き）
- **必須**: 「BETA」リボンバッジ、「ジェリービーンズ風」ロゴ
- **その他**: ストライプ背景、星 5 つレビュー、ソーシャル「Digg this!」風ボタン

#### `y2010` (スキューモーフィズム黄金期)
- **配色**: ベージュ・ブラウン・ダークグレー
- **フォント**: Helvetica / Lucida Sans
- **特徴**: 皮テクスチャ背景、糸縫い目、ぷっくりボタン、影
- **ボタン**: 強いインナーシャドウ + 外側ドロップシャドウ
- **必須**: メモ帳風コンテンツエリア、コルクボード背景

#### `y2015` (マテリアル / フラット元年)
- **配色**: Material Design パレット（`#2196F3` Blue 500, `#F44336` Red 500 等）
- **フォント**: Roboto
- **特徴**: フラット、elevation 影（z-index 1〜24）
- **ボタン**: FAB（Floating Action Button）+ Ripple エフェクト
- **必須**: カードレイアウト、明快な hierarchy

#### `y2020` (ニューモーフィズム + ダーク)
- **配色**: ダークモード（`#1a1a2e` 系）+ 淡いブルー紫アクセント
- **フォント**: Inter / SF Pro Display
- **特徴**: ソフトシャドウ（凸/凹）、グラスモーフィズム
- **ボタン**: 凸型ボタン、ホバーで凹む
- **必須**: 余白多め、洗練されたミニマリズム

#### `fc` (ファミコン)
- **解像度感**: 256×240 のドット感を出す（image-rendering: pixelated）
- **色**: NES パレット 52色から選んだ 4色のみ使用
- **フォント**: Google Fonts `Press Start 2P`
- **タイトル画面**:

  │                            │
  │     M A N N S C H A F T    │
  │                            │
  │      © 2026 ANTHROPIC      │
  │                            │
  │       PUSH START           │  ← 点滅
  │                            │
  └────────────────────────────┘

- **背景**: 8×8 タイルの繰り返し（雲タイル / レンガタイル）
- **ボタン**: 黒枠 1px + 内側カラー塗り、ホバーで反転
- **コナミコマンド**: `↑↑↓↓←→←→BA` 入力で隠しメッセージ表示
- **音（任意・デフォルト OFF）**: Web Audio API でファミコン風矩形波 SE

- **色**: 16-bit カラー（グラデ可）
- **フォント**: `DotGothic16` (Google Fonts) / `k8x12` 風
- **タイトル画面**: パララックス青空 + 雲（マリオワールド風）
- **メニュー**: ドラクエ風青枠ウィンドウ

- **背景**: 16-bit 風グラデ空 + パララックス丘陵
- **モダン版へ戻すボタン**: 「セーブしますか? ▶ はい / いいえ」風の確認ダイアログ

## 4. 確率設計

### 抽選アルゴリズム

```
1. SSR 中（typeof window === 'undefined'）→ 必ず modern を返す
2. クローラ User-Agent を検出 → 必ず modern を返す（クエリより優先）
3. ?era=<theme_id> クエリが有効値 → そのテーマを強制
4. Math.random() を実行
   - 値 < 0.01 (確率 1%)  → レトロテーマ抽選フェーズへ
   - 値 >= 0.01 (確率 99%) → modern を表示
5. レトロテーマ抽選: 8テーマから等確率で1つを選ぶ (Math.floor(Math.random() * 8))
```

### 確率まとめ

| イベント | 確率 | 体感 |
|---|---|---|
| `modern` を見る | 99% | 通常表示 |
| **何らかのレトロテーマを見る** | 1% | 100アクセスに1回 |
| 個別テーマ（各レトロ）を見る | 0.125% | **約1/800** |

### 強制表示（デバッグ + イースターエッグ共有用）

URL クエリ `?era=<theme_id>` で任意のテーマを強制表示できる:

```
https://mannschaft.example/?era=fc       # ファミコン版を強制
https://mannschaft.example/?era=y1998    # Geocities 風を強制
https://mannschaft.example/?era=modern   # モダン版を強制
```

- 強制表示時もコンソールログ・脱出ボタンの仕様は通常通り
- 不正なテーマ ID が指定された場合は `modern` にフォールバック
- クエリ強制表示時は SEO クローラ判定をバイパスせず、`modern` を優先（後述）

### コンポーネント参照規約（命名衝突を防ぐルール）

Nuxt 3 の auto-import は `themes/` 配下のファイルに対して `<LandingThemesLandingTheme1998 />` のような冗長で壊れやすい名前を生成するため、**テーマコンポーネントは `pages/index.vue` で動的 import（switch/case + 固定文字列）で参照する**ルールに統一する。

| 配置 | 参照方法 |
|---|---|
| `components/landing/EraThemeSwitcher.vue` | auto-import (`<LandingEraThemeSwitcher />`) |
| `components/landing/themes/LandingTheme*.vue` | **動的 import 必須** |
| 既存 `components/landing/LandingHero.vue` 等 | auto-import (`<LandingHero />`) |

### `useEraTheme` 設計

```typescript
// frontend/app/types/eraTheme.ts
export type EraTheme =
  | 'modern'
  | 'y1998' | 'y2000' | 'y2005' | 'y2010' | 'y2015' | 'y2020'
  | 'fc' | 'sfc'

/**
 * レトロテーマ一覧。
 * `as const satisfies readonly EraTheme[]` パターンで、
 * (1) 要素型をリテラル union に絞り、かつ (2) EraTheme との整合性を型レベルで保証する。
 * Object.freeze は使わない: `as const` は TS レベルで readonly を付与し、
 * 実用上の改変はエディタ/型チェックで十分防げるため。
 */
export const RETRO_ERA_THEMES = [
  'y1998', 'y2000', 'y2005', 'y2010', 'y2015', 'y2020', 'fc', 'sfc',
] as const satisfies readonly EraTheme[]

export const ALL_ERA_THEMES = [
  'modern', ...RETRO_ERA_THEMES,
] as const satisfies readonly EraTheme[]

/** レトロテーマ全体の発動確率（0.0〜1.0）。1/100 = 0.01 */
export const RETRO_ERA_PROBABILITY = 0.01

/**
 * EraTheme 型の型ガード。
 * `includes()` を使うと型推論が緩むため、専用関数で堅牢化する。
 * `unknown` から始めて文字列型チェック → ホワイトリスト照合の2段で安全に絞り込む。
 */
export function isEraTheme(value: unknown): value is EraTheme {
  if (typeof value !== 'string') return false
  return (ALL_ERA_THEMES as readonly string[]).includes(value)
}

// frontend/app/composables/useEraTheme.ts
import { ALL_ERA_THEMES, RETRO_ERA_THEMES, RETRO_ERA_PROBABILITY, isEraTheme, type EraTheme } from '~/types/eraTheme'

export function useEraTheme() {
  const route = useRoute()

  /**
   * テーマを決定する。優先順位:
   * 1. URL クエリ ?era=xxx が有効値なら強制
   * 2. クローラ User-Agent なら 'modern' 固定（SEO 死守、クエリより優先）
   * 3. それ以外は確率抽選
   *
   * 必ずクライアント側（onMounted 後）で呼ぶこと。SSR 中の呼び出しは modern を返す。
   */
  function pickTheme(): EraTheme {
    // 0. SSR 中は常に modern（ハイドレーション不一致を防ぐ）
    if (typeof window === 'undefined') return 'modern'

    // 1. クローラ判定（クエリより優先 — SEO 詐称防止）
    if (isCrawler()) return 'modern'

    // 2. クエリ強制（配列値・複数指定・null/undefined にも対応）
    //    Nuxt 3 の route.query.era は LocationQueryValue | LocationQueryValue[] 型
    //    = string | null | (string | null)[] | undefined
    const rawQuery = route.query.era
    const queryValue: string | undefined
      = typeof rawQuery === 'string'
        ? rawQuery
        : Array.isArray(rawQuery) && typeof rawQuery[0] === 'string'
          ? rawQuery[0]
          : undefined
    if (isEraTheme(queryValue)) return queryValue

    // 3. 確率抽選
    if (Math.random() >= RETRO_ERA_PROBABILITY) return 'modern'
    const idx = Math.floor(Math.random() * RETRO_ERA_THEMES.length)
    // 上の Math.floor は 0..length-1 を返すため undefined にはならないが、
    // TypeScript の noUncheckedIndexedAccess に備えて非null断言の代わりにフォールバック
    return RETRO_ERA_THEMES[idx] ?? 'modern'
  }

  function isCrawler(): boolean {
    if (typeof navigator === 'undefined') return true // SSR 中は安全側で常に true
    const ua = navigator.userAgent.toLowerCase()
    return /bot|crawler|spider|crawling|googlebot|bingbot|slurp|duckduckbot|baiduspider|yandexbot|sogou|exabot|facebot|ia_archiver|prerender|lighthouse|headlesschrome/i.test(ua)
  }

  return { pickTheme, isCrawler }
}
```

#### 設計上の重要ポイント

| ポイント | 理由 |
|---|---|
| **クローラ判定をクエリより優先** | クローラが `?era=fc` でアクセスしても modern を返す。SEO 詐称防止 |
| **クエリは配列対応** | `?era=fc&era=sfc` のような複数指定でも最初の値だけを採用、エラーにしない |
| **`isEraTheme()` 型ガード関数** | `includes()` 直叩きは型推論が緩むため専用関数で堅牢化 |
| **`Object.freeze`** | 実行時に配列を改変できないことを保証（プロトタイプ汚染対策） |
| **SSR 中は常に modern** | `typeof window === 'undefined'` で早期 return しハイドレーション不一致を防ぐ |
| **`headlesschrome`/`lighthouse` 検出** | CI/監視で叩かれた時もモダン版を返す（テスト安定性） |

### `pages/index.vue` 設計

```vue
<script setup lang="ts">
import { defineAsyncComponent } from 'vue'
import type { EraTheme } from '~/types/eraTheme'

definePageMeta({ layout: 'landing', middleware: 'guest' })

// ... 既存の SeoMeta / JSON-LD はそのまま ...

const currentTheme = ref<EraTheme>('modern')
const { pickTheme } = useEraTheme()
const { loadFont } = useEraThemeFonts()

// SSR では常に modern（SEO 用）
// クライアント mount 後に乱数抽選 → 1% でレトロに切替
onMounted(() => {
  currentTheme.value = pickTheme()
  // 本番環境ではコンソールログを出さない（イースターエッグ希少性 + 乱数ロジック非公開）
  // import.meta.dev は Vite が dev / prod build 時に boolean リテラルに置換するため、
  // 本番 build では if ブロック全体が tree-shake で削除される（SSR/CSR 両方で安全）
  if (currentTheme.value !== 'modern' && import.meta.dev) {
    // eslint-disable-next-line no-console
    console.log(
      `%c🎉 You found a "${currentTheme.value}" easter egg!`,
      'font-size: 16px; color: #ff4081; font-weight: bold;',
    )
  }
})

// レトロフォントの遅延ロード（fc / sfc が選ばれた時だけ <link> を注入）
watch(currentTheme, (newTheme) => {
  if (newTheme === 'fc') loadFont('pressStart2P')
  else if (newTheme === 'sfc') loadFont('dotGothic16')
}, { immediate: true })

/**
 * テーマ ID → コンポーネント解決。
 *
 * 重要:
 * - テンプレート literal による動的 import（`import(\`...${currentTheme}.vue\`)`）は禁止。
 *   ビルダの静的解析が効かず、`currentTheme.value` が想定外の値になった場合に
 *   パストラバーサルや任意モジュール読み込みの温床になる。
 * - 必ず switch/case で固定文字列の import を列挙する（Vite/Rollup の chunk 分割もこの形式が前提）。
 */
const themeComponent = computed(() => {
  switch (currentTheme.value) {
    case 'y1998':
      return defineAsyncComponent(() => import('~/components/landing/themes/LandingTheme1998.vue'))
    case 'y2000':
      return defineAsyncComponent(() => import('~/components/landing/themes/LandingTheme2000.vue'))
    case 'y2005':
      return defineAsyncComponent(() => import('~/components/landing/themes/LandingTheme2005.vue'))
    case 'y2010':
      return defineAsyncComponent(() => import('~/components/landing/themes/LandingTheme2010.vue'))
    case 'y2015':
      return defineAsyncComponent(() => import('~/components/landing/themes/LandingTheme2015.vue'))
    case 'y2020':
      return defineAsyncComponent(() => import('~/components/landing/themes/LandingTheme2020.vue'))
    case 'fc':
      return defineAsyncComponent(() => import('~/components/landing/themes/LandingThemeFc.vue'))
    case 'sfc':
      return defineAsyncComponent(() => import('~/components/landing/themes/LandingThemeSfc.vue'))
    case 'modern':
    default:
      // modern は初回バンドルに含めるため defineAsyncComponent を使わず直接 import
      return defineAsyncComponent(() => import('~/components/landing/themes/LandingThemeModern.vue'))
  }
})

function resetToModern() {
  currentTheme.value = 'modern'
}
</script>

<template>
  <div>
    <!-- SSR 用: 常にモダン版（SEO 用 + クローラ用） -->
    <ClientOnly>
      <component :is="themeComponent" />
      <template #fallback>
        <!-- ハイドレーション中はモダン版を表示してチラつきを防ぐ -->
        <LandingHero />
        <LandingStats />
        <LandingFeatures />
        <LandingSteps />
        <LandingUseCases />
        <LandingFaq />
        <LandingCta />
      </template>
    </ClientOnly>

    <!-- レトロ表示中のみ脱出ボタン -->
    <LandingEraThemeSwitcher
      v-if="currentTheme !== 'modern'"
      :current-theme="currentTheme"
      @reset="resetToModern"
    />
  </div>
</template>
```

### SEO・SSR 戦略

| 観点 | 戦略 |
|------|------|
| **SSR レンダリング** | 常に `modern` を返す。乱数抽選は `onMounted` 後（クライアント側）にのみ実行 |
| **クローラ判定** | `useEraTheme.isCrawler()` で User-Agent 検査。クローラには絶対に `modern` 以外を返さない |
| **JSON-LD / OGP** | テーマに関係なく `modern` 用のメタを SSR で常に出力（既存仕様維持） |
| **ハイドレーション** | `<ClientOnly>` でラップし、サーバー/クライアント間の不一致警告を抑制 |
| **Lighthouse** | レトロテーマは表示頻度 1% のため、Lighthouse スコアへの影響は無視できる |
| **CLS（レイアウトシフト）** | テーマ切替時の最初のチラつきを避けるため、`fallback` で必ずモダン版を表示してから差し替える |

### コナミコマンド検出（fc テーマ内のみ有効）

```typescript
// frontend/app/composables/useKonamiCommand.ts
const KONAMI: readonly string[] = Object.freeze([
  'ArrowUp', 'ArrowUp', 'ArrowDown', 'ArrowDown',
  'ArrowLeft', 'ArrowRight', 'ArrowLeft', 'ArrowRight', 'b', 'a',
])

export function useKonamiCommand(callback: () => void) {
  const buffer = ref<string[]>([])

  function handleKey(e: KeyboardEvent) {
    // IME 変換中は無視（日本語入力中の誤検知防止）
    // `e.keyCode === 229` は deprecated API のため使わず、`isComposing` のみで判定
    if (e.isComposing) return

    // 修飾キー押下中は無視（Ctrl+B などのショートカットと衝突しないように）
    if (e.ctrlKey || e.metaKey || e.altKey) return

    buffer.value = [...buffer.value, e.key.toLowerCase()].slice(-KONAMI.length)
    if (buffer.value.join(',') === KONAMI.join(',').toLowerCase()) {
      callback()
      buffer.value = []
    }
  }

  onMounted(() => window.addEventListener('keydown', handleKey))
  onUnmounted(() => window.removeEventListener('keydown', handleKey))
}
```

**動作スコープ**:
- **fc テーマ表示中のみ** `useKonamiCommand` をマウントする（Modern 含む他テーマでは購読しない）
- 入力成功時の挙動: **隠しメッセージ表示 + 1UP 効果音（任意・効果音 ON 時のみ）**
- 9番目の隠しテーマは作らない（§12 未解決事項で決着済み）

**制約・注記**:
- **モバイル非対応**: 矢印キーがないため、コナミコマンドはデスクトップ・物理キーボード環境専用機能とする
- **IME 対応**: `e.isComposing` で日本語変換中の誤検知を防止
- **修飾キー除外**: Ctrl/Cmd/Alt 押下中は無視し、ブラウザショートカットと衝突しないように

### Web Audio API 効果音（任意・デフォルト OFF）

ファミコン/SFC テーマで「ピコッ」音をブラウザ内合成で鳴らす:

```typescript
let sharedCtx: AudioContext | null = null

/**
 * ブラウザの Autoplay Policy 制約:
 * - ページロード直後は AudioContext が suspended 状態で生成される
 * - 一度でもユーザーが interact（クリック/キー押下/タップ）した後でなければ resume できない
 * - そのため、テーマ画面マウント時にいきなり鳴らすと無音になる
 *
 * 対策: 初回ユーザー操作時に AudioContext を初期化＆resume する
 */
function ensureContext(): AudioContext | null {
  if (typeof window === 'undefined') return null
  if (sharedCtx) return sharedCtx
  try {
    // webkit 互換のため `as any` で取得（WebkitAudioContext は AudioContext と完全互換ではないため
    // 型完全性より実行時互換性を優先する妥協）
    // eslint-disable-next-line @typescript-eslint/no-explicit-any
    const AudioCtx = window.AudioContext ?? (window as any).webkitAudioContext
    if (!AudioCtx) return null
    sharedCtx = new AudioCtx()
    return sharedCtx
  }
  catch {
    return null
  }
}

function playBeep(freq: number, durationMs: number): void {
  const ctx = ensureContext()
  if (!ctx) return
  if (ctx.state === 'suspended') {
    // Autoplay Policy: resume はユーザー操作のコールバック内でしか成功しないため
    // ここで失敗した場合は無音のまま諦める
    void ctx.resume().catch(() => undefined)
  }
  const osc = ctx.createOscillator()
  const gain = ctx.createGain()
  osc.type = 'square' // 矩形波 = ファミコン音色
  osc.frequency.value = freq
  gain.gain.value = 0.1 // 音量を抑えてユーザーを驚かせない
  osc.connect(gain).connect(ctx.destination)
  osc.start()
  osc.stop(ctx.currentTime + durationMs / 1000)
}
```

#### 効果音 ON/OFF と Autoplay Policy

| 状態 | 挙動 |
|---|---|
| 初期状態 | **デフォルト OFF**。ユーザーを驚かせない |
| ユーザーが ON に切替（クリック操作） | このクリック自体が「ユーザー interaction」となり、以降の `playBeep` で音が出る |
| OFF → ON 直後の最初の操作 | 環境によっては最初の1音だけ無音になる可能性。設計上許容（2音目以降は確実に鳴る） |
| 状態保存先 | **localStorage**（key: `mannschaft_era_audio_enabled`）。F12.3 GDPR との整合性は §10 リスク緩和策で議論 |
| AudioContext のライフサイクル | モジュール内シングルトンとして1つだけ生成。破棄しないため SPA ページ遷移中もメモリに残るが、ブラウザタブ終了時に自動解放される。複数生成によるメモリリークを避けるための妥協（1タブあたり数 KB で実害軽微） |

## 6. アクセシビリティ

レトロテーマは視覚的に強い表現を含むため、アクセシビリティ脱出口を必ず確保する。

### 必須対応
- **「🎨 モダン版に戻す」ボタン**: レトロテーマ表示中は常時 `position: fixed; bottom: 16px; right: 16px;` で表示
  - z-index は最前面（9999）
  - キーボード操作可能（`tabindex="0"` + Enter/Space キー対応）
  - スクリーンリーダー向け `aria-label` は **i18n キー `era_themes.switcher.aria_label`** を必ず使用（直書き禁止）
  - **各テーマ内で z-index 9999 を超える要素を作らない**（脱出口隠れ防止 — 実装時に CSS lint で `z-index > 9000` を警告）
- **`prefers-reduced-motion: reduce`** を尊重: 1998 年風の点滅・marquee アニメーションは停止
- **コントラスト**: 各テーマで本文テキストが WCAG AA 基準（4.5:1）を満たす（1998 年風は装飾優先のため、本文部分のみ満たす）
- **音**: 効果音は明示的にユーザーが ON にしない限り鳴らない
- **キーボードのみで脱出可能**: マウス操作を要する形式（hover 表示、ロングプレス等）は禁止

### 推奨対応
- 「これは Mannschaft の隠しテーマです」というツールチップを脱出ボタンに併記
- スクリーンリーダー利用検出（`window.speechSynthesis` 等は不可能なため、`prefers-reduced-motion` で代用）

### バンドルサイズ対策

8 + 1 = 9 テーマ分のコンポーネントすべてを最初からロードするとバンドルが肥大化する。

**対策: 動的 import で各テーマを Lazy ロード**

```typescript
const themeComponent = computed(() => {
  switch (currentTheme.value) {
    case 'y1998': return defineAsyncComponent(() => import('~/components/landing/themes/LandingTheme1998.vue'))
    // ... 他のテーマも同様
    default:      return resolveComponent('LandingThemesModern')
  }
})
```

- `modern` のみ初回バンドルに含める（99% のユーザー）
- レトロ 8 種は別 chunk に分割し、選ばれた1つだけを動的ロード
- これによりレトロが発動した時のみ追加の HTTP リクエスト + 数十 KB ロード

### Google Fonts の遅延読み込み（ホワイトリスト方式）

`Press Start 2P` / `DotGothic16` 等のレトロフォントは、レトロテーマが選ばれた時のみ `<link rel="stylesheet">` を動的注入する。

**重要なセキュリティ要件**:
1. **URL ホワイトリスト**: 任意の URL を読み込めないように `ERA_FONT_WHITELIST` で固定値のみ許可
2. **`referrerpolicy="no-referrer"`**: ユーザーの IP/User-Agent が Google Fonts に漏洩する経路を遮断（GDPR/F12.3 整合性）
3. **`crossorigin="anonymous"`**: クッキー送信を抑止
4. **CSP 対応**: nuxt.config の `app.head.meta` で `style-src` に `https://fonts.googleapis.com`、`font-src` に `https://fonts.gstatic.com` を追加（または nonce 方式）
5. **重複注入防止**: 既に存在する link 要素はスキップ

```typescript
// frontend/app/composables/useEraThemeFonts.ts
type FontKey = 'pressStart2P' | 'dotGothic16'

/** 許可されたフォント URL のホワイトリスト。これ以外の URL は注入できない */
const ERA_FONT_WHITELIST = Object.freeze({
  pressStart2P: 'https://fonts.googleapis.com/css2?family=Press+Start+2P&display=swap',
  dotGothic16: 'https://fonts.googleapis.com/css2?family=DotGothic16&display=swap',
} as const) satisfies Readonly<Record<FontKey, string>>

export function useEraThemeFonts() {
  function loadFont(key: FontKey): void {
    if (typeof document === 'undefined') return // SSR ガード

    const href = ERA_FONT_WHITELIST[key]
    if (!href) return // 型安全のため理論上到達しないが念のため

    // 重複注入防止
    const existing = document.querySelector(`link[data-era-font="${key}"]`)
    if (existing) return

    const link = document.createElement('link')
    link.rel = 'stylesheet'
    link.href = href
    link.crossOrigin = 'anonymous'
    link.referrerPolicy = 'no-referrer'
    link.dataset.eraFont = key
    document.head.appendChild(link)
  }

  return { loadFont }
}
```

**CSP 設定（nuxt.config.ts への追加例）**:

```typescript
// nuxt.config.ts
export default defineNuxtConfig({
  app: {
    head: {
      meta: [
        {
          'http-equiv': 'Content-Security-Policy',
          'content': [
            "default-src 'self'",
            // 外部スタイルシートを許可するだけ。'unsafe-inline' は付けない（XSS 禁忌）。
            // 動的に注入する <link rel="stylesheet"> は外部参照なのでこれで十分。
            "style-src 'self' https://fonts.googleapis.com",
            "font-src 'self' https://fonts.gstatic.com",
            // ... 既存の CSP に追加
          ].join('; '),
        },
      ],
    },
  },
})
```

**重要**: `'unsafe-inline'` は付けない。Google Fonts の動的読み込みは `<link rel="stylesheet" href="...">` という**外部参照**であり、`<style>` タグのインラインスタイルではないため、`'unsafe-inline'` は不要かつ XSS リスクの温床になる。

> 既存の CSP 設定がある場合はそれに `https://fonts.googleapis.com` / `https://fonts.gstatic.com` をマージする。
> CSP を運用していない場合は、F12.6 実装と同時に基本的な CSP の導入を検討すべき（別タスク）。
> Tailwind CSS など既存実装で `<style>` タグを生成している場合は、`'unsafe-inline'` ではなく **nonce 方式** で対応するのが本筋（本設計のスコープ外、F12.6 実装後の別タスクとして起票）。

## 10. リスクと緩和策

| リスク | 影響度 | 緩和策 |
|---|---|---|
| **SEO 低下** | 高 | クローラ User-Agent 判定（クエリより優先）+ SSR 時は常に modern + JSON-LD 固定 |
| **ハイドレーション警告** | 中 | `<ClientOnly>` ラップ + `fallback` で modern を SSR 表示 |
| **バンドルサイズ膨張** | 中 | 各レトロテーマを `defineAsyncComponent` + 動的 import で別 chunk 化 |
| **ユーザー混乱** | 中 | 脱出ボタン常時表示 + dev 環境のみコンソールログでイースターエッグ告知 |
| **アクセシビリティ違反** | 中 | `prefers-reduced-motion` 尊重 + コントラスト確保 + キーボード操作対応 + z-index 制限 |
| **ブランド毀損** | 低 | レトロテーマは「遊び心」として明示。脱出可能性で安全確保 |
| **モバイル崩れ** | 中 | 各テーマで個別にレスポンシブテスト。fc/sfc は `transform: scale()` で固定アスペクト維持 |
| **効果音の不快感** | 低 | デフォルト OFF + ユーザー任意 ON。Autoplay Policy 制約も明記 |
| **クエリパラメータ攻撃** | 中 | `isEraTheme()` 型ガード + ホワイトリスト検証 + `Object.freeze` で改変防止 |
| **動的 import パストラバーサル** | 低 | テンプレート literal 禁止、switch/case で固定文字列のみ |
| **CSP 衝突** | 中 | 動的 link 注入経路を ホワイトリストに固定。Google Fonts ホストを CSP に明記 |
| **本番 console ログによる乱数ロジック露出** | 低 | `import.meta.dev` で本番抑制（イースターエッグ希少性保持） |
| **localStorage と GDPR 整合性** | 低 | 効果音 ON/OFF のみ。個人情報を含まないため §F12.3 の対象外と明示。Cookie バナー（F12.5）の対象にもしない |
| **クローラ User-Agent 偽装** | 低 | 実害は SEO 詐称ではなくむしろ「偽装してまで隠しを見たい人」のみで、本機能のスコープ外と判断 |
| **第三者フォントによる IP 漏洩** | 中 | `referrerpolicy="no-referrer"` + `crossorigin="anonymous"` でリファラ・クッキー送信を抑止 |
| **テーマ素材の著作権** | 中 | 全素材を Anthropic 自社製作 or CC0/MIT のオープン素材に限定。実装時にチェックリスト化 |

- ~~**クエリパラメータ攻撃面**~~ — **解決**: `isEraTheme()` 型ガード関数 + 配列値対応 + `LocationQueryValue` 考慮で堅牢化（§5）
- ~~**Google Fonts プライバシー**~~ — **解決**: ホワイトリスト方式 + `referrerpolicy="no-referrer"` + `crossorigin="anonymous"` で IP/クッキー漏洩を抑止（§7）
- ~~**本番 console ログ漏洩**~~ — **解決**: `import.meta.dev` 環境分岐で本番抑制（§5）
- ~~**Web Audio Autoplay Policy**~~ — **解決**: AudioContext シングルトン + ユーザー interaction 後の resume 仕様を明記（§5）
- ~~**CSP 対応**~~ — **解決**: nuxt.config への `style-src` / `font-src` 追加例を記載（§7）。**`'unsafe-inline'` は使わない**ことを明文化
- ~~**deprecated keyCode API**~~ — **解決**: `e.keyCode === 229` を削除し `e.isComposing` のみに統一（§5）
- ~~**型定義の曖昧さ**~~ — **解決**: `as const satisfies readonly EraTheme[]` パターンに統一（`Object.freeze` は使わない）。`isEraTheme()` は2段階チェックで堅牢化（§5）
- ~~**useEraThemeFonts の呼び出し場所**~~ — **解決**: `pages/index.vue` の `watch(currentTheme, ...)` で fc/sfc 時にのみ `loadFont()` を呼ぶ方針を明記（§5）
