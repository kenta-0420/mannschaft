# F01.8 チーム招待QRコードPDF印刷 設計書

## ステータス: 完了

---

## 1. 概要・目的

チームのADMINまたはDEPUTY_ADMINが、**チームへの招待用QRコードをMannschaftブランドのPDFとして印刷・配布できる機能**を提供する。

### 背景・動機

- 名刺・チラシ・掲示板・回覧板など**オフラインでのチーム参加促進**に対応する
- デジタルが苦手なメンバーへの配慮として、紙のQRコードを渡すことで参加障壁を低減する
- "Mannschaft始めました。〇〇（チーム名）" というブランドメッセージとQRコードを一枚のPDFで配布できる

### 関連機能

| 機能 | 設計書 | 状態 |
|------|--------|------|
| 招待トークン管理 | F01.2 §5.4 | 実装済み |
| PDF生成共通基盤 | F12.1 | 実装済み |
| QR会員証 | F02.1 | 実装済み |

---

## 2. ユースケース・対象ユーザー

### 対象ユーザー

- チームのADMIN（チーム管理者）
- チームのDEPUTY_ADMIN（副管理者）

### ユースケース

1. チームを作成したADMINが、新メンバー勧誘用のチラシを作りたい
2. 既存の招待トークンのQRコードをPDFに印刷して掲示板に貼りたい
3. 有効期限・利用回数付きのイベント限定招待QRを印刷配布したい

### 対象外（スコープ外）

- 組織（Organization）への招待QRコードPDF → 同一パターンで将来拡張可能（§10参照）
- QRコード中央へのMannschaftロゴ埋め込み → Phase 2 候補（§10参照）

## 3. PDF仕様

### 3.1 レイアウト（A4縦向き）

```
┌────────────────────────────────────────┐
│                                        │
│             Mannschaft                 │  ← ブランド名（大、プライマリカラー）
│                                        │
│  ╔══════════════════════════════════╗  │
│  ║                                  ║  │
│  ║     Mannschaft始めました。        ║  │  ← メインメッセージ（太字）
│  ║                                  ║  │
│  ║     〇〇スポーツクラブ           ║  │  ← チーム名（大、強調）
│  ║                                  ║  │
│  ╚══════════════════════════════════╝  │
│                                        │
│           ┌──────────────┐            │
│           │              │            │
│           │   QR CODE    │            │  ← 招待URL QRコード（350×350px）
│           │              │            │
│           └──────────────┘            │
│                                        │
│    このQRコードをスキャンして          │  ← 参加手順説明（小）

│                                        │
│  ─────────────────────────────────    │
│  ※有効期限: 2026-04-28 23:59          │  ← 有効期限（設定時のみ表示）
│  ※最大利用回数: 10回                  │  ← 最大利用数（設定時のみ表示）
│  ─────────────────────────────────    │
│                                        │
│         Powered by Mannschaft          │  ← フッター（小）
└────────────────────────────────────────┘
```

### 3.2 デザイン仕様

| 要素 | 仕様 |
|------|------|
| 用紙サイズ | A4縦向き（210mm × 297mm） |
| 余白 | 上下左右 20mm |
| フォント | NotoSansJP-Regular（既存、F12.1共通基盤） |
| ブランド名フォントサイズ | 36pt、プライマリカラー（`#3B82F6`） |
| メインメッセージ | 18pt、太字 |
| チーム名 | 28pt、太字、プライマリカラー（`#3B82F6`） |
| QRコード | 350×350px、ページ中央配置 |
| 参加手順 | 12pt、グレー（`#555`） |
| 有効期限/利用回数 | 10pt、グレー（設定されている場合のみ表示） |
| フッター | 9pt、ライトグレー（`#999`） |

### 3.3 QRコード仕様

| 項目 | 仕様 |
|------|------|
| エンコード対象 | `https://{frontendBaseUrl}/invite/{token}` |
| サイズ | 350×350 px |
| 誤り訂正レベル | H（30%損傷対応）— 将来のロゴ埋め込み対応余地 |
| 文字コード | UTF-8 |
| マージン | 1 モジュール |
| 出力形式 | PNG → Base64 → Thymeleafテンプレートに埋め込み |
| 生成ライブラリ | ZXing（既存、`InviteService.generateInviteQrCode()` を拡張） |

> **既存実装との差分**: 既存の `GET /api/v1/invite/{token}/qr` エンドポイントはPNG直接返却（300px）。PDF用は350pxでBase64文字列返却のメソッドを別途追加する。

## 4. API設計

### 4.1 エンドポイント

```
GET /api/v1/teams/{teamId}/invite-tokens/{tokenId}/pdf
```

### 4.2 認証・認可

| 項目 | 内容 |
|------|------|
| 認証 | Bearer token必須 |
| 認可 | `teamId` に対してADMINまたはDEPUTY_ADMINロールを持つこと |
| IDORチェック | `tokenId` が `teamId` に属するトークンであることをサーバーで確認 |

> **IDOR対策**: `teamId` の検証なしに `tokenId` だけでアクセスされると他チームのトークンを取得可能になるため、必ず両方のチェックを行う。

### 4.3 リクエスト

| パラメータ | 種別 | 型 | 必須 | 説明 |
|-----------|------|---|------|------|
| teamId | path | Long | ✓ | チームID |
| tokenId | path | Long | ✓ | 招待トークンID |

### 4.4 レスポンス（成功）

```
HTTP 200 OK
Content-Type: application/pdf
Content-Disposition: attachment;
  filename="20260421_invite_qr_team.pdf";
  filename*=UTF-8''20260421_%E6%8B%9B%E5%BE%85QR%E3%82%B3%E3%83%BC%E3%83%89_%E3%83%81%E3%83%BC%E3%83%A0%E5%90%8D.pdf
Content-Length: {バイト数}

{PDFバイナリ}
```

**ファイル名規則**（F12.1命名規約に追加）:
```
{発行日}_招待QRコード_{チーム名}.pdf
例: 20260421_招待QRコード_〇〇スポーツ.pdf
```

### 4.5 エラーレスポンス

| エラーコード | HTTP | 条件 |
|-------------|------|------|
| `TEAM_001` | 404 | チームが存在しない |
| `INVITE_001` | 404 | トークンが存在しない、または `teamId` に属さない |
| `COMMON_002` | 403 | ADMIN/DEPUTY_ADMINでない |
| `PDF_002` | 500 | PDF生成失敗 |

> **注意**: `COMMON_002` が権限不足の 403 エラー（`AccessControlService.checkAdminOrAbove()` がスローする）。`COMMON_003` は 409 Conflict（重複操作）であり混同注意。

> **注意**: 無効化済み（revoked）・期限切れトークンでもPDF生成は許可する（印刷した紙の回収は不可能であり、そもそも無効なトークンのQRコードをスキャンしてもサーバー側で弾かれるため、PDFの存在自体はセキュリティリスクにならない）。ただし無効状態をPDF上に表示する（§3.1の有効期限・利用回数欄）。

### 4.6 レートリミット

既存の招待トークン管理APIのレートリミット範囲内で対応。PDF生成は書き込み操作ではないため個別制限は設けない。

> 将来的にPDF生成が高負荷になる場合は `InviteQrPdfController` 専用の Bucket4j フィルタを追加する。

async function downloadQrPdf(teamId: number, tokenId: number): Promise<void> {

#### `InviteQrPdfService.java`

```java
// com.mannschaft.app.role.service.InviteQrPdfService

@Service
@RequiredArgsConstructor
public class InviteQrPdfService {

    private final InviteService inviteService;
    private final TeamRepository teamRepository;
    private final AccessControlService accessControlService;
    private final PdfGeneratorService pdfGeneratorService;

    public ResponseEntity<byte[]> generateInviteQrPdf(Long teamId, Long tokenId, Long requestUserId) {
        // 認可チェック: ADMIN or DEPUTY_ADMIN（COMMON_002 で 403）
        accessControlService.checkAdminOrAbove(requestUserId, teamId, "TEAM");

        // トークン取得（IDORチェック: tokenId が teamId に属することを確認）
        InviteTokenEntity token = inviteService.findByIdAndTeamId(tokenId, teamId);

        // チーム情報取得
        TeamEntity team = teamRepository.findById(teamId)
            .orElseThrow(() -> new BusinessException(TeamErrorCode.TEAM_NOT_FOUND));

        // QRコード生成（350px, Base64）
        // InviteService 内部で frontendUrl を組み立てるため、tokenStr を渡す（URL直接渡し不可）
        String qrBase64 = inviteService.generateInviteQrCodeAsBase64(token.getToken(), 350);

        // Thymeleafテンプレート変数組み立て
        Map<String, Object> vars = new HashMap<>();
        vars.put("teamName", team.getName());
        vars.put("qrBase64", qrBase64);
        vars.put("expiresAt", token.getExpiresAt());  // null許容
        vars.put("maxUses", token.getMaxUses());       // null許容
        vars.put("usedCount", token.getUsedCount());
        vars.put("isRevoked", token.getRevokedAt() != null);
        vars.put("isExpired", token.getExpiresAt() != null
            && token.getExpiresAt().isBefore(LocalDateTime.now()));

        // PDF生成
        byte[] pdfBytes = pdfGeneratorService.generateFromTemplate("pdf/invite-qr", vars);

        // ファイル名生成
        String fileName = PdfFileNameBuilder.of("招待QRコード")
            .date(LocalDate.now())
            .identifier(team.getName())
            .build();

        return PdfResponseHelper.toResponse(pdfBytes, fileName);
    }
}
```

### 6.2 変更ファイル

#### `InviteService.java` への追加

```java
/**
 * 招待QRコードをBase64エンコードされたPNG文字列として生成する。
 * PDFテンプレートへの埋め込み用。
 *
 * <p>既存の {@code generateInviteQrCode(tokenStr, size)} と同じシグネチャ。
 * URLの組み立ては InviteService 内に閉じ込め、呼び出し元で URL を生成しないこと。</p>
 *
 * @param tokenStr トークン文字列（UUIDv4）
 * @param size     QRコードサイズ（px）
 * @return Base64エンコード済みPNG文字列（data URI プレフィックスなし）
 */
public String generateInviteQrCodeAsBase64(String tokenStr, int size) {
    byte[] qrBytes = generateInviteQrCode(tokenStr, size);  // 既存メソッド（tokenStr → URL変換も内部で実施）
    return Base64.getEncoder().encodeToString(qrBytes);
}

/**
 * 招待トークンをIDとチームIDで取得する（IDORチェック用）。
 *
 * @param tokenId 招待トークンID
 * @param teamId  チームID
 * @return 招待トークンEntity
 * @throws AppException INVITE_001（存在しないまたはチームに属さない）
 */
public InviteTokenEntity findByIdAndTeamId(Long tokenId, Long teamId) {
    return inviteTokenRepository.findByIdAndTeamId(tokenId, teamId)
        .orElseThrow(() -> new BusinessException(RoleErrorCode.ROLE_002));  // INVITE_001 相当（既存 RoleErrorCode 使用）
}
```

## 7. セキュリティ設計

### 7.1 認証・認可

| 脅威 | 対策 |
|------|------|
| 未認証アクセス | Bearer token必須（Spring Security で制御） |
| 他チームのQR PDFを取得（IDOR） | `teamId` + `tokenId` の組み合わせをサーバーで検証。`findByIdAndTeamId()` で両方を確認 |
| 一般メンバーによるPDF取得 | ADMIN/DEPUTY_ADMINのみアクセス可。`AccessControlService.checkAdminOrAbove(userId, teamId, "TEAM")` で確認（COMMON_002/403）|

### 7.2 QRコードのセキュリティ

| 脅威 | 対策 |
|------|------|
| 無効トークンによる参加 | QRコードのスキャン先（`/invite/{token}/join`）でサーバー側が有効性を判定して弾く |
| QRコードの悪用（不特定多数への配布） | 最大利用回数・有効期限を設定することで抑制。それぞれの設定はADMINの責任 |
| PDF内のトークン漏洩 | QRコードのペイロードはUUID v4。漏洩リスクは既存の招待URL共有と同等 |

### 7.3 PDFセキュリティ

| 項目 | 方針 |
|------|------|
| PDFパスワード設定 | 実装しない（用途が印刷・配布のため。保護が必要ならチーム管理者の責任） |
| 印刷禁止フラグ | 設定しない（印刷が主目的のため）|
| 透かし（Watermark） | 実装しない（Phase 1） |

## 11. 実装時の注意事項

1. **`InviteQrPdfService.assertTeamAdminRole()`**: 既存の `UserRoleRepository` と `RoleEntity` の仕組みを使ってADMIN/DEPUTY_ADMINを確認する。既存の認可チェックロジックと重複しないよう、共通ユーティリティ（`SecurityUtils` 系）を活用すること

2. **Thymeleaf `#temporals.format`**: Spring Boot 3.x では `thymeleaf-extras-java8time` は自動包含される。`LocalDateTime` フォーマットには `#temporals.format()` を使用可能

3. **QRコードサイズ**: PDF上での350px指定は72dpiでの表示サイズ。印刷品質を考慮すると300dpi相当の解像度が望ましいが、Flying SaucerはDPI変換を行うため、350pxで実用上の問題はない

4. **チーム名のサニタイズ**: チーム名にHTMLエスケープが必要な文字（`<`, `>`, `&`等）が含まれる可能性がある。Thymeleafの `th:text` 属性は自動エスケープするため、`th:utext` は使用しないこと

5. **`isRevoked`と`isExpired`の優先順位**: 両方trueの場合は`isRevoked`を優先表示する（Thymeleafの`th:if`条件で制御）

6. **長いチーム名の折り返し**: `invite-team-name` は 28pt で表示するため、全角20文字以上のチーム名では2行以上になる場合がある。Flying Saucer は自動折り返しを行うためレイアウト崩れは発生しないが、目視確認項目（§8.3）でチェックすること

7. **Flying Saucer CSS制約**: Flying Saucer は CSS2.1 のみサポート。`border-radius`（CSS3）は動作しないため使用禁止。`box-shadow`・`flexbox`・`grid` も非対応。テンプレート編集時は必ず CSS2.1 範囲内で記述すること
