# F12.1 PDF生成共通基盤 設計書

## 1. 概要

プロジェクト横断で必要な PDF 生成機能を、共通基盤として一括実装する。
各機能モジュールに散在する TODO プレースホルダーを、統一アーキテクチャで置き換える。

### 技術スタック

| ライブラリ | バージョン | 役割 |
|---|---|---|
| **Flying Saucer** (`flying-saucer-pdf-openpdf`) | 9.9.5+ | HTML/CSS → PDF 変換エンジン（`ITextRenderer`） |
| **OpenPDF** | 2.0.3+ | 低レベル PDF 操作（オーバーレイ、結合等） |
| **Thymeleaf** | Spring Boot 管理 | PDF 用 HTML テンプレートエンジン（画面描画には使わない） |
| **Apache Batik** (`batik-transcoder`) | 1.17+ | SVG → PNG 変換（電子印鑑の描画に使用） |
| **NotoSansJP** | フォントファイル同梱 | 日本語ゴシック体（本文・帳票用） |
| **NotoSerifJP** | フォントファイル同梱 | 日本語明朝体（領収書等の公式文書用、任意追加） |

> **注意**: Flying Saucer と openhtmltopdf は別プロジェクト。本設計では **Flying Saucer**（`ITextRenderer`）を採用する。openhtmltopdf（`PdfRendererBuilder`）は使用しない。

> **注意**: Thymeleaf はフロント画面（Vue.js）とは無関係。サーバーサイドで PDF レイアウト用 HTML を生成するためだけに使用する。

### 処理フロー

```
Vue.js → GET /api/v1/.../pdf → Spring Boot Controller
                                    │
                                    ├─ Service がデータ取得
                                    ├─ Thymeleaf が HTML テンプレートを描画（文字列）
                                    ├─ Flying Saucer が HTML → PDF (byte[]) に変換
                                    │   └─ OpenPDF で後処理（スタンプ等）が必要な場合のみ追加処理
                                    ├─ ファイル名を規約に従い生成
                                    └─ Content-Type: application/pdf + Content-Disposition でレスポンス
```

## 2. ファイル命名規約

### 基本フォーマット

```
{YYYYMMDD}_{文書種別}_{識別名}.pdf
```

### 文書種別ごとの命名ルール

| 文書種別 | ファイル名パターン | 例 |
|---|---|---|
| 領収書 | `{発行日}_{領収書}_{宛名様}.pdf` | `20260322_領収書_山田太郎様.pdf` |
| 領収書（無効） | `{発行日}_{領収書_無効}_{宛名様}.pdf` | `20260322_領収書_無効_山田太郎様.pdf` |
| カルテ | `{来院日}_{カルテ}_{患者名様}.pdf` | `20260315_カルテ_佐藤花子様.pdf` |
| 施設予約確認 | `{利用日}_{施設予約}_{施設名}_{予約者様}.pdf` | `20260401_施設予約_会議室A_田中一郎様.pdf` |
| 順位表 | `{大会日}_{順位表}_{大会名}_{部門名}.pdf` | `20260510_順位表_春季大会_Aリーグ.pdf` |
| トーナメント表 | `{大会日}_{トーナメント表}_{大会名}.pdf` | `20260510_トーナメント表_春季大会.pdf` |
| 個人ランキング | `{大会日}_{ランキング}_{大会名}_{統計項目}.pdf` | `20260510_ランキング_春季大会_得点.pdf` |
| 対戦マトリクス | `{大会日}_{対戦表}_{大会名}_{部門名}.pdf` | `20260510_対戦表_春季大会_Aリーグ.pdf` |
| 議事録 | `{開催日}_{議事録}_{議案名}.pdf` | `20260301_議事録_第3回定期総会.pdf` |
| チケット売上 | `{期間開始}_{売上レポート}_{チーム名}.pdf` | `20260301_売上レポート_〇〇スポーツ.pdf` |
| チケット領収書 | `{購入日}_{チケット領収書}_{購入者様}.pdf` | `20260322_チケット領収書_鈴木次郎様.pdf` |
| フォーム回答 | `{提出日}_{フォーム名}_{回答者名}.pdf` | `20260322_入会申込書_高橋三郎.pdf` |
| 招待QRコード | `{発行日}_招待QRコード_{チーム名}.pdf` | `20260421_招待QRコード_〇〇スポーツ.pdf` |

### 命名ルール詳細

- 日付は `yyyyMMdd` 形式（ハイフンなし）
- 人名には「様」を付与（領収書・カルテ等、対外文書の場合）
- 内部帳票（売上レポート等）には敬称不要
- ファイル名に使用不可文字（`/\:*?"<>|`）はアンダースコア `_` に置換
- 長すぎる名前（100文字超）は末尾を切り詰めて `.pdf` で終端（切り詰め後の末尾文字は自然な区切りで打ち切り）

### ファイル名生成ユーティリティ

```java
// com.mannschaft.app.common.pdf.PdfFileNameBuilder
PdfFileNameBuilder.of("領収書")
    .date(receipt.getIssuedDate())
    .identifier(receipt.getRecipientName() + "様")
    .build();
// → "20260322_領収書_山田太郎様.pdf"
```

## 4. 共通コンポーネント設計

### 4.1 PdfGeneratorService

中核サービス。テンプレート名とデータモデルを受け取り、PDF バイト配列を返す。

```java
@Service
public class PdfGeneratorService {

    /**
     * Thymeleaf テンプレートからPDFを生成する。
     *
     * @param templateName テンプレート名（例: "pdf/receipt"）
     * @param variables    テンプレートに渡す変数マップ
     * @return PDF の byte[]
     */
    public byte[] generateFromTemplate(String templateName, Map<String, Object> variables);
}
```

**内部処理**:
1. `SpringTemplateEngine.process()` で HTML 文字列を生成
2. Flying Saucer の `ITextRenderer` で HTML → PDF 変換
3. ベース URI をクラスパスルート（`classpath:/`）に設定し、CSS・フォント・画像の相対パスを解決
4. フォント登録は `PdfFontConfig` で起動時に一括登録

### 4.2 PdfFileNameBuilder

```java
public class PdfFileNameBuilder {

    public static PdfFileNameBuilder of(String documentType);
    public PdfFileNameBuilder date(LocalDate date);
    public PdfFileNameBuilder identifier(String identifier);
    public String build();         // → "20260322_領収書_山田太郎様.pdf"
    public String buildEncoded();  // → URL エンコード済み（Content-Disposition 用）
}
```

**バリデーション**:
- `documentType` と `date` は必須
- `identifier` はオプション（省略時は日付+文書種別のみ）
- ファイル名禁止文字の自動サニタイズ

### 4.3 PdfResponseHelper

各コントローラーでの重複を排除するヘルパー。

```java
public class PdfResponseHelper {

    /**
     * PDF バイト配列から ResponseEntity を生成する。
     * Content-Type, Content-Disposition, Content-Length を自動設定。
     *
     * @param pdfBytes  PDF データ
     * @param fileName  ファイル名（日本語可、RFC 5987 でエンコード）
     * @return ResponseEntity<byte[]>
     */
    public static ResponseEntity<byte[]> toResponse(byte[] pdfBytes, String fileName);
}
```

**Content-Disposition ヘッダー**:
```
Content-Disposition: attachment;
  filename="20260322_receipt.pdf";
  filename*=UTF-8''20260322_%E9%A0%98%E5%8F%8E%E6%9B%B8_%E5%B1%B1%E7%94%B0%E5%A4%AA%E9%83%8E%E6%A7%98.pdf
```
- `filename`: ASCII フォールバック（日本語部分は英語に）
- `filename*`: RFC 5987 UTF-8 エンコード（日本語ファイル名そのまま）

### 4.4 PdfFontConfig

`com.mannschaft.app.config` パッケージに配置する（Spring 設定クラスは `config` パッケージに集約する規約に準拠）。

```java
@Configuration
public class PdfFontConfig {
    // NotoSansJP / NotoSerifJP フォントファイルのリソースパスを定義
    // PdfGeneratorService 起動時に ITextFontResolver.addFont() で一括登録
    // NotoSerifJP が存在しない場合はスキップ（任意追加フォント）
}
```

> フォントは CSS `@font-face` ではなく Java コード側で `ITextFontResolver.addFont()` によりプログラム的に登録する。Flying Saucer の CSS パス解決はクラスパスリソースとの相性が不安定なため、CSS 内の `@font-face` は**フォールバック用の宣言のみ**とし、実際の登録は Java 側で確実に行う。

### 4.5 SvgToPngConverter

電子印鑑の SVG 文字列を PNG バイト配列に変換するユーティリティ。Flying Saucer 単体では SVG レンダリングに対応していないため、Apache Batik で事前変換する。

```java
public class SvgToPngConverter {

    /**
     * SVG 文字列を PNG バイト配列に変換する。
     *
     * @param svgContent SVG 文字列
     * @param width      出力幅（px）
     * @param height     出力高さ（px）
     * @return PNG の byte[]
     */
    public static byte[] convert(String svgContent, int width, int height);
}
```

**利用箇所**: 領収書・フォーム回答で `sealSvg` が指定された場合、テンプレートに渡す前に PNG Base64 に変換する。

## 5. 対象機能別の実装仕様

### 5.1 領収書 PDF（F08.4）

| 項目 | 内容 |
|---|---|
| テンプレート | `pdf/receipt.html`, `pdf/receipt-voided.html` |
| 既存インターフェース | `ReceiptPdfGenerator`（シグネチャ変更なし） |
| 実装クラス | `ReceiptPdfGeneratorPlaceholder` → `ReceiptPdfGeneratorImpl` に置換 |
| ファイル名 | `{発行日}_領収書_{宛名}様.pdf` |
| 特記事項 | ロゴ画像、電子印鑑（SVG → PNG 変換）、カスタムフッター対応。無効化版は OpenPDF 後処理で赤「無効」テキストをオーバーレイ描画（§7.2 CSS制限事項参照） |

**ブリッジパターン**: `ReceiptPdfGeneratorImpl` は既存インターフェースのシグネチャ（`byte[] logoBytes`, `String sealSvg`, `String customFooter`）を維持しつつ、内部で `PdfGeneratorService` に委譲する。引数を `Map<String, Object>` に変換してテンプレートに渡す。`sealSvg` は `SvgToPngConverter` で PNG Base64 に変換してからテンプレートの `<img>` タグに埋め込む。

```java
@Component
@RequiredArgsConstructor
public class ReceiptPdfGeneratorImpl implements ReceiptPdfGenerator {

    private final PdfGeneratorService pdfGeneratorService;

    @Override
    public byte[] generate(ReceiptEntity receipt, List<ReceiptLineItemEntity> lineItems,
                           byte[] logoBytes, String sealSvg, String customFooter) {
        Map<String, Object> vars = new HashMap<>();
        vars.put("receipt", receipt);
        vars.put("lineItems", lineItems);
        if (logoBytes != null) vars.put("logoBase64", Base64.getEncoder().encodeToString(logoBytes));
        if (sealSvg != null) vars.put("sealBase64", Base64.getEncoder().encodeToString(
            SvgToPngConverter.convert(sealSvg, 120, 120)));
        vars.put("customFooter", customFooter);
        return pdfGeneratorService.generateFromTemplate("pdf/receipt", vars);
    }
    // generateVoided() も同様に "pdf/receipt-voided" テンプレートに委譲
}
```

## 6. Gradle 依存関係の追加

```kotlin
// build.gradle.kts に追加
// PDF 生成: Thymeleaf（テンプレートエンジン）+ Flying Saucer（HTML→PDF）+ OpenPDF（低レベル操作）
implementation("org.springframework.boot:spring-boot-starter-thymeleaf")
implementation("org.xhtmlrenderer:flying-saucer-pdf-openpdf:9.9.5")
implementation("com.github.librepdf:openpdf:2.0.3")
// SVG → PNG 変換（電子印鑑の描画に使用）
implementation("org.apache.xmlgraphics:batik-transcoder:1.17")
implementation("org.apache.xmlgraphics:batik-codec:1.17")
```

> `flying-saucer-pdf-openpdf` は Flying Saucer の OpenPDF バックエンド版。iText 依存を避けてライセンス問題をクリアにする。
> `flying-saucer-pdf-openpdf:9.9.5` と `openpdf:2.0.3` の互換性は実装時に動作確認すること。Flying Saucer のリリースノートで OpenPDF 2.x 対応を確認済みのバージョンを使用する。

## 7. PDF テンプレート共通設計

### 7.1 HTML テンプレート構造

```html
<!DOCTYPE html>
<html xmlns:th="http://www.thymeleaf.org">
<head>
    <meta charset="UTF-8"/>
    <!-- Flying Saucer はクラスパスからCSSを解決するため、ベースURI相対パスを使用 -->
    <link rel="stylesheet" href="/css/pdf-common.css"/>
</head>
<body>
    <!-- ヘッダー: ロゴ + 文書タイトル -->
    <div class="pdf-header">
        <img th:if="${logoBase64}" th:src="'data:image/png;base64,' + ${logoBase64}" class="logo"/>
        <h1 th:text="${title}">文書タイトル</h1>
    </div>

    <!-- 本文: 各テンプレートで個別定義 -->
    <div class="pdf-body">
        <!-- template-specific content -->
    </div>

    <!-- フッター: ページ番号 + カスタムテキスト -->
    <div class="pdf-footer">
        <span th:text="${customFooter}"></span>
    </div>
</body>
</html>
```

### 7.2 共通 CSS（pdf-common.css）

```css
/*
 * フォント登録は Java 側 PdfFontConfig で ITextFontResolver.addFont() により行う。
 * 以下の @font-face はフォールバック宣言（CSS 内での font-family 名定義用）。
 */
@font-face {
    font-family: 'NotoSansJP';
    src: url('/fonts/NotoSansJP-Regular.ttf');
    -fs-pdf-font-embed: embed;
    -fs-pdf-font-encoding: Identity-H;
}

@font-face {
    font-family: 'NotoSerifJP';
    src: url('/fonts/NotoSerifJP-Regular.ttf');
    -fs-pdf-font-embed: embed;
    -fs-pdf-font-encoding: Identity-H;
}

body {
    font-family: 'NotoSansJP', sans-serif;
    font-size: 10pt;
    color: #333;
    margin: 0;
    padding: 0; /* 余白は @page margin で制御。二重余白を防止 */
}

/* 明朝体が必要な文書（領収書等）向け */
.font-serif {
    font-family: 'NotoSerifJP', 'NotoSansJP', serif;
}

/* A4 サイズ */
@page {
    size: A4;
    margin: 20mm 15mm;
    @bottom-center {
        content: "ページ " counter(page) " / " counter(pages);
        font-size: 8pt;
        color: #999;
    }
}

.pdf-header { ... }
.pdf-body { ... }
.pdf-footer {
    /* カスタムフッターテキスト用。ページ番号は @page @bottom-center で自動出力 */
    text-align: center;
    font-size: 8pt;
    margin-top: 20px;
}

/* テーブル共通 */
table {
    width: 100%;
    border-collapse: collapse;
}
th, td {
    border: 1px solid #ccc;
    padding: 4px 8px;
    text-align: left;
}

/*
 * 無効スタンプ（領収書用）
 *
 * Flying Saucer は CSS3 の transform / rgba() を未サポートのため、
 * CSS だけでは回転+半透明のスタンプは実現できない。
 * 以下の方式で対応する:
 *   方式A（推奨）: OpenPDF の後処理で赤い「無効」テキストを直接 PDF に描画
 *   方式B: 事前に「無効」スタンプ画像（PNG, 回転+半透明済み）を用意し <img> で配置
 *
 * 方式A の場合、PdfGeneratorService.generateFromTemplate() の戻り値に対して
 * OpenPDF の PdfContentByte でオーバーレイテキストを追加する後処理を行う。
 */
.voided-stamp {
    /* 方式B（画像配置）の場合のスタイル */
    position: absolute;
    top: 30%;
    left: 20%;
    width: 60%;
    opacity: 0.3; /* Flying Saucer は opacity をサポート */
}
```

> **Flying Saucer CSS 制限事項**: `transform`（rotate, translate 等）、`rgba()` 色指定、`flexbox`、`grid` は未サポート。レイアウトは `position: absolute/relative`、`float`、`table` で構成すること。`opacity` プロパティは使用可能。

## 8. 日本語フォント戦略

| 項目 | 方針 |
|---|---|
| ゴシック体 | Google Noto Sans JP（OFL ライセンス、商用利用可） |
| 明朝体 | Google Noto Serif JP（OFL ライセンス、商用利用可、任意追加） |
| ファイル | `NotoSansJP-Regular.ttf`（約 5MB）、`NotoSerifJP-Regular.ttf`（約 5MB、任意） |
| 配置場所 | `src/main/resources/fonts/` |
| 登録方法 | Java 側: `PdfFontConfig` で `ITextFontResolver.addFont()` により起動時に登録。CSS 側: `@font-face` に `-fs-pdf-font-embed: embed` + `-fs-pdf-font-encoding: Identity-H` を宣言 |
| 太字・斜体 | Regular のみ同梱。太字は CSS `font-weight: bold` で疑似対応 |
| 使い分け | 本文・帳票はゴシック体（NotoSansJP）、領収書等の公式文書は明朝体（NotoSerifJP）。CSSクラス `.font-serif` で切替 |

> NotoSansJP-Bold.ttf / NotoSerifJP-Bold.ttf も必要な場合は追加可能だが、PDF サイズが 10MB 近くなるため初期は Regular のみ。
> NotoSerifJP が不要であれば NotoSansJP のみで全帳票ゴシック統一も可。`PdfFontConfig` は存在しないフォントファイルをスキップする。

## 11. パフォーマンス考慮

| 項目 | 方針 |
|---|---|
| テンプレートキャッシュ | Thymeleaf のデフォルトキャッシュを有効化（本番） |
| フォント読み込み | 起動時に 1 回のみ読み込み、以後は再利用 |
| 画像埋め込み | S3 の画像は byte[] 取得 → Base64 変換。1画像あたり上限 1MB、1文書あたり合計 5MB |
| 非同期生成 | 初期は同期生成。レスポンス遅延が問題になった場合は非同期+ポーリングに移行 |
| PDF サイズ上限 | 1 ファイルあたり 10MB を上限とし、超過時はエラー返却 |

## 12. テスト方針

|---|---|
| 単体テスト | `PdfFileNameBuilder` の命名ロジック、禁止文字サニタイズ、`SvgToPngConverter` の変換結果が有効な PNG であること |

| 目視確認 | 日本語フォントが正しく表示されること（フォントレンダリングは自動テスト困難のため手動） |
