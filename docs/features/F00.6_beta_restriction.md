# F00.6 ベータ登録制限

## 概要・目的

ベータテスト期間中、新規ユーザーの自由登録を制限し、招待リンクを持つユーザーのみ登録を許可する機能。
SYSTEM_ADMIN が制限の ON/OFF と招待可能な対象チーム・組織の ID 範囲を管理できる。

---

## DDL

```sql
CREATE TABLE beta_restriction_config (
  id          BIGINT AUTO_INCREMENT PRIMARY KEY,
  is_enabled  BOOLEAN      NOT NULL DEFAULT FALSE,
  max_team_id BIGINT       NULL COMMENT 'このID以下のチームが招待可能（NULL=制限なし）',
  max_org_id  BIGINT       NULL COMMENT 'このID以下の組織が招待可能（NULL=制限なし）',
  updated_by  BIGINT       NULL,
  updated_at  DATETIME(6)  NOT NULL DEFAULT CURRENT_TIMESTAMP(6)
);

INSERT INTO beta_restriction_config (is_enabled, updated_at)
VALUES (FALSE, NOW(6));
```

設計思想:
- 常に1レコードのみ保持するシングルトン設定テーブル
- `max_team_id` / `max_org_id` が null の場合はその軸の制限なし

---

## API 仕様

### GET /api/v1/system-admin/beta-restriction

設定を取得する。SYSTEM_ADMIN のみアクセス可。

**レスポンス:**
```json
{
  "data": {
    "isEnabled": false,
    "maxTeamId": null,
    "maxOrgId": null,
    "updatedAt": "2026-05-05T10:00:00"
  }
}
```

### PUT /api/v1/system-admin/beta-restriction

設定を更新する。SYSTEM_ADMIN のみアクセス可。

**リクエスト:**
```json
{
  "isEnabled": true,
  "maxTeamId": 10,
  "maxOrgId": 5
}
```

**レスポンス:** GET と同様の形式で更新後の設定を返す。

## 登録フロー図

```
[招待者] → 既存 /invite/{token} リンクを送付
                ↓
[未登録ユーザー] /invite/[token].vue を開く
  ├─ ログイン済み → 「参加する」ボタン
  └─ 未ログイン  → 「ログインして参加」+ 「アカウントをお持ちでない方」ボタン
                              ↓
              /register?invite={token} に遷移
                              ↓
         POST /api/v1/auth/register + { ..., inviteToken }
                              ↓
         AuthService.register()
           ① ベータ制限OFF → 従来通り（inviteToken は無視してもよい）
           ② ベータ制限ON + tokenなし → AUTH_042エラー（403）
           ③ ベータ制限ON + token有り
              → BetaRestrictionService.isBetaTokenValid()
                - トークン存在チェック
                - InviteTokenEntity.isValid() チェック
                - teamId <= maxTeamId OR orgId <= maxOrgId チェック
              → 無効 → AUTH_043エラー（403）
              → 有効 → 登録完了 → inviteService.joinByInvite() で自動参加
```

---

## エラーコード一覧

| コード | 内容 | HTTP ステータス |
|---|---|---|
| AUTH_042 | ベータ期間中は招待コードが必要です | 403 Forbidden |
| AUTH_043 | 招待コードが無効またはベータ対象外です | 403 Forbidden |

---

## 管理画面の説明

URL: `/system-admin/beta-restriction`

SYSTEM_ADMIN のみアクセス可能。

- ベータ制限の ON/OFF をトグルスイッチで切り替え
- 招待可能チームID上限・組織ID上限を数値入力（空欄=制限なし）
- 保存ボタンで設定を反映
- 現在の設定を画面下部に表示

---

## テスト観点

1. **ベータ制限OFF時**: inviteToken なしで通常登録できること
2. **ベータ制限OFF時**: inviteToken ありでも通常登録できること
3. **ベータ制限ON + tokenなし**: AUTH_042 エラーが返ること
4. **ベータ制限ON + 無効token**: AUTH_043 エラーが返ること
5. **ベータ制限ON + 有効token (チームID範囲内)**: 登録成功 + 自動参加できること
6. **ベータ制限ON + 有効token (チームID範囲外)**: AUTH_043 エラーが返ること
7. **maxTeamId/maxOrgId が null**: ID 制限なしで有効 token なら通過すること
8. **招待ページ**: 未ログイン時に「アカウントをお持ちでない方」ボタンが表示されること
9. **登録ページ**: invite クエリパラメータがある場合、POST body に inviteToken が含まれること
10. **管理画面**: SYSTEM_ADMIN でない場合に 403 が返ること
