> **ステータス**: 🟢 設計完了

## 1. 概要

「ファミリーチーム」テンプレートと、家族の日常コミュニケーションを支援する5つのライフユーティリティ機能を提供する。ユーティリティ機能はファミリーテンプレート専用ではなく、**全チームのデフォルト機能**として利用可能（ファミリーテンプレートではウィジェット・初期設定が最適化される）。

### ファミリーテンプレート
既存のテンプレートシステム（F01.3）に `FAMILY` テンプレートを追加。チーム作成時に選択すると、家族向けの壁紙・ロール呼称・ウィジェット配置が初期設定される。

### ライフユーティリティ機能（全チーム対象のデフォルト機能）

| # | 機能名 | 概要 | ファミリーでの使い方 | 一般チームでの使い方 |
|---|--------|------|-------------------|-------------------|
| 1 | **帰ったよ通知** | ワンタップで「帰宅しました」をチーム全員に通知 | 子どもの帰宅確認 | 出張・外出先からの帰社連絡 |
| 2 | **お出かけ連絡** | 行き先 + 帰宅予定時刻を共有。予定時刻超過でリマインド | 家族の行動把握 | メンバーの外出・直帰連絡 |
| 3 | **ロール呼称カスタマイズ** | チームごとにロール表示名を変更（例: ADMIN→ボス） | ADMIN→ボス | ADMIN→監督、先生、院長 等 |
| 4 | **困ったらコイントス** | ランダム2択 + カスタム選択肢。結果をチャットに共有 | 「今日の晩ご飯どうする？」 | 「どっちの案で行く？」 |
| 5 | **お買い物リスト** | 共有チェックリスト。メンバー割り当て + 購入済みチェック | 日用品・食材の買い出し | 備品・消耗品の購買管理 |

### 追加ユーティリティ機能

| # | 機能名 | 概要 | ファミリーでの使い方 | 一般チームでの使い方 |
|---|--------|------|-------------------|-------------------|
| 6 | **当番ローテーション** | メンバー×曜日/週の自動ローテーション。ダッシュボードに「今日の当番」表示 | ゴミ出し・料理・掃除当番 | 日直・受付・清掃当番 |
| 7 | **記念日リマインダー** | 登録した記念日の前日+当日にプッシュ通知 | 誕生日・結婚記念日 | メンバー誕生日・創立記念日 |

### テーマ壁紙拡張
既存のテーマ・外観カスタマイズ（デフォルト機能 #25）を拡張し、テンプレート別の壁紙プリセットを追加。ファミリーテンプレートには家族向け壁紙（季節の家族イラスト等）を初期提供。他テンプレートにも業種別壁紙を追加可能。

### テーブル定義

#### `presence_events`

帰ったよ通知・お出かけ連絡のイベント履歴。直近の状態表示 + 履歴参照に使用。

| カラム名 | 型 | NULL | デフォルト | 説明 |
|---------|---|------|-----------|------|
| `id` | BIGINT UNSIGNED | NO | AUTO_INCREMENT | PK |
| `team_id` | BIGINT UNSIGNED | NO | — | FK → teams |
| `user_id` | BIGINT UNSIGNED | NO | — | FK → users |
| `event_type` | VARCHAR(20) | NO | — | `HOME`（帰ったよ）/ `GOING_OUT`（お出かけ） |
| `message` | VARCHAR(100) | YES | NULL | ひとことメッセージ（例: 「お腹すいた〜」「何かいる？」） |
| `destination` | VARCHAR(200) | YES | NULL | お出かけ先（`GOING_OUT` 時のみ） |
| `expected_return_at` | DATETIME | YES | NULL | 帰宅予定時刻（`GOING_OUT` 時のみ） |
| `returned_at` | DATETIME | YES | NULL | 実際の帰宅時刻（`HOME` イベント発生時に直前の `GOING_OUT` を更新） |
| `overdue_level` | TINYINT | NO | 0 | 帰宅遅延通知の段階（0=未通知, 1=15分超過でADMINに通知済, 2=1時間超過でチーム全員に通知済。`GOING_OUT` のみ使用） |
| `created_at` | DATETIME | NO | CURRENT_TIMESTAMP | |

**インデックス**
```sql
INDEX idx_pe_team_user (team_id, user_id, created_at DESC)  -- チーム×ユーザー別の最新状態
INDEX idx_pe_team_latest (team_id, event_type, created_at DESC)  -- チームの最新イベント一覧
INDEX idx_pe_overdue (event_type, returned_at, overdue_level, expected_return_at)  -- 帰宅遅延バッチ用（event_type=GOING_OUT, returned_at IS NULL, overdue_level IN(0,1) で検索）
```

**制約・備考**
- `HOME` イベント作成時: 同一ユーザーの直前の `GOING_OUT`（`returned_at IS NULL`）があれば `returned_at = NOW()` を自動更新
- `GOING_OUT` イベント作成時: 同一ユーザーの直前の `GOING_OUT`（`returned_at IS NULL`）があれば `returned_at = NOW()` を自動クローズしてから新規 INSERT（行き先変更として扱う）
- 連続 `HOME` 送信: 毎回新規 INSERT する（イベントログとして蓄積）。「既に HOME 状態」でもブロックしない。通知は毎回送信される（ファミリーチームでは「ただいま！」を何度言ってもOKという UX）
- `expected_return_at` 超過 + `returned_at IS NULL` のレコードを15分ごとのバッチでチェックし、ADMIN に「{user_name} さんの帰宅予定時刻を過ぎています」通知を送信
- 履歴保持: 90日以上前のレコードは日次バッチで物理削除（長期保存の必要なし）

#### `team_role_aliases`

チームごとにロールの表示名をカスタマイズ。DB の `roles` テーブルは変更せず、表示層のみオーバーライド。

| カラム名 | 型 | NULL | デフォルト | 説明 |
|---------|---|------|-----------|------|
| `id` | BIGINT UNSIGNED | NO | AUTO_INCREMENT | PK |
| `team_id` | BIGINT UNSIGNED | NO | — | FK → teams |
| `role_name` | VARCHAR(30) | NO | — | 対象ロール（`ADMIN` / `DEPUTY_ADMIN` / `MEMBER` / `SUPPORTER`） |
| `display_alias` | VARCHAR(50) | NO | — | 表示名（例: 「ボス」「監督」「先生」） |
| `updated_by` | BIGINT UNSIGNED | NO | — | FK → users |
| `created_at` | DATETIME | NO | CURRENT_TIMESTAMP | |
| `updated_at` | DATETIME | NO | CURRENT_TIMESTAMP ON UPDATE | |

**インデックス**
```sql
UNIQUE KEY uq_tra_team_role (team_id, role_name)  -- チーム×ロールで1エイリアスのみ
```

**制約・備考**
- `SYSTEM_ADMIN` / `GUEST` は呼称変更不可（システムロールのため）
- エイリアスが未設定の場合はデフォルト表示名（`roles.display_name`）をフォールバック
- ファミリーテンプレートの初期値: `ADMIN` →「ボス」、`MEMBER` →「ファミリー」
- API レスポンスの `role` フィールドは常にシステムロール名（`ADMIN` 等）を返し、`role_display` フィールドにエイリアス or デフォルト名を返す

#### `coin_toss_results`

コイントス（ランダム決定）の結果履歴。チャット共有時の参照用。

| カラム名 | 型 | NULL | デフォルト | 説明 |
|---------|---|------|-----------|------|
| `id` | BIGINT UNSIGNED | NO | AUTO_INCREMENT | PK |
| `team_id` | BIGINT UNSIGNED | NO | — | FK → teams |
| `user_id` | BIGINT UNSIGNED | NO | — | FK → users（実行者） |
| `mode` | VARCHAR(20) | NO | 'COIN' | `COIN`（表/裏）/ `CUSTOM`（カスタム選択肢） |
| `options` | JSON | NO | — | 選択肢（例: `["表","裏"]` or `["カレー","パスタ","寿司"]`） |
| `result_index` | TINYINT | NO | — | 当選した選択肢のインデックス（0-based） |
| `question` | VARCHAR(200) | YES | NULL | 質問文（例: 「今日の晩ご飯は？」） |
| `shared_to_chat` | BOOLEAN | NO | FALSE | チャットに結果を共有したか |
| `created_at` | DATETIME | NO | CURRENT_TIMESTAMP | |

**インデックス**
```sql
INDEX idx_ctr_team (team_id, created_at DESC)  -- チーム別の履歴
```

**制約・備考**
- `COIN` モード: `options` はサーバーが `["表","裏"]` に強制上書き（クライアント送信値は無視）。`result_index` は 0 or 1
- `CUSTOM` モード: `options` は 2〜6 個の選択肢（1個以下 or 7個以上は 400。各選択肢は最大50文字）
- チャット共有: `shared_to_chat = true` の場合、F04.2 のチャットに「🪙 {user_name} がコイントスしました！ 質問: {question} → 結果: {result}」を自動投稿
- 履歴保持: 30日以上前のレコードは日次バッチで物理削除

#### `duty_rotations`

当番ローテーション定義。曜日/週単位でメンバーを自動ローテーション。

| カラム名 | 型 | NULL | デフォルト | 説明 |
|---------|---|------|-----------|------|
| `id` | BIGINT UNSIGNED | NO | AUTO_INCREMENT | PK |
| `team_id` | BIGINT UNSIGNED | NO | — | FK → teams |
| `duty_name` | VARCHAR(100) | NO | — | 当番名（例: 「ゴミ出し」「料理当番」「日直」） |
| `rotation_type` | VARCHAR(20) | NO | 'DAILY' | `DAILY`（日替わり）/ `WEEKLY`（週替わり） |
| `member_order` | JSON | NO | — | ローテーション順のメンバー ID 配列（例: `[5, 6, 7]`） |
| `start_date` | DATE | NO | — | ローテーション開始日（この日から member_order[0] がスタート） |
| `icon` | VARCHAR(10) | YES | NULL | 表示アイコン（絵文字。例: 「🗑️」「🍳」） |
| `is_enabled` | BOOLEAN | NO | TRUE | 有効/無効 |
| `created_by` | BIGINT UNSIGNED | NO | — | FK → users |
| `deleted_at` | DATETIME | YES | NULL | 論理削除 |
| `created_at` | DATETIME | NO | CURRENT_TIMESTAMP | |
| `updated_at` | DATETIME | NO | CURRENT_TIMESTAMP ON UPDATE | |

**インデックス**
```sql
INDEX idx_dr_team (team_id, deleted_at, is_enabled)  -- チーム別の有効な当番一覧
```

**制約・備考**
- 1チームあたり最大10個の当番ローテーション
- **今日の担当者の算出**: `(DATEDIFF(TODAY, start_date) / {rotation_days}) % member_count` で member_order のインデックスを計算。`DAILY` は `rotation_days = 1`、`WEEKLY` は `rotation_days = 7`
- `member_order` 内のメンバーがチームから脱退した場合: `ApplicationEvent` で該当メンバー ID を配列から除去。配列が空になったら `is_enabled = false` に自動変更
- ダッシュボードウィジェット「今日の当番」: 有効な全ローテーションの今日の担当者を一覧表示

| `repeat_annually` | BOOLEAN | NO | TRUE | 毎年繰り返すか |

**レスポンス（201 Created）**
```json

    "id": 2,
```

- 同一ユーザーの直前の `GOING_OUT`（`returned_at IS NULL`）があれば自動クローズしてから新規 INSERT

**エラーレスポンス**
| ステータス | 条件 |
|-----------|------|
| 400 | destination が空 / 200文字超過 / expected_return_at が過去 |
| 403 | SUPPORTER / GUEST |

## 5. ビジネスロジック

### 主要フロー

#### ファミリーテンプレート適用フロー
```
1. チーム作成時に template = 'FAMILY' を選択
2. F01.3 の既存テンプレート適用フロー + 以下の初期設定:
   a. team_role_aliases に初期エイリアスを INSERT:
      - ADMIN → 「ボス」
      - MEMBER → 「ファミリー」
   b. shopping_lists に「お買い物リスト」を1つ自動作成
   c. チームの壁紙を FAMILY テンプレートのデフォルト壁紙に設定
   d. team_presence_icons に初期アイコンを INSERT: HOME → 🏡、GOING_OUT → 🚗
   e. ダッシュボードウィジェットに「みんなの居場所」「今日の当番」「直近の記念日」を追加
3. 上記はテンプレート適用時の初期値であり、後からすべて変更・削除可能
```

#### 帰宅遅延リマインドバッチ（3段階エスカレーション）
```
1. Spring @Scheduled バッチが15分ごとに実行
2. presence_events から以下を検索:
   - event_type = 'GOING_OUT'
   - expected_return_at IS NOT NULL
   - returned_at IS NULL
3. 段階別の通知:
   a. 【15分超過】expected_return_at < NOW() - INTERVAL 15 MINUTE AND overdue_level = 0:
      → ADMIN にプッシュ通知「⚠️ {display_name} さんの帰宅予定時刻を15分過ぎています」
      → overdue_level = 1 に更新
   b. 【1時間超過】expected_return_at < NOW() - INTERVAL 1 HOUR AND overdue_level = 1:
      → ADMIN + チーム全員（PRESENCE_OVERDUE 通知 ON のメンバー）にプッシュ通知
        「🚨 {display_name} さんの帰宅予定時刻を1時間以上過ぎています」
      → overdue_level = 2 に更新
   c. overdue_level = 2 以降: 追加通知なし（連絡が取れない場合は電話等の別手段を想定）
4. HOME イベント作成時に overdue_level = 0 にリセット（GOING_OUT の returned_at 更新と同時）
```

#### 記念日通知バッチ
```
1. Spring @Scheduled バッチが日次で実行（09:00 JST）
2. team_anniversaries から以下を検索:
   a. deleted_at IS NULL
   b. repeat_annually = true: 月日が当日 or notify_days_before 日前に該当
   c. repeat_annually = false: 年月日が当日 or notify_days_before 日前に該当
3. 該当する記念日ごとに、チーム全員にプッシュ通知:
   - 事前: 「🎉 {name} まであと{n}日！」
   - 当日: 「🎂 今日は {name} です！」
4. 通知済み管理: 日付 + anniversary_id の組み合わせで重複防止（同日の再実行で二重通知しない）
```

#### 当番ローテーション算出ロジック
```
1. GET /duties/today または GET /duties のレスポンス生成時にリアルタイム算出:
   a. rotation_type = DAILY:
      today_index = DATEDIFF(TODAY, start_date) % member_count
      → member_order[today_index] が今日の担当
   b. rotation_type = WEEKLY:
      week_index = (DATEDIFF(TODAY, start_date) / 7) % member_count
      → member_order[week_index] が今週の担当
2. DB に「今日の担当者」を保存しない（算出コストは O(1) で十分軽量）
3. ダッシュボードウィジェット「今日の当番」: 全有効ローテーションの今日/今週の担当者を一覧表示
```

#### プレゼンス統計
```
1. GET /teams/{teamId}/presence/stats（ADMIN のみ）
2. クエリパラメータ: period（7d / 30d / 90d）
3. presence_events から集計:
   a. メンバー別: 平均帰宅時刻、外出回数、遅延回数（overdue_level > 0）
   b. チーム全体: 外出率の高い曜日、HOME 最多時間帯
4. 新テーブル不要（presence_events からの集計クエリのみ）
```

#### フロントエンド演出のヒント
```
- コイントス CUSTOM モード（3〜6選択肢）: ルーレット風アニメーションを推奨
  - バックエンドの result_index は即座に返却済み。演出はクライアント側の遅延表示
  - ルーレット回転 → 減速 → 結果にピタッと止まる演出（2〜3秒）
- コイントス COIN モード: コイン回転アニメーション（表/裏が切り替わる）
- プレゼンスウィジェット: メンバーのアイコンが HOME なら家の中、GOING_OUT なら外に配置するビジュアル表現
```

#### プレゼンスイベント・履歴クリーンアップ
```
1. Spring @Scheduled バッチが日次で実行（04:00 JST）
2. DELETE FROM presence_events WHERE created_at < NOW() - INTERVAL 90 DAY
3. DELETE FROM coin_toss_results WHERE created_at < NOW() - INTERVAL 30 DAY
4. 削除件数をアプリケーションログに記録
```

### 重要な判定ロジック

- **プレゼンスの自動リセット**: 24時間以上イベントがないメンバーは `GET /presence/status` で `UNKNOWN` として返す（DB は更新しない。クエリ時に `created_at < NOW() - INTERVAL 24 HOUR` を判定）
- **プレゼンスイベントの自動連携**: `HOME` イベント作成時 → 直前の未完了 `GOING_OUT` を自動クローズ。`GOING_OUT` イベント作成時 → 直前の未完了 `GOING_OUT` も自動クローズ（行き先変更として扱う）。常に「最新の1件」が有効な状態を保証
- **ロール呼称の表示フォールバック + Valkey キャッシュ**: `team_role_aliases` にエイリアスがないロールは `roles.display_name`（デフォルト）を返す。全 API レスポンスで `role`（システム名）と `role_display`（表示名）を分離。実装は共通レスポンスモデル（`UserSummaryDto`）の構築ロジックに `TeamRoleAliasService.getDisplayName(teamId, roleName)` を1箇所追加。各 API ドキュメントへの個別修正は不要（`BACKEND_CODING_CONVENTION.md` に共通モデル仕様変更として記載）。**Valkey キャッシュ**（`mannschaft:role_alias:{teamId}`、値: `{"ADMIN":"ボス","MEMBER":"ファミリー"}`、TTL なし）で N+1 クエリを解消。`PUT /role-aliases` 実行時に即時キャッシュ更新。ロール数は最大4のためキャッシュサイズは極小
- **コイントスの公平性**: サーバーサイドの `SecureRandom` で結果を生成。クライアント側の乱数は使用しない
- **お買い物リストのリアルタイム同期**: WebSocket（`/topic/shopping/{teamId}/{listId}`）でアイテムの追加・チェック・削除を全メンバーに即座に配信。オフラインメンバーは次回 `GET /items` で最新状態を取得
- **テンプレート壁紙の選択範囲**: チームは自分のテンプレートの壁紙 + `template_slug = '*'`（共通）の壁紙を選択可能。他テンプレートの壁紙は選択不可
- **プレゼンス通知のテンプレート別デフォルト**: `notification_preferences`（F04.3）に `PRESENCE_HOME` / `PRESENCE_GOING_OUT` の ON/OFF を追加。ファミリーテンプレートはデフォルト ON、他テンプレートはデフォルト OFF。メンバーが個別に ON/OFF 変更可能
- **複数チーム一括送信**: `POST /users/me/presence/home` でプレゼンス通知有効な全所属チームに一括 HOME 送信。ファミリーチーム + 職場チームの両方に「帰りました」をワンタップで伝えられる。チーム指定 API（`POST /teams/{teamId}/presence/home`）も残す（特定チームのみに送りたい場合用）
- **帰宅遅延の3段階エスカレーション**: (1) 15分超過 → ADMIN にリマインド。(2) 1時間超過 → ADMIN + チーム全員に警告。(3) 以降は追加通知なし（電話等の別手段を想定）。`overdue_level`（0/1/2）で段階管理。HOME 時にリセット
- **プレゼンスのひとことメッセージ**: `presence_events.message`（最大100文字）で HOME / GOING_OUT にひとことを添付。通知テキストに `💬 {message}` を追記。未指定の場合はメッセージなし（ワンタップ送信のシンプルさを維持）
- **お買い物リストのテンプレート**: `is_template = true` のリストは「定番セット」として保存。`POST /copy-from-template` でテンプレートのアイテムをアクティブリストに一括コピー（チェック状態はリセット）。毎週の買い出しを効率化
- **お買い物リストのアーカイブ**: `status = ARCHIVED` で完了済みリストを履歴として保持。一覧 API はデフォルトで `ACTIVE` のみ返却（`status=ARCHIVED` フィルタで過去分を参照可能）
- **チェック済み一括リセット**: `PATCH /items/uncheck-all` で全アイテムの `is_checked = false` / `checked_by = NULL` / `checked_at = NULL` に一括リセット。繰り返し利用リストの再利用に
- **当番ローテーション**: `duty_rotations` で DAILY / WEEKLY のローテーションを定義。今日の担当者は `DATEDIFF` ベースでリアルタイム算出（DB に保存しない）。メンバー脱退時は `member_order` から自動除去
- **記念日リマインダー**: `team_anniversaries` で誕生日・記念日を登録。日次バッチ（09:00）で事前通知 + 当日通知。`repeat_annually = true` で毎年繰り返し
- **カスタムプレゼンスアイコン**: `team_presence_icons` で HOME / GOING_OUT の絵文字をチーム単位でカスタマイズ。未設定時はデフォルト（🏠 / 🚶）。ファミリーテンプレート初期値: 🏡 / 🚗
- **プレゼンス統計**: `GET /presence/stats` で ADMIN がメンバーの平均帰宅時刻・外出回数・遅延回数等を確認。新テーブル不要（`presence_events` からの集計）
- **テンプレート変更時の既存データ不変**: FAMILY → SPORTS にテンプレート変更しても、ロール呼称・お買い物リスト・壁紙・プレゼンス履歴は一切変更しない。テンプレートはあくまで「初期値のプリセット」。F01.3 の既存方針（「テンプレート変更は既存チームに遡及しない」）と一貫

## 6. セキュリティ考慮事項

- **プレゼンス情報のスコープ制限**: プレゼンス情報はチーム内のみ閲覧可能。チーム外のユーザーからは一切アクセス不可
- **お出かけ先の情報保護**: `destination` は GPS 位置情報ではなくテキスト入力のみ（位置追跡機能ではない）。リアルタイム位置共有は行わない
- **帰宅遅延通知の段階制御**: 15分超過は ADMIN のみ（全メンバーに送ると晒しになりうる）。1時間超過で初めてチーム全員に通知。以降は追加通知なし。`overdue_level`（0/1/2）で段階管理し重複防止
- **コイントスの悪用防止**: レートリミット（Bucket4j: 1分10回）で連続実行を制限
- **ロール呼称の制限**: 呼称はあくまで表示名のみ。権限・認可ロジックは常にシステムロール名（`ADMIN` 等）で判定。エイリアスで権限が変わることはない
