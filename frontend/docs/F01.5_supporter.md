## 概要

チーム・組織の個別ページ（`/teams/[id]`、`/organizations/[id]`）に、非メンバーがサポーター申請できるボタンと、管理者がサポーターを承認・管理できるタブを追加する。

---

## ユーザー向け機能（非メンバー）

### 表示条件

- `supporterEnabled === true` のチーム・組織
- ログイン済みで、そのチーム・組織のメンバーでない（`roleName === null`）

### UI（ヘッダー右端）

| followStatus | 表示内容                                                                                 |
| ------------ | ---------------------------------------------------------------------------------------- |
| `NONE`       | 「サポーターになる」ボタン（ハートアイコン・セカンダリ枠線）                             |
| `PENDING`    | 「申請中（承認待ち）」テキスト（オレンジ色）+ 取消ボタン                                 |
| `APPROVED`   | 「サポーターです」ボタン（**赤いハートfillアイコン・赤枠線**）→ クリックで確認ダイアログ |

#### APPROVED 時の確認ダイアログ

- **タイトル**：「サポーターをやめますか？」
- **本文**：「{チーム名 or 組織名}のサポーターをやめます。よろしいですか？」
- **ボタン**：「キャンセル」「やめる（danger）」

### フロー

```
ユーザー操作                    フロントエンド              バックエンド
──────────────────────────────────────────────────────────────────────
「サポーターになる」クリック      POST /follow              ─→ autoApprove=true  → status: APPROVED
                                                          ─→ autoApprove=false → status: PENDING
                                GET /follow/status        ← status 返却
                                トースト通知表示

「サポーターです」ボタンクリック  確認ダイアログ表示
「やめる」クリック               DELETE /follow            ─→ PENDING・APPROVED どちらも取消
                                followStatus = NONE
                                ダイアログを閉じる
                                トースト「サポーターをやめました」

「申請中」取消ボタンクリック      DELETE /follow            ─→ PENDING を取消
                                followStatus = NONE
```

## バックエンド実装依頼

> フロントエンドの実装は完了しており、以下のエンドポイントがバックエンドに存在しない状態です。

### 新規実装エンドポイント

チーム（`/api/v1/teams/{teamId}/...`）と組織（`/api/v1/organizations/{orgId}/...`）で同一構成。計16本。

| #   | メソッド | パス                                   | 説明                                       | 優先度   |
| --- | -------- | -------------------------------------- | ------------------------------------------ | -------- |
| 1   | `GET`    | `/follow/status`                       | 自分のフォロー状態取得                     | **必須** |
| 2   | `GET`    | `/supporter-settings`                  | 自動承認設定取得                           | **必須** |
| 3   | `PUT`    | `/supporter-settings`                  | 自動承認設定更新                           | **必須** |
| 4   | `GET`    | `/supporters`                          | 承認済みサポーター一覧（ページネーション） | **必須** |
| 5   | `GET`    | `/supporter-applications`              | 申請一覧（全ステータス、ページネーション） | **必須** |
| 6   | `POST`   | `/supporter-applications/{id}/approve` | 申請を個別承認                             | **必須** |
| 7   | `POST`   | `/supporter-applications/{id}/reject`  | 申請を個別却下                             | **必須** |
| 8   | `POST`   | `/supporter-applications/bulk-approve` | 申請を一括承認                             | **必須** |

### 既存エンドポイントの挙動変更

| エンドポイント                                | 現在の挙動       | 必要な挙動                                                                                 |
| --------------------------------------------- | ---------------- | ------------------------------------------------------------------------------------------ |
| `POST /api/v1/teams/{teamId}/follow`          | 即時フォロー     | `autoApprove=true` → 即時承認（`APPROVED`）<br>`autoApprove=false` → 申請作成（`PENDING`） |
| `DELETE /api/v1/teams/{teamId}/follow`        | 即時アンフォロー | `PENDING`・`APPROVED` 両方のフォローを取消                                                 |
| `POST /api/v1/organizations/{orgId}/follow`   | 同上             | 同上                                                                                       |
| `DELETE /api/v1/organizations/{orgId}/follow` | 同上             | 同上                                                                                       |

### レスポンス型定義

```typescript
// GET /follow/status
{
  data: {
    status: 'NONE' | 'PENDING' | 'APPROVED'
  }
}

// POST /follow（挙動変更後）
// autoApprove=true  → 201 Created: { data: { status: 'APPROVED' } }
// autoApprove=false → 201 Created: { data: { status: 'PENDING' } }

// DELETE /follow
// 204 No Content

// GET /supporter-settings
// PUT /supporter-settings（リクエストボディ: { autoApprove: boolean }）
{
  data: {
    autoApprove: boolean
  }
}

// GET /supporters
// クエリパラメータ: page=0, size=50
{
  data: Array<{
    userId: number
    displayName: string
    avatarUrl: string | null
    followedAt: string // ISO8601 承認日時
  }>
  meta: {
    page: number
    size: number
    totalElements: number
    totalPages: number
  }
}

// GET /supporter-applications
// クエリパラメータ: page=0, size=50
// ※ フロントは全ステータス取得後に PENDING のみ表示する実装のため、
//    バックエンドは全ステータスをそのまま返すこと
{
  data: Array<{
    id: number
    userId: number
    displayName: string
    avatarUrl: string | null
    message: string | null
    status: 'PENDING' | 'APPROVED' | 'REJECTED'
    createdAt: string // ISO8601
  }>
  meta: {
    page: number
    size: number
    totalElements: number
    totalPages: number
  }
}

// POST /supporter-applications/{id}/approve
// POST /supporter-applications/{id}/reject
// 204 No Content

// POST /supporter-applications/bulk-approve
// リクエストボディ: { applicationIds: number[] }
// 204 No Content
```
