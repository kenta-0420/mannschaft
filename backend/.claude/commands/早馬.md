# 早馬（緊急バグ修正）

軍議を省略し、殿様が自ら刀を振るって緊急修正を行う簡易フロー。
火急の事態にのみ用いること。

## 適用条件（以下のすべてを満たすこと）
- ビルドが通らない、テストが落ちている等の **緊急性の高い問題**
- 修正範囲が **1〜3ファイル以内** で収まる軽微な修正
- 設計変更を伴わない（既存設計の範囲内での修正）

## 手順

1. **状況把握**: エラーメッセージ・失敗テストを確認
2. **原因特定**: 該当ファイルを読み、原因を特定
3. **修正実施**: 殿様が直接修正（worktree不要、足軽も不要）
4. **動作確認**: `gradle build` + 該当テストの実行
5. **早馬報告**: マスターに修正内容を報告

## 発言者ラベル規約（必須）
全ての報告に `🏯殿様:` ラベルを先頭に付与すること。

## 早馬報告フォーマット

```
🏯殿様:【早馬】マスター、火急の修正を申し上げます

■ 異変の内容
- {何が起きたか: ビルドエラー、テスト失敗等}

■ 原因
- {原因の簡潔な説明}

■ 修正内容
- {ファイル名}: {修正概要}

■ 動作確認
- ビルド: ✅ 成功
- テスト: ✅ パス

コミットの御裁可をお願いいたします。
```

## 禁止事項
- 設計変更を伴う修正には使わない（`/軍議` を使うこと）
- 4ファイル以上にまたがる修正には使わない
- 「ついでにリファクタリング」は厳禁。火消しに専念すること

## 注意事項
- 早馬は例外措置。濫用すると軍議を経ない変更が積み重なり、設計が崩れる
- 早馬を使った場合、次回の軍議で「早馬で対処した件」として報告すること
