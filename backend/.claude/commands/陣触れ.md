# 陣触れ（フロントエンド起動）

Nuxt 開発サーバーを起動する。
**使い方**: `/伝令` の後に `/陣触れ` を実行。バックエンドとフロントエンドが揃い、開発環境が整う。

## 前提条件

- `/伝令` 済みであること（Spring Boot 稼働中 + swagger.json 取得済み）

## 手順

### 1. 既存プロセス確認

3000 ポートで既に起動していないか確認する:

```bash
if curl -s -o /dev/null -w "%{http_code}" http://localhost:3000 2>/dev/null | grep -q "200"; then
  echo "Nuxt は既に起動中です（スキップ）"
  exit 0
fi
```

### 2. 依存パッケージ確認

node_modules が存在しない場合のみインストールする:

```bash
if [ ! -d "/c/mannschaft-frontend/node_modules" ]; then
  echo "node_modules が見つかりません。npm install を実行します..."
  cd /c/mannschaft-frontend && npm install
fi
```

### 3. Nuxt 開発サーバー起動

バックグラウンドで起動し、ログをファイルに記録する:

```bash
cd /c/mannschaft-frontend && npm run dev > /tmp/nuxt-dev.log 2>&1 &
echo "Nuxt 起動中... (PID: $!)"
```

起動完了を待機（最大60秒）:

```bash
echo "Nuxt 起動待機中..."
for i in $(seq 1 60); do
  if curl -s -o /dev/null -w "%{http_code}" http://localhost:3000 2>/dev/null | grep -q "200"; then
    echo "起動確認 (${i}秒)"
    break
  fi
  sleep 1
  if [ "$i" -eq 60 ]; then
    echo "ERROR: 60秒以内に起動しませんでした"
    echo "--- ログ末尾 ---"
    tail -20 /tmp/nuxt-dev.log
    exit 1
  fi
done
```

### 4. 結果報告

| 項目 | 状態 |
|------|------|
| Nuxt 起動 | ✅ / ❌ |
| URL | http://localhost:3000 |

✅ なら「**陣触れ完了。フロントエンド陣営、出陣準備整いました。http://localhost:3000**」と報告する。
❌ の場合は `/tmp/nuxt-dev.log` の末尾を確認し、原因を提示する。

## 備考

- Nuxt の停止は `/撤収` に追加するか、`lsof -ti:3000 | xargs kill` で手動停止
- ポートが 3000 以外になる場合（競合時）はログで確認する
- `swagger.json` を更新した場合は Nuxt の型生成が自動で走る（Nuxt の設定による）
