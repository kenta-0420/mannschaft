# 陣立て（開発環境セットアップ）

以下の手順を順番に実行し、開発環境を整えてください。

## 1. WSL2 起動 & キープアライブ

WSL2 はアイドル時に自動停止するため、バックグラウンドプロセスで維持する。

```bash
wsl -d Ubuntu-24.04 -- bash -c "nohup sleep infinity </dev/null &>/dev/null & disown; echo 'WSL2 keepalive started'"
```

## 2. Docker コンテナ起動

MySQL 8.0 + Valkey 8 を docker compose で起動する。

```bash
wsl -d Ubuntu-24.04 -- bash -c "cd /mnt/c/cloudeProject && docker compose up -d"
```

## 3. ミドルウェア接続確認

MySQL と Valkey が Windows 側の localhost から到達可能か確認する。
両方とも `TcpTestSucceeded: True` であること。

```bash
powershell.exe -Command "
  Write-Host '=== MySQL (3306) ===' ;
  (Test-NetConnection -ComputerName localhost -Port 3306 -WarningAction SilentlyContinue).TcpTestSucceeded ;
  Write-Host '=== Valkey (6379) ===' ;
  (Test-NetConnection -ComputerName localhost -Port 6379 -WarningAction SilentlyContinue).TcpTestSucceeded
"
```

接続できない場合は、コンテナの状態を確認して対処すること：
```bash
wsl -d Ubuntu-24.04 -- bash -c "docker compose -f /mnt/c/cloudeProject/docker-compose.yml ps"
```

## 4. Gradle ビルド確認

テストはスキップして、コンパイルが通ることを確認する。

```bash
./gradlew build -x test
```

## 5. 結果報告

全ステップの結果を以下の形式で簡潔に報告する：

| 項目 | 状態 |
|------|------|
| WSL2 | ✅ / ❌ |
| MySQL | ✅ / ❌ |
| Valkey | ✅ / ❌ |
| Gradle ビルド | ✅ / ❌ |

全て ✅ なら「**陣立て完了。殿のご下命をお待ちしております。**」と報告する。
❌ がある場合は原因を調査し、自動修復を試みた上で再報告する。
