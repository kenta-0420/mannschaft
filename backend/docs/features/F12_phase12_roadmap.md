## 2. 新規設計が必要な機能（Phase 12 で設計書作成）

### F12.2 フィーチャーフラグ

**価値**: デプロイなしで機能の ON/OFF を切り替える。カナリアリリース・障害時の機能切り離し・A/B テスト。

**概要**:
- `feature_flags` テーブル（key, enabled, target_scope: GLOBAL/ORG/USER）
- SYSTEM_ADMIN が管理画面から操作
- バックエンド: `@FeatureFlag("webhook.test")` アノテーションまたはサービスメソッドで参照
- フロントエンド: `useFeatureFlag("new_dashboard")` カスタムフック
- キャッシュ（TTL 1分）でリクエストごとの DB 問い合わせを回避

---

### F12.3 GDPR / 個人情報管理

**価値**: 法的義務（GDPR・個人情報保護法）への対応。ユーザー自身がデータを管理できる信頼感。

**概要**:
- **データエクスポート**: `GET /api/account/data-export` → 自分の全データを JSON/ZIP でダウンロード
- **アカウント削除申請**: `DELETE /api/account` → 即時論理削除（物理削除は30日後バッチ）
- **データ保持ポリシー管理**: 退会後 N 日でデータ物理削除するバッチ
- 削除前に Stripe サブスクリプションを自動キャンセル

---

### F12.4 セッション管理画面

**価値**: 「見知らぬデバイスからのログイン」を自分で検知・無効化できるセキュリティ機能。

**概要**:
- `user_sessions` テーブル（deviceName, ip, lastActiveAt, createdAt）
- `GET /api/account/sessions` → アクティブセッション一覧
- `DELETE /api/account/sessions/{sessionId}` → 個別セッション無効化
- `DELETE /api/account/sessions` → 現セッション以外を一括ログアウト
- refresh_tokens テーブルと連携（既存 RefreshTokenRepository を活用）

---

### F12.5 フロントエンドエラー追跡

**価値**: ユーザーが黙って離脱している React クラッシュ・API 通信失敗を検知する。

**概要**:
- **Sentry** または互換 OSS（GlitchTip）を導入
- `ErrorBoundary` コンポーネントで React クラッシュをキャッチしてレポート
- API エラー（4xx/5xx）を axios interceptor で一括収集
- ユーザー ID・組織 ID をコンテキストに付与してエラーの影響範囲を特定

## 3. F11.3（UI i18n）の改善アイデア

> F11.3 を実装した後、余力があれば取り込む拡張。

### F12.6 i18n 拡張

**価値**: i18n の実用性・保守性を高める。

**概要**:

| アイデア | 内容 | 優先度 |
|---------|------|:------:|
| **複数形（Pluralization）** | `"1件のエラー"` vs `"3件のエラー"` を ICU メッセージフォーマット（`{count, plural, one{# 件} other{# 件}}` ）で表現 | 高 |
| **日付・数値・通貨のロケール書式** | `Intl.DateTimeFormat` / `Intl.NumberFormat` で `¥1,000` vs `€1.000` を自動適用。フロントエンドの共通ユーティリティとして整備 | 高 |
| **翻訳キー CI チェック** | PR マージ時に「どこかの言語ファイルにキーが存在しない」を検出して CI 失敗にする。`i18next-parser` または独自スクリプト | 高（保守） |
| **翻訳ファイルの遅延読み込み** | 起動時に全6言語をロードせず、使用言語のみ動的 import。初回ロード時間の短縮 | 中 |
| **バックエンド多段フォールバック** | `messages_{locale}.properties` → `messages_en.properties` → `messages.properties` の順で参照。現状は2段（locale → 基底）のみ | 中 |
| **言語プレビュー機能** | プロフィール設定で言語を変更する前に「プレビュー」できるモーダル。適用前に確認できる | 中 |
| **言語別フォント最適化** | 中国語（`Noto Sans SC`）・韓国語（`Noto Sans KR`）・Latin 系は `Inter` など、`font-family` を locale 別に切り替える CSS 変数設計 | 低 |
| **RTL 対応準備** | アラビア語・ヘブライ語追加に備え、CSS を `margin-left` ではなく `margin-inline-start` 等の論理プロパティで記述するルールを導入 | 低（将来） |
| **翻訳管理 Admin UI** | SYSTEM_ADMIN がコードを触らずブラウザから翻訳文言を更新・即時反映できる管理画面 | 将来 |
