# F11.6: スキル・資格管理

> **ステータス**: 🟢 設計完了
> **実装フェーズ**: Phase 11
> **最終更新**: 2026-03-19

---

## 1. 概要

メンバーが保有する資格・免許・スキルを登録・管理し、チーム/組織全体のスキルマトリクスを可視化する機能。資格の有効期限を追跡し、期限切れ前に自動リマインダーを送信することで、組織のコンプライアンスリスクを低減する。F06.2 メンバー紹介が有効な場合に自動で利用可能となる拡張機能であり、整骨院の施術者資格管理、スポーツチームの審判ライセンス管理、企業の IT 資格把握など、業種を問わず活用できる。

---

## 2. スコープ

### 対象ロール
| ロール | 操作可能な範囲 |
|--------|--------------|
| SYSTEM_ADMIN | 全組織・チームの資格データ参照・強制削除 |
| ADMIN | 資格カテゴリの定義・編集・削除、スキルマトリクス閲覧、全メンバーの資格閲覧・CSV エクスポート、メンバーの資格承認・却下 |
| DEPUTY_ADMIN | `MANAGE_MEMBER` 権限を持つ場合: スキルマトリクス閲覧、メンバーの資格閲覧・CSV エクスポート |
| MEMBER | 自分の資格の登録・編集・削除、自分の資格一覧閲覧、証明書画像のアップロード |
| SUPPORTER | 対象外 |
| GUEST | 対象外 |

### 対象レベル
- [x] 組織 (Organization)
- [x] チーム (Team)
- [ ] 個人 (Personal) — 個人は自分の資格のみ登録・閲覧（チーム/組織のスコープ経由）

## 3. DB設計

### テーブル一覧
| テーブル名 | 役割 | 論理削除 |
|-----------|------|---------|
| `skill_categories` | ADMIN が定義する資格カテゴリ（スコープ内共通） | あり |
| `member_skills` | メンバーが登録する個別の資格レコード | あり |
| `skill_expiry_notifications` | 有効期限リマインダーの送信履歴（二重送信防止） | なし |

### テーブル定義

#### `skill_categories`

ADMIN がスコープ内で定義する資格カテゴリ。例:「審判資格」「救急救命」「施術免許」「IT 資格」。

| カラム名 | 型 | NULL | デフォルト | 説明 |
|---------|---|------|-----------|------|
| `id` | BIGINT UNSIGNED | NO | AUTO_INCREMENT | PK |
| `scope_type` | VARCHAR(20) | NO | — | スコープ種別（'TEAM' / 'ORGANIZATION'） |
| `scope_id` | BIGINT UNSIGNED | NO | — | チーム or 組織の ID |
| `name` | VARCHAR(100) | NO | — | カテゴリ名（例: 審判資格） |
| `description` | VARCHAR(500) | YES | NULL | カテゴリの説明 |
| `icon` | VARCHAR(50) | YES | NULL | 表示用アイコン識別子（フロントエンド側のアイコンライブラリキー） |
| `sort_order` | INT | NO | 0 | 表示順 |
| `is_active` | BOOLEAN | NO | TRUE | FALSE = 非表示（新規登録時にカテゴリ候補に表示しない） |
| `created_by` | BIGINT UNSIGNED | YES | NULL | FK → users。ON DELETE SET NULL |
| `created_at` | DATETIME | NO | CURRENT_TIMESTAMP | |
| `updated_at` | DATETIME | NO | CURRENT_TIMESTAMP ON UPDATE | |
| `deleted_at` | DATETIME | YES | NULL | 論理削除 |

**インデックス**
```sql
UNIQUE KEY uq_sc_scope_name (scope_type, scope_id, name, deleted_at)  -- スコープ内カテゴリ名一意性
INDEX idx_sc_scope_order (scope_type, scope_id, sort_order)            -- スコープ別カテゴリ一覧
```

**制約・備考**
```sql
CONSTRAINT chk_sc_scope_type
  CHECK (scope_type IN ('TEAM', 'ORGANIZATION'))
```
- 論理削除: `deleted_at DATETIME nullable`（SoftDeletableEntity 適用）
- カテゴリを論理削除しても、紐づく `member_skills` レコードは保持される（過去の資格記録の完全性維持）
- `is_active = FALSE` にすると新規資格登録時のカテゴリ候補に表示されないが、既存レコードの表示には影響しない

#### `member_skills`

メンバーが登録する個別の資格レコード。1ユーザーが同一カテゴリで複数の資格を登録可能（例:「審判3級」と「審判2級」を別レコードとして記録）。

| カラム名 | 型 | NULL | デフォルト | 説明 |
|---------|---|------|-----------|------|
| `id` | BIGINT UNSIGNED | NO | AUTO_INCREMENT | PK |
| `skill_category_id` | BIGINT UNSIGNED | NO | — | FK → skill_categories（ON DELETE RESTRICT） |
| `user_id` | BIGINT UNSIGNED | NO | — | FK → users（ON DELETE CASCADE） |
| `scope_type` | VARCHAR(20) | NO | — | スコープ種別（'TEAM' / 'ORGANIZATION'）。skill_categories と同一スコープ |
| `scope_id` | BIGINT UNSIGNED | NO | — | チーム or 組織の ID |
| `name` | VARCHAR(200) | NO | — | 資格名（例: 審判3級、柔道整復師、AWS Solutions Architect） |
| `issuer` | VARCHAR(200) | YES | NULL | 発行機関（例: 日本サッカー協会、厚生労働省） |
| `credential_number` | VARCHAR(100) | YES | NULL | 資格番号・免許番号 |
| `acquired_on` | DATE | YES | NULL | 取得日 |
| `expires_at` | DATETIME | YES | NULL | 有効期限。NULL = 永久資格（有効期限なし） |
| `status` | VARCHAR(20) | NO | 'ACTIVE' | 資格ステータス（'ACTIVE' / 'EXPIRED' / 'PENDING_REVIEW'） |
| `certificate_s3_key` | VARCHAR(500) | YES | NULL | 証明書画像の S3 オブジェクトキー |
| `notes` | VARCHAR(500) | YES | NULL | メモ（取得経緯、更新予定等） |
| `verified_at` | DATETIME | YES | NULL | ADMIN による確認日時。NULL = 未確認 |
| `verified_by` | BIGINT UNSIGNED | YES | NULL | FK → users。ON DELETE SET NULL（確認した ADMIN） |
| `version` | BIGINT | NO | 0 | 楽観的ロック用 |
| `created_at` | DATETIME | NO | CURRENT_TIMESTAMP | |
| `updated_at` | DATETIME | NO | CURRENT_TIMESTAMP ON UPDATE | |
| `deleted_at` | DATETIME | YES | NULL | 論理削除 |

**インデックス**
```sql
INDEX idx_ms_user_scope (user_id, scope_type, scope_id)                     -- ユーザー別スコープ内資格一覧
INDEX idx_ms_scope_category (scope_type, scope_id, skill_category_id)       -- スキルマトリクス用（カテゴリ別集計）
INDEX idx_ms_expires (expires_at)                                            -- 有効期限リマインダーバッチ用
INDEX idx_ms_status (scope_type, scope_id, status)                           -- ステータス別フィルタ
UNIQUE KEY uq_ms_user_category_name (user_id, skill_category_id, name, deleted_at)  -- 同一ユーザー・カテゴリ・資格名の重複防止
```

**制約・備考**
```sql
CONSTRAINT chk_ms_scope_type
  CHECK (scope_type IN ('TEAM', 'ORGANIZATION'))

CONSTRAINT chk_ms_status
  CHECK (status IN ('ACTIVE', 'EXPIRED', 'PENDING_REVIEW'))
```
- 論理削除: `deleted_at DATETIME nullable`（SoftDeletableEntity 適用）
- `member_profiles` テーブルとは直接 FK を持たず、`user_id` で JOIN する（`member_profiles` はページ単位のためスコープが異なる）
- `expires_at` が NULL の場合は永久資格として扱い、リマインダー対象外
- `status = 'EXPIRED'` への遷移は日次バッチで自動更新（`expires_at < NOW()` かつ `status = 'ACTIVE'`）
- `status = 'PENDING_REVIEW'` はメンバーが新規登録した直後の初期ステータス。ADMIN が確認（verify）すると `ACTIVE` に遷移。ADMIN 設定で自動承認モードにした場合は登録時に即 `ACTIVE`
- 楽観的ロック: `@Version` 適用（複数管理者の同時編集を考慮）
- 証明書画像: Pre-signed URL でアップロード。ADMIN + 本人のみ閲覧可能。MIME タイプは `image/jpeg`, `image/png`, `image/webp`, `application/pdf` を許可

#### `skill_expiry_notifications`

有効期限リマインダーの送信履歴。同一資格に対する二重送信を防止するためのログテーブル。

| カラム名 | 型 | NULL | デフォルト | 説明 |
|---------|---|------|-----------|------|
| `id` | BIGINT UNSIGNED | NO | AUTO_INCREMENT | PK |
| `member_skill_id` | BIGINT UNSIGNED | NO | — | FK → member_skills（ON DELETE CASCADE） |
| `notification_type` | VARCHAR(30) | NO | — | リマインダー種別（'DAYS_30' / 'DAYS_7'） |
| `sent_at` | DATETIME | NO | CURRENT_TIMESTAMP | 送信日時 |

**インデックス**
```sql
UNIQUE KEY uq_sen_skill_type (member_skill_id, notification_type)  -- 同一資格・種別の二重送信防止
```

**制約・備考**
```sql
CONSTRAINT chk_sen_notification_type
  CHECK (notification_type IN ('DAYS_30', 'DAYS_7'))
```
- 物理削除。`member_skills` 削除時にカスケード削除
- BaseEntity を継承しない軽量ログテーブル

### ER図（テキスト形式）

```
teams/organizations (1) ──── (N) skill_categories
                                    │
skill_categories    (1) ──── (N) member_skills
users               (1) ──── (N) member_skills
                                    │
member_skills       (1) ──── (N) skill_expiry_notifications

-- F06.2 連携（直接 FK なし、user_id で JOIN）
member_skills.user_id ── references ── member_profiles.user_id（間接参照）

-- F04.3 連携（イベント経由）
member_skills → SkillExpiryReminderEvent → notifications（F04.3）
```

## 5. ビジネスロジック

### 資格登録フロー

```
1. MEMBER が POST /skills で資格を登録
2. バックエンドがバリデーション:
   a. skill_category_id がスコープ内かつ is_active = true か確認
   b. 同一ユーザー・カテゴリ・資格名の重複チェック
   c. certificate_s3_key が指定されている場合、S3 に HeadObject で存在確認
3. member_skills レコードを INSERT（status = PENDING_REVIEW）
4. ApplicationEvent（SkillRegisteredEvent）を発行
5. 通知リスナーが ADMIN に「新しい資格が登録されました」と通知（F04.3 連携）
```

### ADMIN 確認（verify）フロー

```
1. ADMIN が PATCH /skills/{id}/verify を呼び出し
2. バックエンドが検証:
   a. 資格の status が PENDING_REVIEW であること
   b. 楽観的ロック（version）の一致確認
3. status を ACTIVE に更新、verified_at と verified_by を記録
4. ApplicationEvent（SkillVerifiedEvent）を発行
5. 通知リスナーが資格登録者に「資格が確認されました」と通知
```

### 有効期限リマインダーバッチ

```
@Scheduled 日次バッチ（08:00 JST）:

1. 期限30日前リマインダー:
   SELECT ms.* FROM member_skills ms
   WHERE ms.expires_at BETWEEN NOW() AND DATE_ADD(NOW(), INTERVAL 30 DAY)
     AND ms.status = 'ACTIVE'
     AND ms.deleted_at IS NULL
     AND NOT EXISTS (
       SELECT 1 FROM skill_expiry_notifications sen
       WHERE sen.member_skill_id = ms.id
         AND sen.notification_type = 'DAYS_30'
     )

2. 各対象レコードに対して:
   a. skill_expiry_notifications に DAYS_30 レコードを INSERT（二重送信防止）
   b. ApplicationEvent（SkillExpiryReminderEvent）を発行
   c. F04.3 プッシュ通知: 「[資格名] の有効期限が30日後に迫っています」

3. 期限7日前リマインダー:
   同様のロジックで notification_type = 'DAYS_7' を処理
   通知メッセージ: 「[資格名] の有効期限が7日後に迫っています。更新をお忘れなく」

4. 期限切れ自動ステータス更新:
   UPDATE member_skills
   SET status = 'EXPIRED', updated_at = NOW()
   WHERE expires_at < NOW()
     AND status = 'ACTIVE'
     AND deleted_at IS NULL
```

### スキルマトリクス生成ロジック

```
1. スコープ内の全 skill_categories（is_active = true かつ deleted_at IS NULL）を取得 → 列定義
2. スコープ内の全メンバー（users テーブルからチーム/組織の所属メンバーを取得）→ 行定義
3. member_skills を LEFT JOIN で取得:
   - status IN ('ACTIVE', 'EXPIRED') のもの（PENDING_REVIEW は除外）
   - 同一カテゴリで複数資格がある場合は ACTIVE 優先、次に expires_at DESC で1件を代表選出
4. ピボット形式に変換して返却
```

### 証明書画像アクセス制御

```
GET /skills/{id}/certificate-url:
1. member_skills レコードを取得
2. アクセス権限チェック:
   a. リクエストユーザーが資格の登録者本人（user_id = currentUser）→ 許可
   b. リクエストユーザーが同一スコープの ADMIN → 許可
   c. それ以外 → 403 Forbidden
3. certificate_s3_key が NULL → 404
4. S3 Pre-signed URL（有効期限: 5分、Content-Disposition: inline）を生成して返却
```

### CSV エクスポートロジック

```
1. ADMIN が GET /skills/export を呼び出し
2. 非同期ジョブ（@Async("job-pool")）として起動
3. スコープ内の全 member_skills を取得（フィルタ条件適用）
4. users テーブルと JOIN してメンバー名を取得
5. CSV を生成し S3 の一時バケットに保存（有効期限: 24時間）
6. ジョブ進捗を Redis に記録（mannschaft:job:export:{jobId}、TTL 24時間）
```

## 6. セキュリティ考慮事項

- **認可チェック**: SkillCategoryService / MemberSkillService の入り口で `teamId`（または `organizationId`）と `currentUser` の所属を必ず検証する
- **証明書画像のプライバシー**: 証明書画像は ADMIN + 本人のみ閲覧可能。他メンバーには `has_certificate` フラグのみ返却し、S3 キーやダウンロード URL は非公開
- **証明書画像のバリデーション**: アップロードされたファイルの Content-Type と Magic Bytes を検証（偽装防止）。許可形式: JPEG, PNG, WebP, PDF
- **EXIF ストリップ**: 画像アップロード時に GPS 情報を含む EXIF データを除去（位置情報漏洩防止）
- **レートリミット**: 証明書画像アップロード API に `Bucket4j` で 1分間に10回の制限
- **ファイルサイズ制限**: 証明書画像は1枚あたり最大 10MB
- **CSV エクスポートの保護**: エクスポートファイルの S3 Pre-signed URL は24時間で失効。ADMIN のみ実行可能
- **資格番号の秘匿**: スキルマトリクスや検索結果に `credential_number` は含めない。詳細画面（ADMIN + 本人のみ）でのみ表示
- **入力サニタイズ**: `name`, `issuer`, `notes` などの文字列フィールドに対して XSS 対策（HTML タグのエスケープ）
