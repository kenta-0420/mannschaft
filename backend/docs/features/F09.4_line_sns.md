# F09.4: LINE/SNS 連携

> **ステータス**: 🟡 設計中
> **実装フェーズ**: Phase 9
> **最終更新**: 2026-03-15
> **関連ドキュメント**: F04.3 プッシュ通知（NotificationChannel 連携）、F08 外部連携（OAuth・トークン管理パターン）、F01.1 認証（ソーシャルログイン）

---

## 1. 概要

LINE Messaging API と Instagram フィードを統合し、組織・チームの外部コミュニケーションと情報発信を強化する機能。

**LINE 連携**: LINE 公式アカウント（BOT）をチーム/組織に紐付け、プラットフォームの通知（スケジュール変更・出欠リマインド・回覧板・掲示板等）を LINE 経由で配信する。ユーザーは自分の LINE アカウントをリンクすることで、プッシュ通知の代わりに LINE メッセージで受信できる。LINE Webhook による双方向メッセージングも対応し、簡易な返信（出欠回答等）を LINE 上で完結させる。

**SNS 連携**: Instagram のフィードをチームページに埋め込み表示する。DB にデータを持たず Redis キャッシュのみで運用し、外部 API の変更に対する保守負荷を最小化する。

> **注**: X（旧 Twitter）フィード連携は API 利用料（Basic プラン月$100〜）の観点から Phase 9 では実装しない。将来的に収益に余裕ができた段階で追加する（FUTURE_CONSIDERATIONS.md に記載）。`sns_feed_configs` テーブルの `provider` カラムは `X` を将来追加可能な設計としている。

### テーブル定義

#### `line_bot_configs`

チーム/組織レベルの LINE 公式アカウント（BOT）設定。

| カラム名 | 型 | NULL | デフォルト | 説明 |
|---------|---|------|-----------|------|
| `id` | BIGINT UNSIGNED | NO | AUTO_INCREMENT | PK |
| `scope_type` | VARCHAR(20) | NO | — | `TEAM` / `ORGANIZATION` |
| `scope_id` | BIGINT UNSIGNED | NO | — | teams.id / organizations.id |
| `channel_id` | VARCHAR(100) | NO | — | LINE Messaging API のチャンネル ID |
| `channel_secret_enc` | VARBINARY(512) | NO | — | チャンネルシークレット（AES-256-GCM 暗号化） |
| `channel_access_token_enc` | VARBINARY(1024) | NO | — | チャンネルアクセストークン（AES-256-GCM 暗号化） |
| `encryption_key_version` | INT UNSIGNED | NO | 1 | 暗号化キーのバージョン（ローテーション対応） |
| `webhook_secret` | VARCHAR(64) | NO | — | Webhook 検証用シークレット（URL-safe ランダム文字列） |
| `bot_user_id` | VARCHAR(50) | YES | NULL | LINE BOT のユーザー ID（初回 Webhook 受信時に自動設定） |
| `is_active` | BOOLEAN | NO | TRUE | BOT の有効/無効 |
| `notification_enabled` | BOOLEAN | NO | TRUE | 通知配信の有効/無効（一時停止用） |
| `configured_by` | BIGINT UNSIGNED | NO | — | FK → users（設定した ADMIN） |
| `created_at` | DATETIME | NO | CURRENT_TIMESTAMP | |
| `updated_at` | DATETIME | NO | CURRENT_TIMESTAMP ON UPDATE | |
| `deleted_at` | DATETIME | YES | NULL | 論理削除 |

**インデックス**
```sql
UNIQUE KEY uq_lbc_scope (scope_type, scope_id, deleted_at)  -- スコープ別に1つの BOT 設定
INDEX idx_lbc_channel (channel_id)                            -- チャンネル ID で検索（Webhook 受信時）
```

**制約・備考**
- `channel_secret_enc` / `channel_access_token_enc`: AES-256-GCM で暗号化。暗号化キーは環境変数 or AWS Secrets Manager で管理（F08 Google Calendar と同一パターン）
- `encryption_key_version`: キーローテーション時にインクリメント。旧バージョンのキーも保持し、復号時にバージョンに応じたキーを使用
- `webhook_secret`: Webhook URL に含めるシークレット。`/api/v1/line/webhook/{webhook_secret}` の形式で外部からのリクエストを検証
- 1スコープにつき1つの LINE BOT のみ設定可能（UNIQUE 制約）
- `is_active = false` の場合、通知は送信されず Webhook も処理しない

#### `user_line_connections`

ユーザーと LINE アカウントの紐付け。LINE のアカウント連携フロー（リンクトークン方式）で確立。

| カラム名 | 型 | NULL | デフォルト | 説明 |
|---------|---|------|-----------|------|
| `id` | BIGINT UNSIGNED | NO | AUTO_INCREMENT | PK |
| `user_id` | BIGINT UNSIGNED | NO | — | FK → users（ON DELETE CASCADE） |
| `line_user_id` | VARCHAR(50) | NO | — | LINE のユーザー ID（U で始まる33文字） |
| `scope_type` | VARCHAR(20) | NO | — | どのスコープの BOT と連携したか |
| `scope_id` | BIGINT UNSIGNED | NO | — | teams.id / organizations.id |
| `display_name` | VARCHAR(100) | YES | NULL | LINE 上の表示名（プロフィール取得時に更新） |
| `line_login_sub` | VARCHAR(50) | YES | NULL | LINE ログイン（OpenID Connect）の sub 値。LINE ログイン済みユーザーの自動連携に使用 |
| `is_active` | BOOLEAN | NO | TRUE | 連携の有効/無効 |
| `linked_at` | DATETIME | NO | CURRENT_TIMESTAMP | アカウント連携日時 |
| `unlinked_at` | DATETIME | YES | NULL | 連携解除日時（解除しても行は保持。再連携時に更新） |

**インデックス**
```sql
UNIQUE KEY uq_ulc_user_scope (user_id, scope_type, scope_id)   -- ユーザー×スコープで1つの LINE 連携
INDEX idx_ulc_line_user (line_user_id)                           -- LINE ユーザー ID で検索（Webhook 受信時）
```

**制約・備考**
- `user_id` ON DELETE CASCADE: ユーザー退会時に連携も削除
- 1ユーザーは複数スコープ（チームA + チームB + 組織）それぞれの BOT と連携可能
- `line_user_id` は LINE BOT ごとに異なる（同一ユーザーでも BOT が違えば別の ID）
- `line_login_sub`: LINE ログイン（F01.1）済みのユーザーは、Social API（`GET /friendship/v1/status`）で BOT との友だち関係と `userId` を取得し、自動的にアカウント連携を成立させる（リンクトークン方式の手動フロー不要）。LINE ログインを使わないユーザーは従来通りリンクトークン方式で連携
- 連携解除時は `is_active = false` + `unlinked_at = NOW()`（行は削除しない。再連携時の履歴保持）

LINE 送信メッセージの履歴。デバッグ・配信レート管理・ADMIN の配信確認に使用。

| カラム名 | 型 | NULL | デフォルト | 説明 |
|---------|---|------|-----------|------|
| `id` | BIGINT UNSIGNED | NO | AUTO_INCREMENT | PK |
| `config_id` | BIGINT UNSIGNED | NO | — | FK → line_bot_configs |
| `line_user_id` | VARCHAR(50) | NO | — | 送信先の LINE ユーザー ID |
| `message_type` | VARCHAR(30) | NO | — | `TEXT` / `FLEX` / `TEMPLATE` |
| `notification_type` | VARCHAR(50) | YES | NULL | 元の通知タイプ（`SCHEDULE_UPDATED` / `CIRCULATION_REMIND` 等） |
| `content_summary` | VARCHAR(500) | NO | — | メッセージ内容の要約（本文は保存しない。個人情報保護） |
| `status` | VARCHAR(20) | NO | — | `SENT` / `FAILED` / `RATE_LIMITED` |
| `error_message` | VARCHAR(500) | YES | NULL | 送信失敗時のエラーメッセージ |
| `line_request_id` | VARCHAR(100) | YES | NULL | LINE API のレスポンスに含まれる request_id |
| `created_at` | DATETIME | NO | CURRENT_TIMESTAMP | |

**インデックス**
```sql
INDEX idx_lml_config (config_id, created_at DESC)      -- BOT 別の配信履歴
INDEX idx_lml_status (status, created_at DESC)          -- 失敗メッセージの確認
```

**制約・備考**
- INSERT のみ（更新・削除不可。監査ログと同一設計）
- `content_summary`: メッセージ本文をそのまま保存しない。「[スケジュール変更] 3/20 定例会議の時間が変更されました」のような要約のみ
- 保持期間: 90日。日次バッチで古いレコードを物理削除
- LINE の配信レート制限（月間無料枠: 200通。有料プランで増枠）の管理にも使用

### リクエスト／レスポンス仕様

#### `POST /api/v1/line/webhook/{webhookSecret}`

**認可**: 不要（LINE プラットフォームからのリクエスト。署名検証で認証）

LINE Messaging API からの Webhook イベントを受信する。

**リクエストヘッダ**
```
X-Line-Signature: {HMAC-SHA256 署名}
```

**リクエストボディ**: LINE Webhook Event Object（LINE 公式仕様に準拠）

**レスポンス（200 OK）**: 空レスポンス（LINE は 200 以外をリトライするため即時返却）

**処理フロー**
```
1. webhookSecret で line_bot_configs を検索 → 存在しなければ 404
2. X-Line-Signature を channel_secret で HMAC-SHA256 検証 → 不一致は 403
3. イベント種別に応じて非同期処理（@Async）:
   a. follow: user_line_connections に INSERT（新規連携）
   b. unfollow: user_line_connections.is_active = false
   c. message: 双方向メッセージング処理（出欠回答等）
   d. accountLink: リンクトークンによるアカウント連携確定
4. 200 を即時返却（非同期処理の完了を待たない）
```

---

#### `POST /api/v1/line/configs`

**認可**: ADMIN

LINE BOT 設定を作成する。

**リクエストボディ**
```json
{
  "scope_type": "TEAM",
  "scope_id": 1,
  "channel_id": "1234567890",
  "channel_secret": "abc123...",
  "channel_access_token": "eyJ..."
}
```

**レスポンス（201 Created）**
```json
{
  "data": {
    "id": 1,
    "scope_type": "TEAM",
    "scope_id": 1,
    "channel_id": "1234567890",
    "webhook_url": "https://api.mannschaft.app/api/v1/line/webhook/x7k9m2...",
    "is_active": true,
    "notification_enabled": true,
    "created_at": "2026-03-15T10:00:00"
  }
}
```

**補足**
- `channel_secret` / `channel_access_token` はレスポンスに含めない（セキュリティ）
- `webhook_url` は自動生成。ADMIN がこの URL を LINE Developers Console の Webhook URL に設定する
- 作成後に `POST /configs/{id}/test` でテストメッセージ送信を推奨

**エラーレスポンス**
| ステータス | 条件 |
|-----------|------|
| 400 | 必須フィールド未入力 / channel_id 形式不正 |
| 403 | ADMIN でない |
| 409 | 同一スコープに既に BOT 設定が存在 |

#### `POST /api/v1/line/link`

**認可**: ログイン済みユーザー（MEMBER 以上）

LINE アカウント連携を開始する。リンクトークン方式で安全に連携。

**リクエストボディ**
```json
{
  "scope_type": "TEAM",
  "scope_id": 1
}
```

**レスポンス（200 OK）**
```json
{
  "data": {
    "link_url": "https://access.line.me/dialog/bot/accountLink?linkToken=xxx&nonce=yyy",
    "expires_in": 600
  }
}
```

**補足**
- LINE のリンクトークン API を呼び出し、ユーザーを LINE のアカウント連携画面にリダイレクト
- ユーザーが LINE 側で承認すると、Webhook の `accountLink` イベントが発火 → `user_line_connections` に INSERT
- リンクトークンの有効期限: 10分

**エラーレスポンス**
| ステータス | 条件 |
|-----------|------|
| 400 | scope_type / scope_id 不正 |
| 404 | 対象スコープに LINE BOT が設定されていない |
| 409 | 既にリンク済み |

#### `GET /api/v1/sns/feeds`

**認可**: MEMBER 以上。`is_public = true` の場合は認証不要

SNS フィードを取得する。Redis キャッシュから返却し、キャッシュミス時は外部 API にフェッチ。

**クエリパラメータ**
| パラメータ | 型 | 必須 | 説明 |
|-----------|---|------|------|
| `scope_type` | String | Yes | `TEAM` / `ORGANIZATION` |
| `scope_id` | Long | Yes | 対象スコープの ID |
| `provider` | String | No | `INSTAGRAM`（Phase 9。将来 `X` 追加時に複数対応） |

**レスポンス（200 OK）**
```json
{
  "data": {
    "instagram": {
      "account_id": "@mannschaft_official",
      "posts": [
        {
          "id": "17895695668004550",
          "media_type": "IMAGE",
          "media_url": "https://scontent.cdninstagram.com/...",
          "caption": "チーム活動の様子 #mannschaft",
          "permalink": "https://www.instagram.com/p/...",
          "timestamp": "2026-03-14T15:30:00+0900"
        }
      ],
      "cached_at": "2026-03-15T10:00:00",
      "cache_ttl_minutes": 15
    },
    "x": null  // 将来追加予定（Phase 9 では未実装）
  }
}
```

**補足**
- Redis キャッシュキー: `sns_feed:{scope_type}:{scopeId}:{provider}`
- キャッシュヒット: Redis から即時返却
- キャッシュミス: 外部 API にリクエスト → Redis にセット（TTL = `cache_ttl_minutes`）→ 返却
- 外部 API エラー時: 期限切れキャッシュがあれば stale データを返却 + `is_stale: true` フラグ付与。なければ空配列

## 5. ビジネスロジック

### 主要フロー

#### LINE 通知配信フロー
```
1. F04.3 の NotificationService が通知イベントを生成
2. NotificationChannel の実装として LineChannel.send() が呼び出される:
   a. 対象ユーザーの user_line_connections を検索（is_active = true）
   b. 連携がなければスキップ（LINE 以外のチャンネルにフォールバック）
   c. 連携がある場合:
      - line_bot_configs から channel_access_token を復号
      - 通知タイプに応じてメッセージ構築:
        - TEXT: シンプルなテキストメッセージ
        - FLEX: リッチコンテンツ（ボタン付き。出欠回答等）
      - LINE Messaging API の Push API を呼び出し
      - line_message_logs に結果を INSERT
3. LINE API レスポンスに応じた処理:
   a. 200: status = SENT
   b. 429 (Rate Limited): status = RATE_LIMITED → 指数バックオフでリトライ（最大3回）
   c. 400/401/403: status = FAILED → error_message に記録。リトライしない
```

#### LINE アカウント連携フロー
```
1. ユーザーが POST /line/link を呼び出し
2. バックエンドが LINE のリンクトークン API を呼び出し:
   - POST https://api.line.me/v2/bot/user/{botUserId}/linkToken
   - linkToken + nonce を取得
3. フロントエンドが link_url にリダイレクト
4. ユーザーが LINE アプリ上で「連携する」を承認
5. LINE が Webhook で accountLink イベントを送信:
   a. nonce を検証（リプレイ攻撃防止）
   b. user_line_connections に INSERT（user_id + line_user_id + scope）
   c. ウェルカムメッセージを送信:「Mannschaft との連携が完了しました。通知を LINE で受信できます。」
```

#### LINE 双方向メッセージング（出欠回答）
```
1. Flex Message で出欠確認を送信:
   - 「3/20 定例会議に出席しますか？」
   - ボタン: [出席] [欠席] [未定]
2. ユーザーがボタンをタップ → Webhook の postback イベント受信
3. postback.data をパース（例: action=attendance&schedule_id=123&response=ATTEND）
4. F05 スケジュール機能の出欠回答 API を内部呼び出し
5. 応答メッセージ送信:「出席で登録しました ✓」
```

#### SNS フィードキャッシュフロー
```
1. GET /sns/feeds リクエスト受信
2. Redis キャッシュチェック:
   a. キャッシュヒット → 即時返却
   b. キャッシュミス → 3 へ
3. sns_feed_configs から API クレデンシャルを復号
4. Instagram Graph API を呼び出し:
   - GET /me/media?fields=id,media_type,media_url,caption,permalink,timestamp
5. レスポンスを整形 → Redis にセット（TTL = cache_ttl_minutes）
6. クライアントに返却
7. 外部 API エラー時:
   a. stale キャッシュがあれば返却（is_stale: true 付与）
   b. なければ空配列 + error フラグ
```

### 重要な判定ロジック

- **LINE 通知のチャンネル優先度**: ユーザーが LINE 連携済みの場合、F04.3 の通知設定で「LINE 優先」を選択可能。未選択の場合はプッシュ通知と LINE の両方に送信（デフォルト: 両方）
- **LINE 月間送信数の管理**: LINE の無料プランは月200通。`line_message_logs` の月間 SENT 件数で残数を管理。残数が10%以下になったら ADMIN に通知。0通になったら送信をスキップし、プッシュ通知にフォールバック
- **LINE BOT の複数スコープ**: 同一の LINE チャンネルを複数スコープで使い回すことは不可（`channel_id` の UNIQUE 制約は設けないが、アプリ層で警告表示）。理由: Webhook の振り分けが複雑になるため
- **SNS トークンの有効期限管理**: Instagram のアクセストークンは60日で期限切れ。期限30日前・7日前・1日前に ADMIN に通知。期限切れ後はフィード取得をスキップし、空配列を返却
- **SNS API 障害時の対応**: 外部 API のタイムアウト（5秒）後、stale キャッシュがあればそれを返却。完全なキャッシュミスの場合はフロントエンドで「フィードを取得できませんでした」を表示

## 6. セキュリティ考慮事項

- **トークン暗号化**: `channel_secret_enc` / `channel_access_token_enc` / `api_key_enc` 等は AES-256-GCM で暗号化。暗号化キーは環境変数 or AWS Secrets Manager（F08 Google Calendar と同一パターン）
- **Webhook 署名検証**: LINE からの Webhook リクエストは `X-Line-Signature` を `channel_secret` で HMAC-SHA256 検証。不一致は 403 で即時拒否
- **Webhook URL のシークレット**: `/api/v1/line/webhook/{webhookSecret}` の形式で、`webhookSecret` を知らない第三者は Webhook URL にアクセスできない（署名検証の前段階のフィルタ）
- **リプレイ攻撃防止**: アカウント連携の nonce は1回限り有効。使用済み nonce は Redis に TTL 付きで保存し、重複使用を拒否
- **レートリミット**: LINE 通知は LINE API のレートリミットに従う（1秒あたりの送信数制限）。SNS フィード取得は 1分間に10回（キャッシュにより通常は1回/TTL）
- **個人情報保護**: `line_message_logs` にメッセージ本文を保存しない（content_summary のみ）。`line_user_id` は LINE BOT 固有の ID であり、LINE アカウントの電話番号・メールアドレスとは紐付かない
- **SNS トークン露出防止**: SNS フィード API のレスポンスにトークン情報を含めない。管理 API でも暗号化カラムの値は返却しない（存在の有無のみ `has_api_key: true/false` で表示）
- **監査ログ**: LINE BOT 設定の作成・更新・削除、アカウント連携・解除を `audit_logs` に記録
