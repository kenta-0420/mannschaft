# F11.3: 書類テンプレート・フォームビルダー

> **ステータス**: 🟢 設計完了
> **実装フェーズ**: Phase 11
> **最終更新**: 2026-03-19
> **モジュール種別**: デフォルト機能 #30（全チーム・組織で利用可能）
> **関連ドキュメント**: F11.1 ワークフロー・承認エンジン（提出後の承認連携）、F05.3 電子印鑑（PDF 出力時の押印連携）、F05.4 アンケート・投票（フィールド定義の参考パターン）、F04.3 プッシュ通知（リマインダー連携）

---

## 1. 概要

組織・チーム内で使用する定型書類（入会申込書、退会届、同意書、各種届出）をノーコードのフォームビルダーで作成・配布・収集する機能。ADMIN がドラッグ＆ドロップでフィールドを配置してフォームテンプレートを設計し、MEMBER 以上がフォームに記入・提出する。提出後は F11.1 承認エンジンとの連携で承認フローに乗せることができ、承認完了後は F05.3 電子印鑑連携で押印済み PDF を生成する。メンバー情報のプレースホルダ自動入力、提出一覧の検索・CSV エクスポート、有効期限による自動クローズ、未提出者へのリマインダー通知にも対応する。

**主要ユースケース**:
- 整骨院: 問診票の電子化 → 来院前にスマホで記入
- マンション管理組合: 届出書（リフォーム届、ペット届、駐車場申請）の電子化
- 企業: 社内申請書（経費精算、休暇届、備品申請）
- 学校/PTA: 行事参加同意書、アレルギー調査票
- スポーツチーム: 入団申込書、保険加入同意書

## 3. DB設計

### テーブル一覧
| テーブル名 | 役割 | 論理削除 |
|-----------|------|---------|
| `form_templates` | フォームテンプレート（名称・説明・設定） | あり |

### テーブル定義

#### `form_templates`

フォームテンプレート。ADMIN がスコープ（チーム/組織）ごとに作成する。

| カラム名 | 型 | NULL | デフォルト | 説明 |
|---------|---|------|-----------|------|
| `id` | BIGINT UNSIGNED | NO | AUTO_INCREMENT | PK |
| `scope_type` | VARCHAR(20) | NO | — | スコープ種別（`TEAM` / `ORGANIZATION`） |
| `scope_id` | BIGINT UNSIGNED | NO | — | teams.id または organizations.id |
| `name` | VARCHAR(100) | NO | — | テンプレート名（例: 「入会申込書」「退会届」） |
| `description` | VARCHAR(500) | YES | NULL | テンプレートの説明 |
| `icon` | VARCHAR(30) | YES | NULL | アイコン（emoji or アイコン名） |
| `color` | VARCHAR(7) | YES | NULL | テーマカラー（hex。例: `#3B82F6`） |
| `status` | VARCHAR(20) | NO | 'DRAFT' | ステータス（`DRAFT` / `PUBLISHED` / `CLOSED`） |
| `requires_approval` | BOOLEAN | NO | FALSE | 提出時に F11.1 承認エンジンへ連携するか |
| `workflow_template_id` | BIGINT UNSIGNED | YES | NULL | FK → workflow_templates。`requires_approval = TRUE` の場合の承認テンプレート |
| `is_seal_on_pdf` | BOOLEAN | NO | FALSE | PDF 出力時に承認者の電子印を埋め込むか（F05.3 連携） |
| `deadline` | DATETIME | YES | NULL | 回答期限（NULL = 無期限。期限到達で自動 CLOSED） |
| `allow_edit_after_submit` | BOOLEAN | NO | FALSE | 提出後の編集を許可するか（承認前の PENDING 状態のみ） |
| `auto_fill_enabled` | BOOLEAN | NO | TRUE | メンバー情報のプレースホルダ自動入力を有効にするか |
| `max_submissions_per_user` | SMALLINT | NO | 1 | 1ユーザーあたりの最大提出回数（1 = 1回のみ、0 = 無制限） |
| `sort_order` | INT | NO | 0 | 表示順 |
| `preset_id` | BIGINT UNSIGNED | YES | NULL | FK → system_form_presets。プリセットから作成した場合のみ |
| `submission_count` | INT UNSIGNED | NO | 0 | 提出数（denormalize） |
| `target_count` | INT UNSIGNED | YES | NULL | 対象者数（公開時のスコープメンバー数をスナップショット。提出率算出用） |
| `created_by` | BIGINT UNSIGNED | NO | — | FK → users。ON DELETE RESTRICT |
| `published_at` | DATETIME | YES | NULL | 公開日時 |
| `closed_at` | DATETIME | YES | NULL | クローズ日時（手動 or 自動） |
| `version` | BIGINT | NO | 0 | 楽観的ロック用 |
| `created_at` | DATETIME | NO | CURRENT_TIMESTAMP | |
| `updated_at` | DATETIME | NO | CURRENT_TIMESTAMP ON UPDATE | |
| `deleted_at` | DATETIME | YES | NULL | 論理削除日時（SoftDeletableEntity 適用） |

**インデックス**
```sql
INDEX idx_ft_scope (scope_type, scope_id, status, sort_order)           -- スコープ別のフォーム一覧
INDEX idx_ft_deadline (status, deadline)                                  -- 自動クローズバッチ用
INDEX idx_ft_created_by (created_by, created_at DESC)                    -- 作成者別一覧
```

**制約・備考**
- `scope_type` + `scope_id` はポリモーフィック参照。FK 制約なし（アプリ層で検証）
- 1スコープあたり最大 **50 テンプレート**（`status` に関わらず論理削除されていないもの）
- `status` のライフサイクル: `DRAFT` → `PUBLISHED` → `CLOSED`
  - `DRAFT`: 下書き。フィールドの追加・変更が可能。提出不可
  - `PUBLISHED`: 公開中。提出受付中。**フィールドの追加・変更・削除は不可**（提出データの整合性を保つため）
  - `CLOSED`: クローズ済み。新規提出不可。閲覧・PDF ダウンロードのみ
- `requires_approval = TRUE` の場合、`workflow_template_id` は必須。提出時に F11.1 の external API 経由でワークフロー申請を自動作成（`source_type = 'FORM_SUBMISSION'`）
- `is_seal_on_pdf = TRUE` の場合、`requires_approval` も `TRUE` であること（承認なしで押印はできない）
- `max_submissions_per_user = 0`: 問診票のように来院ごとに提出が必要なケース向け
- テンプレートを論理削除しても、既存の提出データは閲覧・PDF ダウンロード可能

#### `form_template_fields`

テンプレートのフィールド定義。ADMIN がフォームビルダーで配置する各フィールドの設定。

| カラム名 | 型 | NULL | デフォルト | 説明 |
|---------|---|------|-----------|------|
| `id` | BIGINT UNSIGNED | NO | AUTO_INCREMENT | PK |
| `template_id` | BIGINT UNSIGNED | NO | — | FK → form_templates。ON DELETE CASCADE |
| `field_key` | VARCHAR(50) | NO | — | フィールドキー（英数字・アンダースコア。内部識別用） |
| `field_label` | VARCHAR(100) | NO | — | 表示ラベル（例: 「氏名」「生年月日」「住所」） |
| `field_type` | VARCHAR(20) | NO | — | フィールド型（後述） |
| `is_required` | BOOLEAN | NO | FALSE | 必須フィールドか |
| `placeholder` | VARCHAR(200) | YES | NULL | プレースホルダーテキスト |
| `auto_fill_key` | VARCHAR(50) | YES | NULL | プレースホルダ自動入力キー（例: `member_name`, `member_email`） |
| `options_json` | JSON | YES | NULL | 選択肢（`DROPDOWN` / `RADIO` / `CHECKBOX` 型用。例: `["男性", "女性", "その他"]`） |
| `validation_json` | JSON | YES | NULL | バリデーションルール（例: `{"min_length": 1, "max_length": 200, "pattern": "^[0-9]+$"}`） |
| `description` | VARCHAR(500) | YES | NULL | フィールドの補足説明（入力欄の下に表示） |
| `max_file_size_mb` | SMALLINT | YES | NULL | `FILE` 型の場合の最大ファイルサイズ（MB）。デフォルト: 10MB |
| `allowed_mime_types` | JSON | YES | NULL | `FILE` 型の許可 MIME タイプ（例: `["application/pdf", "image/jpeg"]`）。NULL = デフォルト許可リスト |
| `sort_order` | INT | NO | 0 | 表示順（フォームビルダーでのドラッグ＆ドロップ位置） |
| `created_at` | DATETIME | NO | CURRENT_TIMESTAMP | |
| `updated_at` | DATETIME | NO | CURRENT_TIMESTAMP ON UPDATE | |

**インデックス**
```sql
INDEX idx_ftf_template (template_id, sort_order)          -- テンプレート別のフィールド一覧
UNIQUE KEY uq_ftf_key (template_id, field_key)             -- テンプレート内でフィールドキー一意
```

**制約・備考**
- 1テンプレートあたり最大 **30 フィールド**
- `field_type` の種類:
  - `TEXT`: テキスト（1行。最大 200 文字）
  - `NUMBER`: 数値（DECIMAL(12,2)）
  - `DATE`: 日付（YYYY-MM-DD）
  - `EMAIL`: メールアドレス（形式バリデーション付き）
  - `PHONE`: 電話番号（ハイフン許容の形式バリデーション付き）
  - `DROPDOWN`: ドロップダウン選択（`options_json` から単一選択）
  - `RADIO`: ラジオボタン（`options_json` から単一選択）
  - `CHECKBOX`: チェックボックス（`options_json` から複数選択）
  - `TEXTAREA`: テキストエリア（複数行。最大 2,000 文字）
  - `FILE`: ファイル添付（S3 Pre-signed URL 方式）
  - `SIGNATURE`: 署名欄（Canvas で手書き → PNG として S3 保存）
  - `SECTION`: セクション見出し（入力フィールドなし。フォームの区切り表示用）
  - `DESCRIPTION`: 説明テキスト（入力フィールドなし。注意事項等の表示用）
- `auto_fill_key` の対応表:
  - `member_name` → `users.last_name` + `users.first_name`
  - `member_email` → `users.email`
  - `member_number` → `member_profiles.member_number`（チーム内会員番号）
  - `member_phone` → `users.phone_number`
  - `team_name` → `teams.name`
  - `org_name` → `organizations.name`
  - `current_date` → 提出時の日付（YYYY-MM-DD）
- `auto_fill_key` が設定されたフィールドは提出時に自動入力される。ユーザーが手動で上書き可能
- `SECTION` / `DESCRIPTION` 型は入力値を持たない。表示専用フィールド
- フィールド定義の変更ルール:
  - テンプレートが `DRAFT` の場合: 追加・削除・変更すべて可能
  - テンプレートが `PUBLISHED` / `CLOSED` の場合: 変更不可

#### `form_submissions`

フォーム提出レコード。MEMBER 以上が作成する。

| カラム名 | 型 | NULL | デフォルト | 説明 |
|---------|---|------|-----------|------|
| `id` | BIGINT UNSIGNED | NO | AUTO_INCREMENT | PK |
| `scope_type` | VARCHAR(20) | NO | — | スコープ種別（`TEAM` / `ORGANIZATION`） |
| `scope_id` | BIGINT UNSIGNED | NO | — | teams.id または organizations.id |
| `template_id` | BIGINT UNSIGNED | NO | — | FK → form_templates。ON DELETE RESTRICT |
| `submitted_by` | BIGINT UNSIGNED | NO | — | FK → users（提出者）。ON DELETE RESTRICT |
| `status` | VARCHAR(20) | NO | 'DRAFT' | ステータス（`DRAFT` / `SUBMITTED` / `PENDING` / `APPROVED` / `REJECTED` / `RETURNED`） |
| `workflow_request_id` | BIGINT UNSIGNED | YES | NULL | FK → workflow_requests。承認連携時の申請 ID |
| `submitted_at` | DATETIME | YES | NULL | 提出日時（`DRAFT` → `SUBMITTED` / `PENDING` に遷移した日時） |
| `completed_at` | DATETIME | YES | NULL | 完了日時（`APPROVED` に遷移した日時。承認不要の場合は `SUBMITTED` 時） |
| `pdf_file_key` | VARCHAR(500) | YES | NULL | 生成済み PDF の S3 オブジェクトキー |
| `pdf_generated_at` | DATETIME | YES | NULL | PDF 生成日時 |
| `version` | BIGINT | NO | 0 | 楽観的ロック用 |
| `created_at` | DATETIME | NO | CURRENT_TIMESTAMP | |
| `updated_at` | DATETIME | NO | CURRENT_TIMESTAMP ON UPDATE | |
| `deleted_at` | DATETIME | YES | NULL | 論理削除日時（SoftDeletableEntity 適用） |

**インデックス**
```sql
INDEX idx_fs_scope (scope_type, scope_id, template_id, status, submitted_at DESC)   -- スコープ別の提出一覧
INDEX idx_fs_submitted_by (submitted_by, status, submitted_at DESC)                  -- 提出者別の一覧
INDEX idx_fs_template (template_id, status, submitted_at DESC)                       -- テンプレート別の提出一覧
INDEX idx_fs_workflow (workflow_request_id)                                            -- ワークフロー申請からの逆引き
```

**制約・備考**
- `status` 遷移:
  - `DRAFT` → `SUBMITTED`（提出。`requires_approval = FALSE` の場合、最終ステータス）
  - `DRAFT` → `PENDING`（提出。`requires_approval = TRUE` の場合、承認待ちへ）
  - `PENDING` → `APPROVED`（承認完了。F11.1 WorkflowApprovedEvent 受信時）
  - `PENDING` → `REJECTED`（却下。F11.1 WorkflowRejectedEvent 受信時）
  - `PENDING` → `RETURNED`（差し戻し。F11.1 から差し戻し時）
  - `RETURNED` → `PENDING`（修正後の再提出）
  - `DRAFT` → 削除可能（論理削除）
- `submitted_by` ON DELETE RESTRICT: 提出者の退会時、提出データがある場合はエラー。ADMIN が先に提出を削除するか退会処理を完了させる
- 論理削除: `DRAFT` の提出のみ削除可能。`SUBMITTED` 以降は証跡として削除不可
- PDF 生成: `SUBMITTED` / `APPROVED` 状態の提出に対して実行可能。`is_seal_on_pdf = TRUE` かつ `APPROVED` の場合、承認者の電子印を PDF に埋め込む

#### `form_submission_values`

提出のフィールド値。EAV（Entity-Attribute-Value）パターン。フィールド単位の検索・集計を可能にするため JSON 方式ではなくレコード方式で保存する。

| カラム名 | 型 | NULL | デフォルト | 説明 |
|---------|---|------|-----------|------|
| `id` | BIGINT UNSIGNED | NO | AUTO_INCREMENT | PK |
| `submission_id` | BIGINT UNSIGNED | NO | — | FK → form_submissions。ON DELETE CASCADE |
| `field_id` | BIGINT UNSIGNED | NO | — | FK → form_template_fields。ON DELETE RESTRICT |
| `field_key` | VARCHAR(50) | NO | — | フィールドキー（スナップショット。フィールド削除後も値を識別するため） |
| `text_value` | TEXT | YES | NULL | テキスト系の値（`TEXT` / `EMAIL` / `PHONE` / `TEXTAREA` / `DROPDOWN` / `RADIO`） |
| `number_value` | DECIMAL(12,2) | YES | NULL | 数値型の値（`NUMBER`） |
| `date_value` | DATE | YES | NULL | 日付型の値（`DATE`） |
| `boolean_value` | BOOLEAN | YES | NULL | 真偽値（単一 `CHECKBOX` の場合） |
| `json_value` | JSON | YES | NULL | 複数選択の値（`CHECKBOX` 型で複数選択時。例: `["オプションA", "オプションB"]`） |
| `file_s3_key` | VARCHAR(500) | YES | NULL | ファイルの S3 オブジェクトキー（`FILE` / `SIGNATURE` 型） |
| `file_original_name` | VARCHAR(255) | YES | NULL | アップロード時の元ファイル名（`FILE` 型） |
| `file_size` | INT UNSIGNED | YES | NULL | ファイルサイズ（バイト） |
| `file_mime_type` | VARCHAR(100) | YES | NULL | MIME タイプ |
| `created_at` | DATETIME | NO | CURRENT_TIMESTAMP | |
| `updated_at` | DATETIME | NO | CURRENT_TIMESTAMP ON UPDATE | |

**インデックス**
```sql
UNIQUE KEY uq_fsv_submission_field (submission_id, field_id)              -- 提出内でフィールド値一意
INDEX idx_fsv_field_text (field_id, text_value(100))                      -- フィールド値でのテキスト検索
INDEX idx_fsv_field_number (field_id, number_value)                       -- フィールド値での数値検索
INDEX idx_fsv_field_date (field_id, date_value)                           -- フィールド値での日付検索
```

**制約・備考**
- 1提出あたり最大 **30 レコード**（テンプレートの最大フィールド数に対応）
- `SECTION` / `DESCRIPTION` 型のフィールドはレコードを作成しない（入力値なし）
- フィールド型ごとの値カラム使い分け:
  - `TEXT` / `EMAIL` / `PHONE` / `DROPDOWN` / `RADIO`: `text_value`
  - `TEXTAREA`: `text_value`
  - `NUMBER`: `number_value`
  - `DATE`: `date_value`
  - `CHECKBOX`（複数選択）: `json_value`
  - `FILE`: `file_s3_key` + `file_original_name` + `file_size` + `file_mime_type`
  - `SIGNATURE`: `file_s3_key`（Canvas で描画 → PNG として S3 保存）
- `field_key` はスナップショットとして保持。テンプレートのフィールドが将来削除されても、既存の提出値を参照時に「（削除済みフィールド: {field_key}）」として表示可能
- 署名フィールド: フロントエンドの Canvas API で手書き → PNG にエクスポート → S3 にアップロード → `file_s3_key` に保存。署名画像の最大サイズ: 500KB

#### `system_form_presets`

SYSTEM_ADMIN が管理する標準テンプレートプリセット。チーム/組織が「入会申込書」「退会届」等のプリセットを選んで自分のテンプレートとして複製する。

| カラム名 | 型 | NULL | デフォルト | 説明 |
|---------|---|------|-----------|------|
| `id` | BIGINT UNSIGNED | NO | AUTO_INCREMENT | PK |
| `name` | VARCHAR(100) | NO | — | プリセット名（例: 「入会申込書」「退会届」「イベント参加同意書」） |
| `description` | VARCHAR(500) | YES | NULL | プリセットの説明 |
| `category` | VARCHAR(30) | NO | — | カテゴリ（`MEMBERSHIP` / `CONSENT` / `APPLICATION` / `NOTIFICATION` / `OTHER`） |
| `icon` | VARCHAR(30) | YES | NULL | アイコン |
| `color` | VARCHAR(7) | YES | NULL | テーマカラー |
| `fields_json` | JSON | NO | — | フィールド定義の JSON 配列（複製時に `form_template_fields` に展開） |
| `is_active` | BOOLEAN | NO | TRUE | 有効か（FALSE = カタログに表示しない） |
| `sort_order` | INT | NO | 0 | 表示順 |
| `created_by` | BIGINT UNSIGNED | NO | — | FK → users（SYSTEM_ADMIN）。ON DELETE RESTRICT |
| `version` | BIGINT | NO | 0 | 楽観的ロック用 |
| `created_at` | DATETIME | NO | CURRENT_TIMESTAMP | |
| `updated_at` | DATETIME | NO | CURRENT_TIMESTAMP ON UPDATE | |
| `deleted_at` | DATETIME | YES | NULL | 論理削除日時 |

**インデックス**
```sql
INDEX idx_sfp_category (category, is_active, sort_order)    -- カテゴリ別のプリセット一覧
```

**制約・備考**
- `fields_json` の形式:
```json
[
  {
    "field_key": "name",
    "field_label": "氏名",
    "field_type": "TEXT",
    "is_required": true,
    "auto_fill_key": "member_name",
    "sort_order": 1
  },
  {
    "field_key": "email",
    "field_label": "メールアドレス",
    "field_type": "EMAIL",
    "is_required": true,
    "auto_fill_key": "member_email",
    "sort_order": 2
  }
]
```
- プリセットからテンプレートを作成する場合: `fields_json` を解析して `form_template_fields` にレコードを INSERT。作成後はテンプレートとプリセットは独立（プリセット変更はテンプレートに影響しない）
- `category` は表示時のグルーピング用。CHECK 制約で値を制限:
```sql
CONSTRAINT chk_sfp_category
```

## 5. ビジネスロジック

### テンプレート作成フロー

```
1. ADMIN がフォームビルダーでフィールドを配置
2. POST /forms/templates でテンプレート作成（status = DRAFT）
3. ADMIN がプレビューで確認
4. POST /forms/templates/{id}/publish で公開（status = PUBLISHED）
   a. target_count にスコープ内メンバー数をスナップショット
   b. published_at をセット
   c. ApplicationEvent (FormTemplatePublishedEvent) を発行
5. 通知サービスがイベントを受信:
   a. スコープ内の全 MEMBER にプッシュ通知（「新しいフォームが公開されました」）
```

### プリセットからのテンプレート作成フロー

```
1. ADMIN がプリセットカタログから「入会申込書」を選択
2. POST /forms/templates に preset_id を含めてリクエスト
3. Service 層で以下を実行:
   a. system_form_presets から fields_json を取得
   b. fields_json を解析して form_template_fields にレコードを INSERT
   c. form_templates.preset_id に参照元を記録
4. 作成後のテンプレートはプリセットから独立（ADMIN が自由にフィールドを編集可能）
```

### 提出フロー（承認なし）

```
1. MEMBER がフォーム入力画面を開く
2. auto_fill_enabled = TRUE の場合:
   a. auto_fill_key が設定されたフィールドを users / member_profiles から自動入力
   b. ユーザーが手動で上書き可能
3. POST /forms/templates/{id}/submissions で下書き作成（status = DRAFT）
4. MEMBER が署名欄に手書き署名（Canvas → PNG → S3 アップロード）
5. POST /forms/submissions/{id}/submit で提出
   a. 必須フィールドのバリデーション
   b. status を SUBMITTED に更新、submitted_at をセット
   c. completed_at をセット（承認不要のため提出 = 完了）
   d. submission_count をインクリメント
   e. ApplicationEvent (FormSubmissionCompletedEvent) を発行
6. 通知サービスがイベントを受信:
   a. ADMIN に通知（「田中太郎が入会申込書を提出しました」）
```

### 提出フロー（承認あり）

```
1. MEMBER が提出（上記フローのステップ1〜4と同じ）
2. POST /forms/submissions/{id}/submit で提出
   a. 必須フィールドのバリデーション
   b. status を PENDING に更新、submitted_at をセット
   c. F11.1 の external API を内部呼び出し:
      POST /workflows/templates/{workflowTemplateId}/requests/external
      - source_type: "FORM_SUBMISSION"
      - source_id: form_submissions.id
      - title: "{テンプレート名} — {提出者名}"
      - auto_submit: true
   d. 返却された workflow_request_id を form_submissions に保存
   e. submission_count をインクリメント
3. F11.1 の承認フローが進行（F11.1 のビジネスロジックに従う）
4. 承認完了時:
   a. WorkflowApprovedEvent を FormSubmissionService が購読
   b. source_type = "FORM_SUBMISSION" を確認
   c. form_submissions.status を APPROVED に更新、completed_at をセット
   d. ApplicationEvent (FormSubmissionApprovedEvent) を発行
   e. 通知: 提出者に「{テンプレート名} が承認されました」
5. 却下時:
   a. WorkflowRejectedEvent を購読
   b. form_submissions.status を REJECTED に更新
   c. 通知: 提出者に「{テンプレート名} が却下されました」
6. 差し戻し時:
   a. WorkflowReturnedEvent を購読
   b. form_submissions.status を RETURNED に更新
   c. 通知: 提出者に「{テンプレート名} が差し戻されました。修正して再提出してください」
```

### PDF 生成フロー

```
1. 提出者または ADMIN が PDF 生成 API を呼び出し
2. Service 層で以下を実行:
   a. form_submissions + form_submission_values を取得
   b. form_template_fields の定義に基づきフィールド順にレイアウトを構築
   c. OpenPDF でドキュメントを生成:
      - ヘッダー: テンプレート名 + 提出日時 + 提出者名
      - ボディ: フィールドラベル + 値を順に配置
      - SIGNATURE フィールド: S3 から署名 PNG をダウンロードして画像として埋め込み
      - FILE フィールド: ファイル名のみ記載（添付ファイル自体は別途ダウンロード）
   d. is_seal_on_pdf = TRUE かつ status = APPROVED の場合:
      - workflow_request_approvers から承認者の seal_stamp_logs を取得
      - 各承認者の electronic_seals.svg_data を取得
      - SVG → PNG 変換して PDF のフッター領域に押印画像を配置
      - 押印者名 + 承認日時をキャプション表示
   e. 生成した PDF を S3 にアップロード
   f. form_submissions.pdf_file_key, pdf_generated_at を更新
3. PDF ダウンロード URL は Pre-signed URL（有効期限: 5分）で発行
```

### 自動クローズバッチ

```
1. 日次バッチ（01:00 JST）で実行（@Scheduled）
2. form_templates.status = 'PUBLISHED' かつ deadline IS NOT NULL かつ deadline < 現在日時 を取得
3. 各テンプレートの status を CLOSED に更新、closed_at をセット
4. ApplicationEvent (FormTemplateClosedEvent) を発行
5. 通知: テンプレート作成者（ADMIN）に「{テンプレート名} の回答期限が終了しました」
```

### リマインダーバッチ

```
1. 日次バッチ（09:00 JST）で実行（@Scheduled）
2. form_templates.status = 'PUBLISHED' かつ deadline IS NOT NULL を取得
3. 各テンプレートについて:
   a. deadline の前日（24時間前）かどうかを判定
   b. 前日の場合: 未提出者を特定（スコープ内 MEMBER のうち、form_submissions に DRAFT 以外のレコードがないユーザー）
   c. 未提出者にプッシュ通知（F04.3 連携）: 「{テンプレート名} の提出期限が明日です」
   d. 通知はテンプレートごとに1日1回のみ（重複送信防止）
```

### プレースホルダ自動入力ロジック

```
1. 提出作成時に auto_fill_enabled = TRUE のテンプレートの場合
2. form_template_fields で auto_fill_key が設定されているフィールドを抽出
3. 各 auto_fill_key に対応する値を取得:
   - member_name: users.last_name + ' ' + users.first_name
   - member_email: users.email
   - member_number: member_profiles.member_number（スコープ内）
   - member_phone: users.phone_number
   - team_name: teams.name（scope_type = TEAM の場合）
   - org_name: organizations.name（scope_type = ORGANIZATION の場合）
   - current_date: LocalDate.now().toString()
4. リクエストにユーザーが手動で値を入力済みの場合はそちらを優先
5. 自動入力された値は form_submission_values にそのまま保存
```

### 重要な判定ロジック

- **テンプレートの公開可否**: `DRAFT` かつフィールドが1つ以上存在すること。`requires_approval = TRUE` の場合、`workflow_template_id` が設定済みであること
- **提出回数制限**: `max_submissions_per_user > 0` の場合、`form_submissions` から `submitted_by` + `template_id` で `DRAFT` 以外の件数をカウント。上限到達時は 409 エラー
- **提出の可視性**: ADMIN は全提出を閲覧可能。MEMBER は自分の提出のみ
- **DRAFT 状態での編集**: `form_submission_values` の全フィールドを編集可能
- **RETURNED 状態での編集**: `allow_edit_after_submit = TRUE` の場合のみ編集可能。`FALSE` の場合は再提出不可（ADMIN に問い合わせ）
- **テンプレート削除**: `DRAFT` テンプレートは論理削除可能。`PUBLISHED` / `CLOSED` テンプレートは `CLOSED` に遷移させてからのみ削除可能（提出データの証跡保持のため）
- **CSV エクスポート**: EAV パターンのため、Service 層でピボット処理（フィールドを列として展開）。`SECTION` / `DESCRIPTION` 型はスキップ
- **署名フィールドの検証**: `SIGNATURE` 型の提出値には `file_s3_key` が必須。S3 に実在するかを `HeadObject` で確認

### 通知トリガー一覧

| トリガー | 通知先 | 条件 |
|---------|-------|------|
| テンプレート公開 | スコープ内の全 MEMBER | テンプレートが PUBLISHED に遷移 |
| フォーム提出（承認なし） | ADMIN | MEMBER が SUBMITTED に遷移 |
| フォーム提出（承認あり） | F11.1 の承認者 | MEMBER が PENDING に遷移（F11.1 経由で通知） |
| 承認完了 | 提出者 | WorkflowApprovedEvent 受信時 |
| 却下 | 提出者 | WorkflowRejectedEvent 受信時 |
| 差し戻し | 提出者 | WorkflowReturnedEvent 受信時 |
| リマインダー（自動） | 未提出の MEMBER | deadline の前日（日次バッチ） |
| リマインダー（手動） | 未提出の MEMBER | ADMIN が手動送信 |
| テンプレートクローズ | テンプレート作成者（ADMIN） | deadline 到達で自動クローズ |

## 6. セキュリティ考慮事項

- **認可チェック**: FormTemplateService / FormSubmissionService の入り口で `scope_type` / `scope_id` と `currentUser` の所属を必ず検証する。提出の閲覧は提出者本人・ADMIN に限定
- **レートリミット**: 提出作成 API に `Bucket4j` で 1時間に30回の制限を適用（大量提出によるスパム防止）。リマインダー手動送信は1テンプレートあたり1日3回
- **楽観的ロック**: `form_templates` と `form_submissions` に `version` カラムを適用。更新時に `version` の一致を検証
- **EAV 値のバリデーション**: `form_submission_values` の値を `form_template_fields` の定義に基づき厳密にバリデーション（型チェック・必須チェック・文字数チェック・選択肢チェック・メール形式・電話番号形式）
- **ファイルアップロード**: Pre-signed URL 方式。`contentType` ホワイトリスト検証 + `fileSize` 上限チェック。署名 PNG は最大 500KB、一般ファイルは最大 10MB
- **署名画像のプライバシー**: 署名 PNG の S3 パスは推測不可能な UUID ベース。Pre-signed URL（有効期限: 5分）でのみアクセス可能
- **PDF 内の個人情報**: PDF には提出者の個人情報が含まれるため、ダウンロード URL の有効期限を5分に制限。監査ログにダウンロード操作を記録
- **XSS 防止**: テキスト系フィールドの値は HTML エスケープして PDF に出力。`options_json` の値もサニタイズ
- **CSV インジェクション防止**: CSV エクスポート時、セル値の先頭が `=`, `+`, `-`, `@` の場合はシングルクォートでエスケープ
