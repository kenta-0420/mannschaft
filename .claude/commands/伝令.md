# 伝令（Spring Boot起動 + API仕様取得）

フロントエンド開発に必要なバックエンドサーバーを起動し、OpenAPI仕様（swagger.json）を取得する。
**使い方**: `/陣立て` の後に `/伝令` を実行。この2つでフロントエンド開発の準備が整う。

## 前提条件

- `/陣立て` 済みであること（Docker + ビルド確認が完了している状態）

## 手順

### 1. Spring Boot 起動（既存プロセス確認）

まず 8080 ポートで既に起動していないか確認する:

```bash
if curl -s -o /dev/null -w "%{http_code}" http://localhost:8080/v3/api-docs 2>/dev/null | grep -q "200"; then
  echo "Spring Boot は既に起動中です。swagger.json 取得に進みます。"
fi
```

起動していない場合のみバックグラウンドで起動する:

```bash
./gradlew bootRun &
```

起動完了を待機（最大90秒）:

```bash
echo "Spring Boot 起動待機中..."
for i in $(seq 1 90); do
  if curl -s -o /dev/null -w "%{http_code}" http://localhost:8080/v3/api-docs 2>/dev/null | grep -q "200"; then
    echo "起動確認 (${i}秒)"
    break
  fi
  sleep 1
  if [ "$i" -eq 90 ]; then
    echo "ERROR: 90秒以内に起動しませんでした"
    exit 1
  fi
done
```

### 2. swagger.json 取得

フロントエンドリポジトリに出力する:

```bash
curl -s http://localhost:8080/v3/api-docs -o /c/mannschaft/frontend/swagger.json
echo "swagger.json → C:\\mannschaft\\frontend\\swagger.json"
```

取得内容を検証:

```bash
if [ -s /c/mannschaft/frontend/swagger.json ] && grep -q '"openapi"' /c/mannschaft/frontend/swagger.json; then
  echo "検証OK: OpenAPI仕様を正常に取得"
  PATHS=$(grep -c '"/' /c/mannschaft/frontend/swagger.json || echo "0")
  echo "パス数（概算）: ${PATHS}"
else
  echo "ERROR: swagger.json の取得に失敗しました"
fi
```

### 3. 結果報告

**Spring Boot は停止しない**。フロントエンド開発中のAPI通信に必要なため、起動したまま維持する。

| 項目 | 状態 |
|------|------|
| Spring Boot 起動 | ✅ / ❌ |
| swagger.json 取得 | ✅ / ❌ |

全て ✅ なら「**伝令完了。バックエンド稼働中、API仕様をフロントエンド陣営に届けました。**」と報告する。
❌ がある場合は原因を調査し、対処方法を提示する。

## 備考

- Spring Boot の停止は `/撤収` で Docker と一緒に行う
- swagger.json の再取得だけしたい場合（APIを変更した後など）は、Spring Boot が既に起動中ならステップ2のみ実行される
