# 陣立て2（開発環境セットアップ — PC2）

**PC2 固有の注意事項：**
- Docker は Docker Desktop ではなく **WSL2 内のエンジン**（`dockerd`）を使用。Docker Desktop は未使用・未インストール。
- Windows 側の `localhost` から WSL2 内 Docker コンテナには原則届かないため、**ミドルウェア確認・Spring Boot 起動はWSL2内から**行う。

以下の手順を順番に実行し、開発環境を整えてください。

## 1. WSL2 起動 & キープアライブ

WSL2 はアイドル時に自動停止するため、`setsid` でセッション分離してバックグラウンドプロセスを維持する。

```bash
wsl -d Ubuntu-22.04 -- bash -c "(setsid nohup sleep 86400 </dev/null >/tmp/keep.log 2>&1 &) ; sleep 1 ; pgrep -af 'sleep 86400'"
```

`pgrep` でプロセス番号が表示されれば成功。

## 2. Docker コンテナ起動

WSL2 内の Docker エンジンで MySQL 8.0 + Valkey 8 を起動する。

```bash
wsl -d Ubuntu-22.04 -- bash -c "docker compose -f /mnt/d/mannschaft/docker-compose.yml up -d 2>&1"
```

## 3. ミドルウェア接続確認

**WSL2 内から** コンテナの疎通を確認する（Windows 側 localhost からは届かない）。
※ PC2 ユーザーは `docker` グループに参加済みのため `sudo` 不要。

```bash
wsl -d Ubuntu-22.04 -- bash -c "docker compose -f /mnt/d/mannschaft/docker-compose.yml ps 2>&1"
```

MySQL が `healthy` / `running`、Valkey が `running` であること。

```bash
wsl -d Ubuntu-22.04 -- bash -c "docker exec mannschaft-mysql mysql -u mannschaft -pmannschaft -e 'SELECT 1' 2>&1 && echo 'MySQL OK'"
wsl -d Ubuntu-22.04 -- bash -c "docker exec mannschaft-valkey redis-cli ping 2>&1"
```

MySQL が `1 / MySQL OK`、Valkey が `PONG` であること。

## 4. Gradle ビルド確認

テストはスキップして、JAR をビルドする（Gradle は Windows 側から実行してよい）。

```bash
export JAVA_HOME="/c/Program Files/Eclipse Adoptium/jdk-21.0.10.7-hotspot" && cd D:/mannschaft/backend && ./gradlew bootJar -x test
```

## 5. Spring Boot 起動

**WSL2 内の Java** で起動する（Windows 側 JVM から Docker 内 MySQL へは接続不可のため）。

```bash
wsl -d Ubuntu-22.04 -- bash -c "mkdir -p /mnt/d/mannschaft/backend/logs && > /mnt/d/mannschaft/backend/logs/bootRun.log"
```

```bash
wsl -d Ubuntu-22.04 -- bash -c "
  (setsid nohup java -Xmx1g -XX:+UseG1GC \
    -jar /mnt/d/mannschaft/backend/build/libs/app-0.0.1-SNAPSHOT.jar \
    >> /mnt/d/mannschaft/backend/logs/bootRun.log 2>&1 &)
  sleep 2
  pgrep -af 'app-0.0.1-SNAPSHOT' && echo 'JVM started'
"
```

起動確認（最大240秒待機 — Flyway 460本以上のマイグレーションがあるためコールドスタートは時間がかかる）：

```bash
wsl -d Ubuntu-22.04 -- bash -c "
for i in \$(seq 1 30); do
  if curl -s --connect-timeout 3 http://localhost:8080/actuator/health 2>/dev/null | grep -q '\"status\":\"UP\"'; then
    echo 'Spring Boot UP'; exit 0
  fi
  echo \"Waiting... (\$i/30)\"; sleep 8
done
echo 'TIMEOUT — ログを確認:'
tail -30 /mnt/d/mannschaft/backend/logs/bootRun.log
"
```

タイムアウトしてもJVMが生存中（`pgrep java`）かつCPU使用中なら、Flywayマイグレーション実行中の可能性があるので待ち続けること（実測最大21分）。

## 6. 結果報告

全ステップの結果を以下の形式で簡潔に報告する：

| 項目 | 状態 |
|------|------|
| WSL2 | ✅ / ❌ |
| MySQL | ✅ / ❌ |
| Valkey | ✅ / ❌ |
| Gradle ビルド（bootJar） | ✅ / ❌ |
| Spring Boot (8080) | ✅ / ❌ |

全て ✅ なら「**陣立て完了。殿のご下命をお待ちしております。**」と報告する。
❌ がある場合は原因を調査し、自動修復を試みた上で再報告する。
