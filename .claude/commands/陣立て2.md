# 陣立て2（開発環境セットアップ — PC2）

以下の手順を順番に実行し、開発環境を整えてください。

## 1. WSL2 起動 & キープアライブ

WSL2 はアイドル時に自動停止するため、バックグラウンドプロセスで維持する。

```bash
wsl -d Ubuntu-22.04 -- bash -c "nohup sleep infinity </dev/null &>/dev/null & disown; echo 'WSL2 keepalive started'"
```

## 2. Docker コンテナ起動

MySQL 8.0 + Valkey 8 を docker compose で起動する。

```bash
wsl -d Ubuntu-22.04 -- bash -c "sudo docker compose -f /mnt/d/mannschaft/docker-compose.yml up -d"
```

## 3. ミドルウェア接続確認

MySQL と Valkey が Windows 側の localhost から到達可能か確認する。
両方とも `TcpTestSucceeded: True` であること。

```bash
powershell.exe -Command "
  Write-Host '=== MySQL (3306) ===' ;
  (Test-NetConnection -ComputerName localhost -Port 3306 -WarningAction SilentlyContinue).TcpTestSucceeded ;
  Write-Host '=== Valkey (6379) ===' ;
  (Test-NetConnection -ComputerName localhost -Port 6379 -WarningAction SilentlyContinue).TcpTestSucceeded
"
```

接続できない場合は、コンテナの状態を確認して対処すること：
```bash
wsl -d Ubuntu-22.04 -- bash -c "sudo docker compose -f /mnt/d/mannschaft/docker-compose.yml ps"
```

## 4. Gradle ビルド確認

テストはスキップして、コンパイルが通ることを確認する。

```bash
export JAVA_HOME="/c/Program Files/Eclipse Adoptium/jdk-21.0.10.7-hotspot" && cd D:/mannschaft/backend && ./gradlew build -x test
```

## 5. Spring Boot 起動

`gradlew bootRun` は Windows で JVM クラッシュ（0xC0000005）が発生するため、
`bootJar` でビルドし `java -jar` で直接起動する。

```bash
export JAVA_HOME="/c/Program Files/Eclipse Adoptium/jdk-21.0.10.7-hotspot" && cd D:/mannschaft/backend && ./gradlew bootJar -x test && nohup java -Xmx1g -XX:+UseG1GC -XX:+HeapDumpOnOutOfMemoryError -XX:HeapDumpPath=logs/heap-dump.hprof -jar build/libs/app-0.0.1-SNAPSHOT.jar >logs/bootRun.log 2>&1 &
```

起動確認（最大60秒待機）：

```bash
for i in $(seq 1 12); do
  if curl -s --connect-timeout 3 http://localhost:8080/actuator/health | grep -q '"status":"UP"'; then
    echo "Spring Boot UP"; break
  fi
  echo "Waiting... ($i/12)"; sleep 5
done
```

## 6. 結果報告

全ステップの結果を以下の形式で簡潔に報告する：

| 項目 | 状態 |
|------|------|
| WSL2 | ✅ / ❌ |
| MySQL | ✅ / ❌ |
| Valkey | ✅ / ❌ |
| Gradle ビルド | ✅ / ❌ |
| Spring Boot (8080) | ✅ / ❌ |

全て ✅ なら「**陣立て完了。殿のご下命をお待ちしております。**」と報告する。
❌ がある場合は原因を調査し、自動修復を試みた上で再報告する。
