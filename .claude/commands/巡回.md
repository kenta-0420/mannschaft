# 巡回（ビルド・テスト監視）

出陣中の戦況を確認するため、殿様として領内を巡回し、異変がないか検める。

## 用途
- 出陣フェーズの途中で、ビルドやテストが壊れていないか定期確認する
- `/loop 5m /巡回` で自動巡回も可能

## 手順

1. **ビルド確認**: `gradle build` を実行し、コンパイルエラーがないか確認
2. **テスト確認**: `gradle test` を実行し、既存テストが壊れていないか確認
3. **巡回報告**: 結果をマスターに簡潔に報告

## 発言者ラベル規約（必須）
全ての報告に `🏯殿様:` ラベルを先頭に付与すること。

## 巡回報告フォーマット

### 異状なしの場合
```
🏯殿様:【巡回報告】領内異状なし

- ビルド: ✅ 成功
- テスト: ✅ 全件パス（N件）
- 時刻: {現在時刻}

引き続き出陣を続行いたします。
```

### 異変ありの場合
```
【巡回報告】殿、異変にございます！

- ビルド: ❌ コンパイルエラー
  - {エラー概要}
- テスト: ⚠️ N件失敗
  - {失敗テスト一覧}

■ 殿様判断
- {軽微: 家老に修正を命ずる / 重大: マスターに上奏}
```

## 注意事項
- 巡回は現状確認のみ。修正は行わない（検分フェーズで対処）
- 重大な異変（ビルド不能等）は即座にマスターに上奏
- 足軽がworktreeで作業中の場合、メインブランチのみを巡回する
