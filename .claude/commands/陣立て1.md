# 陣立て1（開発環境セットアップ — PC1）

以下の手順を順番に実行し、開発環境を整えてください。

## 1. WSL2 keepalive 設定

WSL2 はアイドル時に自動停止するため、setsid で確実にバックグラウンド維持する。
**keepalive なしだとコンテナが再起動して Spring Boot 起動時に MySQL が未起動になる。**

```bash
wsl -d Ubuntu-24.04 -- bash -c "(setsid nohup sleep 86400 </dev/null >/tmp/keep.log 2>&1 &); sleep 1; pgrep -af 'sleep 86400'"
```

出力に PID が表示されれば OK。

## 2. WSL2 mirroredモード確認

PC1 では `.wslconfig` に `networkingMode=mirrored` が必要。
ないと Windows JVM から localhost:3306 に接続できない。

```bash
if [ -f "/c/Users/kenta/.wslconfig" ]; then
  echo "EXISTS: $(cat /c/Users/kenta/.wslconfig)"
else
  echo "[wsl2]" > /c/Users/kenta/.wslconfig
  echo "networkingMode=mirrored" >> /c/Users/kenta/.wslconfig
  echo "作成完了 → WSL2再起動が必要"
  powershell.exe -Command "wsl --shutdown"
  wsl -d Ubuntu-24.04 -- bash -c "(setsid nohup sleep 86400 </dev/null >/tmp/keep.log 2>&1 &); sleep 1; pgrep -af 'sleep 86400'"
fi
```

## 3. Docker コンテナ起動

MySQL 8.0 + Valkey 8 を docker compose で起動する。

```bash
wsl -d Ubuntu-24.04 -- bash -c "cd /mnt/c/Claude/mannschaft && docker compose up -d"
```

## 4. MySQL 起動完了待機

**Spring Boot 起動前に必ず MySQL が "ready for connections" になるまで待つこと。**

```bash
wsl -d Ubuntu-24.04 -- bash -c "
for i in \$(seq 1 20); do
  STATUS=\$(docker logs mannschaft-mysql --tail 3 2>&1)
  if echo \"\$STATUS\" | grep -q 'ready for connections'; then
    echo 'MySQL UP'; break
  fi
  echo \"待機 \$i/20...\"; sleep 5
done
"
```

## 5. ミドルウェア接続確認

WSL2内からポート疎通確認（mirroredモードではnetstatで見えなくても実際には繋がる）。

```bash
wsl -d Ubuntu-24.04 -- bash -c "nc -zv localhost 3306 2>&1; nc -zv localhost 6379 2>&1"
```

両方 `succeeded!` であること。

## 6. Gradle ビルド確認

テストはスキップして、コンパイルが通ることを確認する。

```bash
export JAVA_HOME="/c/Program Files/Eclipse Adoptium/jdk-21.0.10.7-hotspot" && cd /c/Claude/mannschaft/backend && ./gradlew bootJar -x test 2>&1 | tail -5
```

## 7. Spring Boot 起動

`gradlew bootRun` は Windows で JVM クラッシュ（0xC0000005）が発生するため、
`bootJar` でビルドし `java -jar` で直接起動する。

```bash
export JAVA_HOME="/c/Program Files/Eclipse Adoptium/jdk-21.0.10.7-hotspot" && cd /c/Claude/mannschaft/backend && nohup java -Xmx1g -XX:+UseG1GC -XX:+HeapDumpOnOutOfMemoryError -XX:HeapDumpPath=logs/heap-dump.hprof -jar build/libs/app-0.0.1-SNAPSHOT.jar >logs/bootRun.log 2>&1 &
echo "PID: $!"
```

起動確認（最大240秒待機）：

```bash
for i in $(seq 1 30); do
  if curl -s --connect-timeout 3 http://localhost:8080/actuator/health | grep -q '"status":"UP"'; then
    echo "Spring Boot UP"; break
  fi
  echo "Waiting... ($i/30)"; sleep 8
done
```

全ステップの結果を以下の形式で簡潔に報告する：

| 項目 | 状態 |
|------|------|
| WSL2 keepalive | ✅ / ❌ |
| mirroredモード | ✅ / ❌ |
| MySQL | ✅ / ❌ |
| Valkey | ✅ / ❌ |
| Gradle ビルド | ✅ / ❌ |

全て ✅ なら「**陣立て完了。殿のご下命をお待ちしております。**」と報告する。
